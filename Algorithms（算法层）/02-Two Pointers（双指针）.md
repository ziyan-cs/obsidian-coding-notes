#算法 #双指针 #指针
## 一页速览

- **双指针**：用两个位置一起移动来处理序列问题
- **核心特点**：不是暴力枚举全部区间，而是利用指针移动缩小范围
- **常见类型**：左右指针、快慢指针、滑动窗口
- **核心优势**：常把 `O(n^2)` 降到 `O(n)` 或 `O(n log n)`
- **常见操作**：夹逼查找、去重、分组、区间维护、链表找环基础
- **高频场景**：有序数组、回文判断、区间问题、链表问题、字符串处理
- **最常错**：左右移动条件写反、窗口更新顺序错、快慢指针关系想乱、以为双指针只能用于数组

## 一、先总后分：这一节到底学什么

### 1. 双指针解决什么问题

- 有些题目本来会想到两层循环暴力枚举
- 但如果序列有顺序性或区间性质
- 就可以用两个指针一起移动，减少重复计算

### 2. 双指针最核心看什么

- 两个指针分别表示什么
- 谁该移动，什么时候移动
- 这个移动是否能保证不漏答案
- 当前题目属于哪一种双指针模型

### 3. 当前阶段先抓主线

- 会理解左右指针和快慢指针
- 会写最基础的有序数组双指针
- 会理解滑动窗口本质也是双指针
- 会根据题目判断该移动左边还是右边

---

## 二、双指针是什么

### 1. 基本理解

- 双指针就是同时维护两个位置
- 让它们按规则移动
- 从而完成查找、统计或维护区间

### 2. 当前阶段怎么理解

- 这两个指针不一定都从头开始
- 也不一定都同速移动
- 关键在于：**两个位置一起工作**

### 3. 当前阶段先记一句话

- **双指针 = 用两个位置的配合代替部分暴力枚举**

---

## 三、双指针的几种常见模型

### 1. 左右指针

- 一个在左，一个在右
- 常用于有序数组、回文串、夹逼查找

### 2. 快慢指针

- 两个指针速度不同
- 常用于链表找中点、找环、删除节点问题

### 3. 滑动窗口

- 本质也是双指针
- 左右边界一起维护一个有效区间

### 4. 当前阶段先记

- **数组题常见左右指针 / 滑动窗口**
- **链表题常见快慢指针**

---

## 四、左右指针

### 1. 基本思路

- 左指针在开头
- 右指针在结尾
- 根据当前情况决定移动哪一边

### 2. 典型例子：有序数组两数之和

```cpp
int l = 0, r = n - 1;
while (l < r) {
	int sum = a[l] + a[r];
	if (sum == target) {
		cout << l << " " << r << '\\n';
		break;
	} else if (sum < target) {
		++l;
	} else {
		--r;
	}
}
```

### 3. 当前阶段怎么理解

- 和太小，就让左边变大
- 和太大，就让右边变小
- 因为数组有序，所以移动方向有依据

### 4. 当前阶段先记

- **有序 + 找和 / 找差 / 夹逼**，很容易想到左右指针

---

## 五、快慢指针

### 1. 基本思路

- 一个指针走得快
- 一个指针走得慢
- 用速度差解决问题

### 2. 典型例子：找链表中点

```cpp
ListNode* slow = head;
ListNode* fast = head;
while (fast != nullptr && fast->next != nullptr) {
	slow = slow->next;
	fast = fast->next->next;
}
```

### 3. 当前阶段怎么理解

- `fast` 每次走两步
- `slow` 每次走一步
- 当 `fast` 到末尾时，`slow` 就在中间附近

### 4. 高频用途

- 找中点
- 判断链表有无环
- 删除倒数第 k 个节点基础

---

## 六、滑动窗口

### 1. 基本思路

- 维护一个区间 `[l, r]`
- 让区间随着条件变化而扩大或缩小
- 常用于子串、子数组、区间统计问题

### 2. 当前阶段怎么理解

- `r` 负责把新元素纳入窗口
- `l` 负责在条件不满足时缩小窗口

### 3. 典型例子：最短满足条件子数组先有印象

```cpp
int l = 0;
for (int r = 0; r < n; ++r) {
	// 加入 a[r]
	while (条件满足) {
		// 更新答案
		// 移走 a[l]
		++l;
	}
}
```

### 4. 当前阶段先记

- **滑动窗口本质就是“区间版双指针”**

---

## 七、为什么双指针能更快

### 1. 暴力思路的问题

- 常常会重复检查很多状态
- 比如每个区间都从头算一遍

### 2. 双指针的优化点

- 指针通常只往前走，不回头
- 所以很多题总移动次数是线性的

### 3. 当前阶段怎么理解

- 不是双指针天然就快
- 而是它利用了“有序性”或“区间可维护性”

### 4. 当前阶段先记

- **双指针快的原因，往往是避免了重复枚举**

---

## 八、双指针最常见应用

### 1. 有序数组两数之和

- 左右夹逼

### 2. 回文串判断

```cpp
int l = 0, r = s.size() - 1;
while (l < r) {
	if (s[l] != s[r]) return false;
	++l;
	--r;
}
return true;
```

### 3. 数组去重

- 常在有序数组中用双指针保留不重复元素

### 4. 子串 / 子数组问题

- 常用滑动窗口

### 5. 链表问题

- 常用快慢指针

---

## 九、左右指针为什么常配合有序数组

### 1. 因为移动方向才有依据

- 在有序数组里，左边增大通常会让值变大
- 右边减小通常会让值变小

### 2. 如果无序

- 就不能直接靠大小判断移动哪边
- 这时双指针未必适用，或者需要先排序

### 3. 当前阶段先记

- **左右指针高频前提：序列有顺序性或单调性**

---

## 十、当前阶段最常见写法

### 1. 左右指针模板

```cpp
int l = 0, r = n - 1;
while (l < r) {
	if (条件成立) {
		// 更新答案
	}
	if (需要左移) ++l;
	else --r;
}
```

### 2. 快慢指针模板

```cpp
auto slow = head;
auto fast = head;
while (fast != nullptr && fast->next != nullptr) {
	slow = slow->next;
	fast = fast->next->next;
}
```

### 3. 滑动窗口模板

```cpp
int l = 0;
for (int r = 0; r < n; ++r) {
	while (窗口不合法) {
		++l;
	}
	// 更新答案
}
```

### 4. 回文判断模板

```cpp
int l = 0, r = s.size() - 1;
while (l < r) {
	if (s[l] != s[r]) return false;
	++l;
	--r;
}
return true;
```

## 十一、现代 C++ 的基础习惯

### 1. 先想清楚两个指针各自代表什么

- 左边界
- 右边界
- 快指针
- 慢指针
- 这一层比代码本身更重要

### 2. 先想“为什么能移动这边”

- 如果移动没有依据
- 很容易漏答案或错答案

### 3. 看到这些关键词先想到双指针

- 有序数组
- 回文
- 子串 / 子数组
- 区间
- 链表中点 / 找环

### 4. 以后会继续补

- 去重模板
- 环形链表完整证明
- 更复杂滑动窗口
- 三数之和类题型
- 但当前阶段先把基础模型学稳

---

## 十二、高频易错点

1. 左右指针移动方向最容易写反
2. 快慢指针循环条件很容易漏判空
3. 滑动窗口更新顺序容易错
4. 双指针不是只会写 `while (l < r)`
5. 左右指针常依赖有序性
6. 回文判断时别忘了同时移动两边
7. 快慢指针不只用于链表，也能用于数组区间思想
8. 先定义指针含义，再写代码，不然很容易乱

---

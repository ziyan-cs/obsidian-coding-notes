#算法 #递归
## 一页速览

- **递归**：函数自己调用自己来解决问题
- **核心特点**：把大问题拆成更小、同类型的子问题
- **核心前提**：必须有递归出口
- **核心优势**：写树、分治、回溯类问题时非常自然
- **常见结构**：递归出口、当前处理、递归子问题
- **高频场景**：树遍历、分治、回溯、递归定义数学问题
- **最常错**：忘写出口、递归方向不收敛、返回值含义不清、把递归过程想乱

## 一、先总后分：这一节到底学什么

### 1. 递归解决什么问题

- 有些问题本身就能拆成很多个更小的同类问题
- 如果每一层做的事情都相似
- 用递归会比手动展开更自然

### 2. 递归最核心看什么

- 递归函数到底表示什么
- 递归出口是什么
- 每一层递归做了什么
- 子问题规模有没有变小

### 3. 当前阶段先抓主线

- 会理解递归的基本思想
- 会写最简单的递归函数
- 会看懂树和分治里的递归写法
- 会判断一段递归为什么可能出错

---

## 二、递归是什么

### 1. 基本理解

- 递归就是函数在自己的函数体里调用自己
- 但不是无脑一直调用
- 而是每次把问题缩小一点

### 2. 一个最基础例子

```cpp
void f(int n) {
	if (n == 0) return;
	cout << n << '\\n';
	f(n - 1);
}
```

### 3. 当前阶段怎么理解

- `f(5)` 会去调用 `f(4)`
- `f(4)` 会去调用 `f(3)`
- 一直缩小到 `f(0)` 停下

### 4. 当前阶段先记一句话

- **递归 = 函数通过不断缩小问题来调用自己**

---

## 三、递归的 3 个核心要素

### 1. 递归函数的定义

- 先说清楚这个函数“要解决什么问题”
- 比如：`f(n)` 表示求前 `n` 项和

### 2. 递归出口

- 什么时候不再继续调用
- 也叫“终止条件”

### 3. 递归转移

- 当前问题如何变成更小的子问题

### 4. 当前阶段先记

- **定义 + 出口 + 缩小问题**，这是递归最核心的框架

---

## 四、为什么一定要有递归出口

### 1. 没有出口会怎样

- 函数会无限调用自己
- 最后导致栈溢出

### 2. 一个错误例子

```cpp
void f(int n) {
	cout << n << '\\n';
	f(n - 1);
}
```

- 这段代码没有停止条件
- 会一直调用下去

### 3. 正确思路

```cpp
void f(int n) {
	if (n == 0) return;
	cout << n << '\\n';
	f(n - 1);
}
```

### 4. 当前阶段先记

- **递归出口不是配角，而是递归能不能成立的前提**

---

## 五、递归为什么要“缩小问题”

### 1. 基本理解

- 每次递归都必须更接近出口
- 否则递归不会结束

### 2. 怎么判断有没有缩小

- 参数是不是变小了
- 状态空间是不是缩小了
- 子树是不是更短了

### 3. 当前阶段怎么理解

- 递归不是“重复调用”这么简单
- 更重要的是“每次调用都比上一次更接近结束”

### 4. 当前阶段先记

- **递归要想正确，必须朝着出口一步步逼近**

---

## 六、最基础递归例子：阶乘

### 1. 问题

- 求 `n!`

### 2. 递归思路

- `n! = n * (n - 1)!`
- 最小情况：`0! = 1`

### 3. 模板

```cpp
int fact(int n) {
	if (n == 0) return 1;
	return n * fact(n - 1);
}
```

### 4. 当前阶段怎么理解

- 当前层只做一件事：乘上 `n`
- 剩下的交给更小的子问题去算

---

## 七、最基础递归例子：斐波那契先有印象

### 1. 定义

- `f(n) = f(n - 1) + f(n - 2)`

### 2. 写法

```cpp
int fib(int n) {
	if (n <= 1) return n;
	return fib(n - 1) + fib(n - 2);
}
```

### 3. 当前阶段怎么理解

- 这个例子很适合理解递归定义
- 但效率不高
- 所以后面会学记忆化和 DP

### 4. 当前阶段先记

- **不是所有递归都高效，但很多递归都很自然**

---

## 八、递归调用栈怎么理解

### 1. 基本理解

- 每次函数调用，系统都会保存当前现场
- 等子调用返回后，再继续执行后面的代码

### 2. 一个简单例子

```cpp
void f(int n) {
	if (n == 0) return;
	cout << "enter " << n << '\\n';
	f(n - 1);
	cout << "back " << n << '\\n';
}
```

### 3. 当前阶段怎么理解

- 递归先一路往下压
- 到出口后，再一层层返回

### 4. 当前阶段先记

- **递归的执行顺序常常是“先下去，再回来”**

---

## 九、递归在树里为什么特别自然

### 1. 因为树本身就是递归结构

- 一个节点下面还是左子树和右子树
- 子树本身还是树

### 2. 典型例子：二叉树遍历

```cpp
void preorder(TreeNode* root) {
	if (root == nullptr) return;
	cout << root->val << '\\n';
	preorder(root->left);
	preorder(root->right);
}
```

### 3. 当前阶段怎么理解

- 先处理当前节点
- 剩下就是“同样的问题”交给左右子树

### 4. 当前阶段先记

- **树题很多时候一眼就该想到递归**

---

## 十、递归和分治的关系

### 1. 分治是什么

- 把大问题拆成小问题
- 分别解决后再合并

### 2. 为什么常配合递归

- 因为“拆成更小问题”本来就很适合递归表达

### 3. 例子先有印象

- 快速排序
- 归并排序
- 二分查找的递归版

### 4. 当前阶段先记

- **递归常是分治的表达方式之一**

---

## 十一、什么时候适合用递归

### 1. 问题本身有递归定义

- 阶乘
- 斐波那契
- 汉诺塔

### 2. 数据结构本身有递归性质

- 树
- 图的 DFS 基础

### 3. 需要“枚举所有可能”

- 回溯
- 全排列
- 子集问题

### 4. 当前阶段先记

- **树、分治、回溯，是递归最高频的 3 大场景**

---

## 十二、当前阶段最常见写法

### 1. 基础递归模板

```cpp
void dfs(int n) {
	if (出口条件) return;
	// 处理当前层
	dfs(更小子问题);
}
```

### 2. 有返回值递归模板

```cpp
int solve(int n) {
	if (出口条件) return 基础答案;
	return 由更小子问题组成的答案;
}
```

### 3. 树递归模板

```cpp
void traverse(TreeNode* root) {
	if (root == nullptr) return;
	traverse(root->left);
	traverse(root->right);
}
```

### 4. 当前阶段怎么理解

- 真正变化的，通常不是框架
- 而是“当前层做什么”和“出口是什么”

---

## 十三、现代 C++ 的基础习惯

### 1. 先定义函数含义再写递归

- 比如先说清：`dfs(u)` 到底表示什么
- 否则递归会很容易写乱

### 2. 先写出口，再写递归体

- 这是最稳的顺序
- 也更不容易漏终止条件

### 3. 看到这些关键词先想到递归

- 树
- 分治
- 回溯
- 自相似结构
- 一层层拆小问题

### 4. 以后会继续补

- 记忆化搜索
- 回溯完整模板
- 递归转迭代
- 栈帧和调用栈更细节的理解
- 但当前阶段先把基本递归框架学稳

---

## 十四、高频易错点

1. 忘记写递归出口
2. 子问题没有变小，导致无限递归
3. 不清楚函数含义，递归体越写越乱
4. 返回值递归时容易漏掉 `return`
5. 递归调用顺序不同，结果可能完全不同
6. 递归不一定高效，有时会重复计算
7. 树题里左右子树递归顺序不要写反
8. 先想“递归函数表示什么”，再想“怎么调用自己”

---

## 十五、最简模板

### 基础递归模板

```cpp
void solve(int n) {
	if (n == 0) return;
	// 处理当前层
	solve(n - 1);
}
```

### 阶乘模板

```cpp
int fact(int n) {
	if (n == 0) return 1;
	return n * fact(n - 1);
}
```

### 树递归模板

```cpp
void preorder(TreeNode* root) {
	if (root == nullptr) return;
	cout << root->val << '\\n';
	preorder(root->left);
	preorder(root->right);
}
```

### 返回值递归模板

```cpp
int solve(int n) {
	if (基础情况) return 基础答案;
	return solve(更小问题);
}
```

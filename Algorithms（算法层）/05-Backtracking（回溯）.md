#算法 #回溯 #递归
## 一页速览

- **回溯**：通过试探、递归、撤销选择来枚举所有可能解
- **核心特点**：先做选择，发现不合适就退回来重新选
- **核心本质**：递归 + 状态恢复
- **核心优势**：适合求所有方案、所有排列、所有组合
- **常见结构**：路径、选择列表、结束条件、撤销操作
- **高频场景**：全排列、组合、子集、括号生成、N 皇后、数独基础
- **最常错**：忘记撤销状态、结束条件写错、路径和答案混用、把回溯和普通递归混成一类

## 一、先总后分：这一节到底学什么

### 1. 回溯解决什么问题

- 有些题目不是只求一个答案
- 而是要求出所有可能方案
- 或者要求在很多选择里找到满足条件的那些解
- 这时回溯会非常自然

### 2. 回溯最核心看什么

- 当前路径表示什么
- 当前还能选什么
- 什么时候得到一个完整解
- 回来时要撤销什么状态

### 3. 当前阶段先抓主线

- 会理解回溯的基本思想
- 会写最基础的回溯模板
- 会看懂排列、组合、子集类题目
- 会知道为什么一定要“做选择后再撤销”

---

## 二、回溯是什么

### 1. 基本理解

- 回溯就是先尝试一种选择
- 如果后面走不通，就退回来
- 再尝试别的选择

### 2. 怎么理解

- 很像走迷宫
- 先走一条路
- 发现死路就回头
- 再换一条路继续试

### 3. 当前阶段先记一句话

- **回溯 = 试一个，走不通就退回来重试**

---

## 三、回溯和递归的关系

### 1. 为什么回溯常写成递归

- 因为每往下一层，就是在处理更小的一步选择
- 递归很适合表示这种“一层一层试”的过程

### 2. 但回溯不等于所有递归

- 递归更广
- 回溯是递归的一类高频应用
- 它特别强调“尝试”和“撤销”

### 3. 当前阶段先记

- **回溯本质上是带撤销操作的递归搜索**

---

## 四、回溯的 4 个核心要素

### 1. 路径 `path`

- 当前已经做出的选择

### 2. 选择列表

- 当前还能选什么

### 3. 结束条件

- 什么时候说明已经形成一个完整解

### 4. 撤销选择

- 当前这条路走完后，要把状态恢复

### 5. 当前阶段先记

- **路径、选择、结束条件、撤销**，是回溯最核心的框架

---

## 五、为什么一定要撤销选择

### 1. 基本理解

- 你试过一个选择之后
- 还要回到原来的状态，去试下一个选择

### 2. 如果不撤销会怎样

- 状态会被污染
- 后面的分支会在错误基础上继续跑
- 最后结果会错

### 3. 一个最基础结构

```cpp
path.push_back(x);
dfs(...);
path.pop_back();
```

### 4. 当前阶段先记

- **回溯最容易漏的，不是递归调用，而是递归回来的那一步恢复现场**

---

## 六、回溯模板先记住

### 1. 通用模板

```cpp
void dfs(...) {
	if (结束条件) {
		ans.push_back(path);
		return;
	}

	for (每一个可选项) {
		做选择;
		dfs(...);
		撤销选择;
	}
}
```

### 2. 当前阶段怎么理解

- 先判断当前是不是一个完整解
- 如果不是，就尝试所有可能选择
- 每试一个，都要在回来时恢复状态

### 3. 当前阶段先记

- **回溯模板真正固定的，不是题目细节，而是“选、递归、撤销”这 3 步**

---

## 七、最基础例子：全排列

### 1. 问题理解

- 给定若干不同数字
- 求它们所有可能排列顺序

### 2. 回溯思路

- 每一层决定当前位置放哪个数
- 已经用过的数不能重复选

### 3. 模板

```cpp
vector<vector<int>> ans;
vector<int> path;
bool used[100] = {false};

void dfs(vector<int>& nums) {
	if (path.size() == nums.size()) {
		ans.push_back(path);
		return;
	}

	for (int i = 0; i < nums.size(); ++i) {
		if (used[i]) continue;
		used[i] = true;
		path.push_back(nums[i]);
		dfs(nums);
		path.pop_back();
		used[i] = false;
	}
}
```

### 4. 当前阶段怎么理解

- `used[i]` 记录这个数有没有被选过
- 选了就标记
- 回来时要取消标记

---

## 八、最基础例子：子集

### 1. 问题理解

- 给定一个集合
- 求它所有子集

### 2. 回溯思路

- 对每个元素，都有两种选择
    - 选
    - 不选

### 3. 模板

```cpp
vector<vector<int>> ans;
vector<int> path;

void dfs(vector<int>& nums, int start) {
	ans.push_back(path);
	for (int i = start; i < nums.size(); ++i) {
		path.push_back(nums[i]);
		dfs(nums, i + 1);
		path.pop_back();
	}
}
```

### 4. 当前阶段怎么理解

- 每到一个状态，当前路径本身就是一个合法子集
- 然后继续尝试往后加元素

---

## 九、最基础例子：组合

### 1. 问题理解

- 从 `1 ~ n` 中选出 `k` 个数

### 2. 回溯思路

- 当前层选一个数
- 下一层只能从后面继续选
- 这样可以避免重复

### 3. 当前阶段怎么理解

- 组合和排列的区别，不在代码长短
- 而在于“顺序是否重要”
- 组合里 `1,2` 和 `2,1` 算同一种

### 4. 当前阶段先记

- **排列常用 `used[]`**
- **组合常用 `start` 控制起点**

---

## 十、回溯为什么常会超时

### 1. 因为它本质在枚举很多可能

- 分支一多，状态数会迅速膨胀

### 2. 当前阶段怎么理解

- 回溯本来就不轻
- 所以能剪枝时一定要剪

### 3. 什么叫剪枝

- 某条路已经不可能得到合法答案
- 那就不要继续往下搜了

### 4. 当前阶段先记

- **回溯的核心除了会搜，还要会尽早停掉没必要的分支**

---

## 十一、剪枝先有印象

### 1. 基本理解

- 剪枝就是提前结束不可能成功的递归分支

### 2. 一个简单例子

- 选组合时，剩余数字数量不够了
- 那就没必要继续选下去

### 3. 当前阶段怎么理解

- 剪枝不会改变答案正确性
- 它只是减少无效搜索

### 4. 当前阶段先记

- **回溯题会不会写是一回事，会不会剪枝又是一回事**

---

## 十二、回溯最常见应用

### 1. 全排列

- 每个位置选一个还没用过的元素

### 2. 组合

- 从后面继续选，避免重复

### 3. 子集

- 每个元素本质上都在“选 / 不选”之间

### 4. 括号生成

- 左右括号数量受限制
- 非法状态要及时剪枝

### 5. 棋盘问题

- N 皇后
- 数独
- 迷宫搜索类问题

---

## 十三、当前阶段最常见写法

### 1. 回溯通用模板

```cpp

	if (结束条件) {
		ans.push_back(path);
		return;
	}
	for (每个选择) {
		做选择;
		dfs(...);
		撤销选择;
	}
}
```

### 2. 排列模板核心

```cpp
if (used[i]) continue;
used[i] = true;
path.push_back(nums[i]);
dfs(nums);
path.pop_back();
used[i] = false;
```

### 3. 组合 / 子集模板核心

```cpp

	path.push_back(nums[i]);
	dfs(nums, i + 1);

```

### 4. 当前阶段怎么理解

- 排列重点是“这个元素用没用过”
- 组合重点是“下一层从哪里开始选”

## 十四、现代 C++ 的基础习惯

### 1. 先定义清楚路径含义

- `path` 到底表示当前已选元素
- 还是当前构造中的字符串
- 这一步非常关键

### 2. 先想结束条件，再写循环

- 什么时候收集答案
- 什么时候停止递归
- 先想明白，代码才不容易乱

### 3. 看到这些关键词先想到回溯

- 所有方案
- 所有排列
- 所有组合
- 搜索所有可能
- 棋盘放置

### 4. 以后会继续补

- 去重回溯
- 剪枝优化
- 多维状态回溯
- 回溯和 DFS 的更细区别
- 但当前阶段先把基本模板学稳

---

## 十五、高频易错点

1. 做了选择却忘记撤销
2. 路径 `path` 和答案 `ans` 容易混
3. 结束条件写错会导致漏解或死递归
4. 排列和组合的写法不要混
5. 排列常要 `used[]`，组合常要 `start`
6. 不剪枝时很多题会很慢
7. 回溯不是暴力乱搜，而是有规则地搜索
8. 先想“当前状态表示什么”，再想“下一层怎么选”

---

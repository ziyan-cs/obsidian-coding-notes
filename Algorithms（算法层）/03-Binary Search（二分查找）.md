#算法 #二分查找 #二分 #查找
## 一页速览

- **二分查找**：在有序序列里不断折半缩小搜索范围
- **核心前提**：序列有序，或答案具有单调性
- **核心思想**：每次排除一半不可能区域
- **常见类型**：查找某值、找左边界、找右边界、答案二分
- **核心优势**：常把查找复杂度从 `O(n)` 降到 `O(log n)`
- **高频场景**：有序数组查找、边界定位、最小可行值、最大可行值
- **最常错**：循环条件写错、`mid` 计算不稳、左右边界更新反、把“找值”和“找边界”混成一种写法

## 一、先总后分：这一节到底学什么

### 1. 二分查找解决什么问题

- 有些题目如果从头到尾一个个找，效率很低
- 但如果数据有序，或者答案满足单调性
- 就可以每次排除一半范围，更快找到结果

### 2. 二分查找最核心看什么

- 当前搜索区间是什么
- `mid` 表示什么
- 判断后该丢左边还是右边
- 题目是在找“某个值”，还是在找“边界”

### 3. 当前阶段先抓主线

- 会写最基础的有序数组二分
- 会理解左边界和右边界写法
- 会知道答案二分也是二分查找
- 会根据题目判断区间该怎么缩

## 二、二分查找是什么

### 1. 基本理解

- 二分查找就是在有序范围里找目标
- 每次看中间位置
- 根据比较结果，丢掉一半范围

### 2. 为什么能这样做

- 因为序列有序
- 中间值一比较，就能知道哪一半不可能有答案

### 3. 当前阶段先记一句话

- **二分查找 = 利用有序性，每次砍掉一半搜索范围**

---

## 三、最基础的二分查找

### 1. 典型问题

- 在有序数组中找某个值是否存在

### 2. 基础模板

```cpp
int l = 0, r = n - 1;
while (l <= r) {
	int mid = l + (r - l) / 2;
	if (a[mid] == target) {
		return mid;
	} else if (a[mid] < target) {
		l = mid + 1;
	} else {
		r = mid - 1;
	}
}
return -1;
```

### 3. 当前阶段怎么理解

- `a[mid] == target`：找到了
- `a[mid] < target`：答案只可能在右边
- `a[mid] > target`：答案只可能在左边

### 4. 当前阶段先记

- **最基础二分是在有序数组里找一个确定值**

---

## 四、为什么复杂度是 `O(log n)`

### 1. 每次都砍掉一半

- 第 1 次看整个区间
- 第 2 次只剩一半
- 第 3 次再剩一半

### 2. 当前阶段怎么理解

- 范围越来越小
- 所以速度比顺序查找快很多

### 3. 当前阶段先记

- **二分查找快，不是因为比较神奇，而是因为每次排除一半**

---

## 五、`mid` 为什么常这样写

### 1. 常见写法

```cpp
int mid = l + (r - l) / 2;
```

### 2. 为什么不总写 `(l + r) / 2`

- 因为在某些语言和大数据范围下，`l + r` 可能溢出
- `l + (r - l) / 2` 更稳妥

### 3. 当前阶段先记

- **写二分时，优先养成稳妥的 `mid` 写法**

---

## 六、找左边界

### 1. 什么叫左边界

- 在一堆相同值里
- 找最左边那个满足条件的位置

### 2. 典型模板

```cpp
int l = 0, r = n - 1;
while (l <= r) {
	int mid = l + (r - l) / 2;
	if (a[mid] >= target) {
		r = mid - 1;
	} else {
		l = mid + 1;
	}
}
```

### 3. 当前阶段怎么理解

- 当 `a[mid] >= target` 时
- 说明左边界可能在 `mid` 或更左边
- 所以要继续收缩右边界

### 4. 当前阶段先记

- **找左边界时，满足条件后常继续往左收**

---

## 七、找右边界

### 1. 什么叫右边界

- 在一堆相同值里
- 找最右边那个满足条件的位置

### 2. 典型模板

```cpp
int l = 0, r = n - 1;
while (l <= r) {
	int mid = l + (r - l) / 2;
	if (a[mid] <= target) {
		l = mid + 1;
	} else {
		r = mid - 1;
	}
}
```

### 3. 当前阶段怎么理解

- 当 `a[mid] <= target` 时
- 说明右边界可能在 `mid` 或更右边
- 所以要继续收缩左边界向右找

### 4. 当前阶段先记

- **找右边界时，满足条件后常继续往右收**

---

## 八、为什么“找值”和“找边界”不能完全混写

### 1. 找值的目标

- 找到一个位置就可以停

### 2. 找边界的目标

- 找到了还不能停
- 因为还要继续确认是不是最左或最右

### 3. 当前阶段怎么理解

- 这两类题看起来很像
- 但循环里的边界更新逻辑不一样

### 4. 当前阶段先记

- **二分最容易错的，不是二分本身，而是题型没分清**

---

## 九、答案二分先有印象

### 1. 什么叫答案二分

- 不一定在数组里二分
- 而是在“答案范围”上二分

### 2. 典型思路

- 猜一个答案 `mid`
- 检查这个答案是否可行
- 如果可行，就尝试更小或更大
- 如果不可行，就往另一边找

### 3. 当前阶段怎么理解

- 本质还是二分
- 只是二分的对象从“下标”变成了“答案空间”

### 4. 当前阶段先记

- **只要答案具有单调性，就可能能做答案二分**

---

## 十、什么时候能想到二分查找

### 1. 明显有序数组

- 最直接的二分场景

### 2. 要找边界

- 第一个大于等于某值
- 最后一个小于等于某值
- 这类都很像边界二分

### 3. 答案有单调性

- 某个值可行，比它更大也都可行
- 或某个值不可行，比它更小也都不可行

### 4. 当前阶段先记

- **看到“有序”或“单调”，就要立刻想到二分**

---

## 十一、当前阶段最常见写法

### 1. 找某个值模板

```cpp
int l = 0, r = n - 1;
while (l <= r) {
	int mid = l + (r - l) / 2;
	if (a[mid] == target) return mid;
	if (a[mid] < target) l = mid + 1;
	else r = mid - 1;
}
return -1;
```

### 2. 找左边界模板

```cpp
int l = 0, r = n - 1;
while (l <= r) {
	int mid = l + (r - l) / 2;
	if (a[mid] >= target) r = mid - 1;
	else l = mid + 1;
}
```

### 3. 找右边界模板

```cpp
int l = 0, r = n - 1;
while (l <= r) {
	int mid = l + (r - l) / 2;
	if (a[mid] <= target) l = mid + 1;
	else r = mid - 1;
}
```

### 4. 答案二分模板

```cpp
int l = 最小答案, r = 最大答案;
while (l <= r) {
	int mid = l + (r - l) / 2;
	if (check(mid)) {
		r = mid - 1;
	} else {
		l = mid + 1;
	}
}
```

## 十二、现代 C++ 的基础习惯

### 1. 先确定区间定义

- 是 `[l, r]` 还是别的形式
- 一旦定了，整段代码都要统一

### 2. 先想清楚循环结束后答案落在哪

- 有些题最后答案在 `l`
- 有些题最后答案在 `r`
- 不要机械背模板不思考

### 3. 看到这些关键词先想到二分

- 有序数组
- 查找某值
- 第一个 / 最后一个满足条件
- 最小可行 / 最大可行
- 单调性

### 4. 以后会继续补

- 开区间二分写法
- 浮点数二分
- 更复杂答案二分
- `lower_bound` / `upper_bound`
- 但当前阶段先把最基本闭区间二分学稳

---

## 十三、高频易错点

1. 二分查找通常依赖有序性或单调性
2. `mid` 推荐写成 `l + (r - l) / 2`
3. `while (l <= r)` 和边界更新必须配套
4. 找值和找边界的写法不能完全混用
5. 找左边界时满足条件后常继续往左缩
6. 找右边界时满足条件后常继续往右缩
7. 循环结束后别忘了检查答案是否合法
8. 二分不是只能查数组，也能二分答案

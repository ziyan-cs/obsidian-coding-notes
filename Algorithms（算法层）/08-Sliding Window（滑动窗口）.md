#算法 #滑动窗口
## 一页速览

- **滑动窗口**：用一个动态区间在序列上滑动来解决问题
- **核心特点**：右边扩张纳入元素，左边收缩移出元素
- **核心本质**：双指针维护一个合法或最优区间
- **核心优势**：常把区间暴力枚举从 `O(n^2)` 优化到 `O(n)`
- **常见结构**：窗口右移、条件检查、窗口收缩、答案更新
- **高频场景**：最长子串、最短子数组、字符统计、区间计数、固定长度窗口
- **最常错**：扩张和收缩顺序写反、窗口条件判断错、答案更新时机不对、把滑动窗口和普通双指针混成一类

## 一、先总后分：这一节到底学什么

### 1. 滑动窗口解决什么问题

- 有些题目需要处理连续子数组或连续子串
- 如果暴力枚举所有区间，通常会很慢
- 这时可以用一个动态区间边走边维护答案

### 2. 滑动窗口最核心看什么

- 窗口 `[l, r]` 表示什么
- 什么时候扩大窗口
- 什么时候缩小窗口
- 什么时候更新答案

### 3. 当前阶段先抓主线

- 会理解滑动窗口的基本思想
- 会写最基础的窗口模板
- 会分清固定窗口和可变窗口
- 会判断窗口该在什么时候收缩

## 二、滑动窗口是什么

### 1. 基本理解

- 滑动窗口本质上是一个连续区间
- 这个区间会随着指针移动不断变化
- 常用两个指针 `l` 和 `r` 表示左右边界

### 2. 怎么理解

- `r` 往右走，表示把新元素加入窗口
- `l` 往右走，表示把左边旧元素移出窗口

### 3. 当前阶段先记一句话

- **滑动窗口 = 用一个可移动区间维护当前有效信息**

---

## 三、滑动窗口和双指针的关系

### 1. 为什么说它属于双指针

- 因为它也有两个位置一起移动
- 左边界和右边界共同维护区间

### 2. 但它和普通左右夹逼不完全一样

- 左右夹逼常是两边朝中间逼近
- 滑动窗口更像“右边扩，左边缩”
- 重点在于维护一个连续区间

### 3. 当前阶段先记

- **滑动窗口是双指针的一种高频模型，专门处理连续区间问题**

---

## 四、什么时候能想到滑动窗口

### 1. 题目对象是连续区间

- 连续子数组
- 连续子串

### 2. 题目问这些内容很常见

- 最长
- 最短
- 满足条件的区间个数
- 某种限制下的最大 / 最小值

### 3. 当前阶段怎么理解

- 只要题目强调“连续”
- 就要立刻想到滑动窗口的可能性

### 4. 当前阶段先记

- **连续子数组 / 子串问题，是滑动窗口的最强信号**

---

## 五、固定长度窗口

### 1. 基本思路

- 窗口长度固定，比如始终是 `k`
- 每次右边加入一个元素
- 如果长度超过 `k`，左边移出一个元素

### 2. 模板

```cpp
int l = 0;
for (int r = 0; r < n; ++r) {
	// 加入 a[r]
	if (r - l + 1 > k) {
		// 移出 a[l]
		++l;
	}
	if (r - l + 1 == k) {
		// 更新答案
	}
}
```

### 3. 当前阶段怎么理解

- 固定窗口重点不在“是否合法”
- 而在于始终保持长度刚好为 `k`

### 4. 高频用途

- 长度为 `k` 的最大和 / 最小和
- 固定长度字符串统计

---

## 六、可变长度窗口

### 1. 基本思路

- 右边先不断扩张
- 当窗口不合法时，再移动左边收缩

### 2. 模板

```cpp
int l = 0;
for (int r = 0; r < n; ++r) {
	// 加入 a[r]
	while (窗口不合法) {
		// 移出 a[l]
		++l;
	}
	// 更新答案
}
```

### 3. 当前阶段怎么理解

- `r` 负责探索更大的区间
- `l` 负责恢复合法性

### 4. 当前阶段先记

- **可变窗口核心就是：右边扩，左边按条件缩**

---

## 七、最经典例子：最长无重复子串

### 1. 问题理解

- 给一个字符串
- 求最长的不含重复字符的连续子串长度

### 2. 滑动窗口思路

- 用窗口维护当前没有重复字符的区间
- 如果加入新字符后出现重复
- 就不断移动左边，直到窗口重新合法

### 3. 模板

```cpp
vector<int> cnt(256, 0);
int l = 0, ans = 0;
for (int r = 0; r < s.size(); ++r) {
	cnt[s[r]]++;
	while (cnt[s[r]] > 1) {
		cnt[s[l]]--;
		++l;
	}
	ans = max(ans, r - l + 1);
}
```

### 4. 当前阶段怎么理解

- 右边先把字符纳入窗口
- 如果重复了，就左边收缩
- 收缩到没有重复为止

---

## 八、最经典例子：最短满足条件子数组

### 1. 问题理解

- 找和至少为 `target` 的最短连续子数组

### 2. 滑动窗口思路

- 右边不断扩张，让和变大
- 一旦已经满足条件，就尽量缩左边，让区间更短

### 3. 模板

```cpp
int l = 0, sum = 0, ans = INT_MAX;
for (int r = 0; r < n; ++r) {
	sum += nums[r];
	while (sum >= target) {
		ans = min(ans, r - l + 1);
		sum -= nums[l];
		++l;
	}
}
```

### 4. 当前阶段怎么理解

- 满足条件后不能立刻停
- 要继续缩左边，看看能不能更短

### 5. 当前阶段先记

- **找最短满足条件区间时，常常是在“满足后继续缩”**

---

## 九、为什么滑动窗口能更快

### 1. 暴力的问题

- 每个区间都重新计算
- 会重复处理很多元素

### 2. 滑动窗口的优化点

- 元素通常进入窗口一次
- 也最多离开窗口一次
- 所以总操作次数接近线性

### 3. 当前阶段怎么理解

- 它不是把所有区间都枚举完整
- 而是边移动边维护结果

### 4. 当前阶段先记

- **滑动窗口快的原因，是避免了重复地重新计算整个区间**

---

## 十、窗口里常维护什么

### 1. 和 `sum`

- 子数组和问题最常见

### 2. 计数数组 / 哈希表

- 统计字符、数字出现次数

### 3. 当前窗口长度

- `r - l + 1`

### 4. 当前最大值 / 最小值先有印象

- 更复杂时会结合单调队列

### 5. 当前阶段先记

- **窗口不是光靠两个指针，还常要配合统计信息一起维护**

---

## 十一、什么时候更新答案

### 1. 固定窗口

- 通常在窗口长度刚好等于目标长度时更新

### 2. 可变窗口

- 常在窗口合法时更新
- 或在满足条件后缩窗口的过程中更新

### 3. 当前阶段怎么理解

- 答案更新时机取决于题目目标
- 最长问题和最短问题，更新时机常不一样

### 4. 当前阶段先记

- **滑动窗口最容易错的不只是怎么缩，还有“什么时候记答案”**

---

## 十二、当前阶段最常见写法

### 1. 固定窗口模板

```cpp
int l = 0;
for (int r = 0; r < n; ++r) {
	// 加入 nums[r]
	if (r - l + 1 > k) {
		// 移出 nums[l]
		++l;
	}
	if (r - l + 1 == k) {
		// 更新答案
	}
}
```

### 2. 可变窗口模板

```cpp
int l = 0;
for (int r = 0; r < n; ++r) {
	// 加入 nums[r]
	while (窗口不合法) {
		// 移出 nums[l]
		++l;
	}
	// 更新答案
}
```

### 3. 字符统计模板

```cpp
vector<int> cnt(256, 0);
int l = 0;
for (int r = 0; r < s.size(); ++r) {
	cnt[s[r]]++;
	while (条件不满足) {
		cnt[s[l]]--;
		++l;
	}
}
```

### 4. 和维护模板

```cpp
int l = 0, sum = 0;
for (int r = 0; r < n; ++r) {
	sum += nums[r];
	while (条件不满足) {
		sum -= nums[l];
		++l;
	}
}
```

## 十三、现代 C++ 的基础习惯

### 1. 先定义窗口含义

- `[l, r]` 里到底表示什么
- 当前窗口是否合法的标准是什么
- 这一步必须先想清楚

### 2. 先想窗口为什么要收缩

- 是因为重复了
- 因为和太大 / 太小
- 因为长度超了
- 收缩条件必须非常明确

### 3. 看到这些关键词先想到滑动窗口

- 连续子串
- 连续子数组
- 最长
- 最短
- 满足条件的区间

### 4. 以后会继续补

- 单调队列优化窗口最值
- 更复杂字符频率窗口
- 至多 / 恰好 / 至少这几类窗口题
- 但当前阶段先把基本模板和合法性判断学稳

---

## 十四、高频易错点

1. 滑动窗口本质是连续区间，不适合不连续选择问题
2. 固定窗口和可变窗口不要混
3. 右边扩张后，左边是否要收缩必须判断清楚
4. 更新答案的时机很容易写错
5. 窗口统计信息要和左右边界同步更新
6. 最短区间问题常在满足条件后继续缩
7. 最长区间问题常在保持合法时更新答案
8. 先想“窗口合法条件是什么”，再写代码

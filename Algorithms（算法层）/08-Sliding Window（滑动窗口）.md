#algorithm #sliding-window #two-pointers

## ⚡ TL;DR（快速决策）

- 题目处理的是连续子串 / 连续子数组 → 先想滑动窗口
- 需要最长 / 最短 / 满足条件的连续区间 → 滑动窗口高频
- 固定长度窗口 → 重点维护窗口大小
- 可变长度窗口 → 重点维护窗口合法性
- 滑动窗口题真正关键通常不在“两个指针怎么动”，而在“什么时候扩右边、什么时候缩左边、什么时候更新答案”

## 🧩 Core Idea（核心本质）

- **本质**：滑动窗口是用一个可移动的连续区间来维护当前有效信息
- **关键机制**：右指针负责扩张窗口纳入新元素，左指针负责在必要时收缩窗口恢复条件
- **核心价值**：常把连续区间的暴力枚举从 `O(n^2)` 优化到 `O(n)`

## 🔧 Usage Patterns（可复用代码模板）

### 1. 固定长度窗口骨架

```cpp
int l = 0;
for (int r = 0; r < n; ++r) {
	// 加入 nums[r]
	if (r - l + 1 > k) {
		// 移出 nums[l]
		++l;
	}
	if (r - l + 1 == k) {
		// 更新答案
	}
}
```

### 2. 可变长度窗口骨架

```cpp
int l = 0;
for (int r = 0; r < n; ++r) {
	// 加入 nums[r]
	while (窗口不合法) {
		// 移出 nums[l]
		++l;
	}
	// 更新答案
}
```

### 3. 最长无重复子串

```cpp
vector<int> cnt(256, 0);
int l = 0, ans = 0;
for (int r = 0; r < s.size(); ++r) {
	cnt[s[r]]++;
	while (cnt[s[r]] > 1) {
		cnt[s[l]]--;
		++l;
	}
	ans = max(ans, r - l + 1);
}
```

### 4. 最短满足条件子数组

```cpp
int l = 0, sum = 0, ans = INT_MAX;
for (int r = 0; r < n; ++r) {
	sum += nums[r];
	while (sum >= target) {
		ans = min(ans, r - l + 1);
		sum -= nums[l];
		++l;
	}
}
```

### 5. 固定窗口维护和

```cpp
int l = 0, sum = 0;
for (int r = 0; r < n; ++r) {
	sum += nums[r];
	if (r - l + 1 > k) {
		sum -= nums[l];
		++l;
	}
}
```

### 6. 字符统计窗口

```cpp
vector<int> cnt(256, 0);
for (int r = 0, l = 0; r < s.size(); ++r) {
	cnt[s[r]]++;
	while (条件不满足) {
		cnt[s[l]]--;
		++l;
	}
}
```

### 7. “满足后继续缩”先有印象

```cpp
while (窗口已经满足条件) {
	更新答案;
	缩左边;
}
```

### 8. “窗口含义先定义清楚”

```cpp
当前窗口 [l, r] 表示什么？
窗口合法条件是什么？
```

## ⚠️ Pitfalls（高频错误）

- 滑动窗口只适用于连续区间，不适合不连续选择问题
- 固定窗口和可变窗口不要混写
- 扩右边后，左边是否要缩必须有明确条件，不能凭感觉写
- 更新答案的时机非常容易错，最长和最短题的时机常不同
- 统计信息必须和指针移动同步更新，否则窗口状态会失真
- `while` 和 `if` 用错，窗口可能缩不够或缩过头
- 字符 / 数值计数题里，忘记移出左端元素时同步减计数是高频 bug
- 很多滑动窗口题真正难点不是模板，而是“窗口合法条件”怎么定义

## 🚀 Performance / Tips（性能优化）

- 一看到“连续子串 / 连续子数组”就要警觉窗口信号
- 滑动窗口真正高频的价值是：

```cpp
元素通常只进窗口一次
也只出窗口一次
```

- 写窗口前优先先想清：

```cpp
窗口里维护什么信息
何时扩右边
何时缩左边
何时更新答案
```

- 最长问题常在“窗口合法时更新”，最短问题常在“满足条件后继续缩时更新”
- 如果条件需要维护最大值 / 最小值，可能要和单调队列结合

## 🧪 Common Scenarios（常见使用场景）

- **最长无重复子串**：字符频率 + 可变窗口
- **最短满足条件子数组**：和维护 + 收缩窗口
- **固定长度最大和 / 最小和**：固定窗口
- **字符统计问题**：频率数组 / 哈希表 + 窗口
- **至多 / 至少 / 恰好类区间问题**：窗口合法性维护
- **连续区间优化问题**：用窗口替代暴力枚举

## 🧾 Minimal Template（最小可运行模板）

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
	ios::sync_with_stdio(false);
	cin.tie(nullptr);

	string s = "abcabcbb";
	vector<int> cnt(256, 0);
	int l = 0, ans = 0;

	for (int r = 0; r < s.size(); ++r) {
		cnt[s[r]]++;
		while (cnt[s[r]] > 1) {
			cnt[s[l]]--;
			++l;
		}
		ans = max(ans, r - l + 1);
	}

	cout << ans << '\n';
	return 0;
}
```

## 📌 One-liner Summary（一句话总结）

👉 滑动窗口的核心不是“两个指针一起动”，而是用 **一个连续区间 + 可维护的窗口状态**，把连续区间问题从重复枚举转成线性推进。

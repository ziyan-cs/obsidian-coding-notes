## 一页速览

- **排序**：把一组数据按一定顺序重新排列
- **最常见目标**：升序或降序
- **核心关注**：时间复杂度、空间复杂度、稳定性
- **常见方法**：冒泡、选择、插入、快速、归并
- **入门阶段重点**：先会写简单排序，再理解高效排序
- **高频场景**：数组处理、去重前预处理、查找前预处理、面试算法题
- **最常错**：循环边界写错、交换条件写反、复杂度记混、稳定性分不清

## 一、先总后分：这一节到底学什么

### 1. 排序解决什么问题

- 很多数据在处理前是无序的
- 排好序后，查找、统计、去重、比较都会更方便
- 所以排序是算法题里的高频基础操作

### 2. 排序最核心看什么

- 排序规则是什么
- 每种方法怎么比较和交换
- 时间复杂度大概是多少
- 什么场景该想到哪种排序

### 3. 当前阶段先抓主线

- 会理解排序的基本目标
- 会写冒泡、选择、插入这 3 个基础排序
- 对快速排序和归并排序先建立正确印象
- 会区分稳定性和复杂度

---

## 二、排序是什么

### 1. 基本理解

- 排序就是把一组数据按某种顺序排列
- 最常见的是从小到大

```cpp
int arr[5] = {5, 2, 4, 1, 3};
```

排序后：

```cpp
1 2 3 4 5
```

### 2. 当前阶段先记一句话

- **排序 = 调整元素位置，让整体更有序**

### 3. 常见目标

- 升序：从小到大
- 降序：从大到小

---

## 三、排序常看的 3 个指标

### 1. 时间复杂度

- 看运行快不快
- 数据量越大，这个越重要

### 2. 空间复杂度

- 看是否需要额外空间
- 有些排序原地完成，有些需要辅助数组

### 3. 稳定性

- 如果两个元素本来相等，排序后相对顺序不变，就叫稳定

### 当前阶段先记

- **快不快**：时间复杂度
- **占不占额外空间**：空间复杂度
- **相等元素顺序变不变**：稳定性

---

## 四、冒泡排序

### 1. 基本思路

- 每一轮都把较大的元素“冒”到后面
- 相邻元素两两比较，不合适就交换

### 2. 最基础写法

```cpp
for (int i = 0; i < n - 1; ++i) {
	for (int j = 0; j < n - 1 - i; ++j) {
		if (arr[j] > arr[j + 1]) {
			swap(arr[j], arr[j + 1]);
		}
	}
}
```

### 3. 当前阶段怎么理解

- 第 1 轮把最大值放到最后
- 第 2 轮把次大值放到倒数第二
- 每轮少看一个尾部元素

### 4. 特点

- 稳定
- 写法直观
- 效率较低

---

## 五、选择排序

### 1. 基本思路

- 每一轮先找到最小值
- 再把它放到当前该放的位置

### 2. 最基础写法

```cpp
for (int i = 0; i < n - 1; ++i) {
	int minPos = i;
	for (int j = i + 1; j < n; ++j) {
		if (arr[j] < arr[minPos]) {
			minPos = j;
		}
	}
	swap(arr[i], arr[minPos]);
}
```

### 3. 当前阶段怎么理解

- 第 1 轮选最小的放第 1 个位置
- 第 2 轮选剩下最小的放第 2 个位置

### 4. 特点

- 不稳定
- 思路清楚
- 效率也不高

---

## 六、插入排序

### 1. 基本思路

- 把数组分成“已排序部分”和“未排序部分”
- 每次取一个新元素，插入到前面合适位置

### 2. 最基础写法

```cpp
for (int i = 1; i < n; ++i) {
	int key = arr[i];
	int j = i - 1;
	while (j >= 0 && arr[j] > key) {
		arr[j + 1] = arr[j];
		--j;
	}
	arr[j + 1] = key;
}
```

### 3. 当前阶段怎么理解

- 前面部分保持有序
- 新元素往前找位置
- 比它大的元素整体后移

### 4. 特点

- 稳定
- 小规模数据表现不错
- 比冒泡和选择更常见一些

---

## 七、快速排序先有印象

### 1. 基本思路

- 选一个基准值 `pivot`
- 把比它小的放左边，比它大的放右边
- 再递归处理左右两边

### 2. 当前阶段怎么理解

- 它的核心不是“不断交换”本身
- 而是“分治 + 划分”

### 3. 特点

- 平均速度很快
- 面试和算法题里很常见
- 但最坏情况会退化
- 一般认为不稳定

### 4. 当前阶段先记

- **高频高效排序，重点理解分治思想**

---

## 八、归并排序先有印象

### 1. 基本思路

- 先不断二分
- 再把两个有序部分合并起来

### 2. 当前阶段怎么理解

- 归并排序重点在“合并两个有序数组”
- 整体也是分治思想

### 3. 特点

- 稳定
- 时间复杂度稳定
- 但需要额外空间

### 4. 当前阶段先记

- **稳定、高效、要额外空间**

---

## 九、几种常见排序先记结论

|排序|平均时间复杂度|稳定性|当前阶段定位|
|---|---|---|---|
|冒泡排序|`O(n^2)`|稳定|入门理解交换过程|
|选择排序|`O(n^2)`|不稳定|入门理解选最值|
|插入排序|`O(n^2)`|稳定|入门重点掌握|
|快速排序|平均 `O(n log n)`|不稳定|高频重点|
|归并排序|`O(n log n)`|稳定|高频重点|

---

## 十一、现代 C++ 的基础习惯

### 1. 刷题先会手写基础排序

- 冒泡、选择、插入最好手写熟
- 它们能帮你建立排序的基本感觉

### 2. 做工程更常直接用库排序

```cpp
#include <algorithm>
sort(arr, arr + n);
```

- 实际开发里，很多时候直接用 `sort`
- 但面试 / 学习阶段还是要理解原理

### 3. 先区分“会写”和“会选”

- 会写基础排序
- 会在题目里判断大概该用哪一类排序

### 4. 当前阶段怎么学最有效

- 先稳住 3 个基础排序
- 再理解快速排序和归并排序的思想
- 最后再背复杂度和稳定性

---

## 十二、高频易错点

1. 内层循环边界很容易写错
2. `j + 1` 访问时更要小心越界
3. 交换条件写反会导致排序方向错
4. `O(n^2)` 和 `O(n log n)` 容易记混
5. 稳定和不稳定容易混
6. 快速排序和归并排序都属于分治思想
7. 插入排序的核心是“后移”，不是频繁交换
8. 工程里常用库排序，考试里常考手写思想

---

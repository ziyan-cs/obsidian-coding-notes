#algorithm #dynamic-programming #dp

## ⚡ TL;DR（快速决策）

- 题目要求最优值、方案数、可行性，且存在重复子问题 → 先想 DP
- 裸递归会重复计算很多相同子问题 → 很可能要做动态规划
- 写 DP 前先确定：状态是什么、怎么转移、初始值是什么、遍历顺序怎么定
- DP 题真正关键通常不在“会不会开数组”，而在“状态定义是否准确”
- 一旦状态定义不清，后面的转移和初始化基本都会跟着乱

## 🧩 Core Idea（核心本质）

- **本质**：动态规划是在保存子问题答案的基础上，递推出更大问题的答案
- **关键机制**：把问题拆成有重叠的子问题，并利用状态转移避免重复计算
- **核心价值**：常把指数级递归优化到多项式复杂度，是求最优值、方案数和可行性问题的高频方法

## 🔧 Usage Patterns（可复用代码模板）

### 1. 一维 DP 骨架

```cpp
vector<int> dp(n + 1);
初始化;
for (int i = 1; i <= n; ++i) {
	dp[i] = 状态转移;
}
```

### 2. 爬楼梯

```cpp
vector<int> dp(n + 1);
dp[0] = 1;
dp[1] = 1;
for (int i = 2; i <= n; ++i) {
	dp[i] = dp[i - 1] + dp[i - 2];
}
```

### 3. 二维 DP 骨架

```cpp
vector<vector<int>> dp(n + 1, vector<int>(m + 1));
初始化边界;
for (int i = 1; i <= n; ++i) {
	for (int j = 1; j <= m; ++j) {
		dp[i][j] = 状态转移;
	}
}
```

### 4. 网格路径

```cpp
for (int i = 1; i <= n; ++i) {
	for (int j = 1; j <= m; ++j) {
		dp[i][j] = dp[i - 1][j] + dp[i][j - 1];
	}
}
```

### 5. 记忆化搜索

```cpp
int dfs(int x) {
	if (memo[x] != -1) return memo[x];
	if (出口条件) return 基础答案;
	return memo[x] = 由更小问题得到的结果;
}
```

### 6. 最值型 DP 先有印象

```cpp
dp[i] = max(dp[i - 1], 某种转移结果);
```

### 7. 方案数型 DP 先有印象

```cpp
dp[i] += dp[i - 1];
```

### 8. 空间压缩先有印象

```cpp
int a = 1, b = 1;
for (int i = 2; i <= n; ++i) {
	int c = a + b;
	a = b;
	b = c;
}
```

## ⚠️ Pitfalls（高频错误）

- 状态定义不清，转移方程大概率会跟着错
- 初始化常常比转移更容易漏
- 遍历顺序必须和依赖关系一致，不能随便写循环方向
- 把 DP 和贪心混掉：DP 是综合多个状态，贪心是当前直接选
- 不是所有递归都值得做 DP，关键看是否有重叠子问题
- 记忆化搜索本质上也是 DP，不要把它当成完全不同的方法
- 空间优化做得太早，常会让逻辑更乱
- 很多 DP 题真正难点不是写代码，而是先把状态翻译对

## 🚀 Performance / Tips（性能优化）

- 写 DP 前优先先想清：

```cpp
dp[i] / dp[i][j] 表示什么
它由哪些状态转移而来
最小状态是什么
```

- DP 真正高频的价值是：

```cpp
避免重复计算
系统化地比较子问题结果
把最优值 / 方案数推出来
```

- 一看到“最优值 / 方案数 / 可不可行”就要警觉 DP 信号
- 重复子问题明显时，可以先写记忆化搜索，再改成自底向上
- 如果状态一团乱，先别急着写代码，先把表格含义写出来

## 🧪 Common Scenarios（常见使用场景）

- **线性 DP**：爬楼梯、打家劫舍、最大子数组和
- **二维 DP**：网格路径、最长公共子序列、编辑距离
- **背包问题**：容量与价值转移
- **区间 DP**：括号、合并、博弈类区间问题
- **记忆化搜索**：递归 + 缓存
- **可行性 / 方案数问题**：是否能做到、一共有多少种方法

## 🧾 Minimal Template（最小可运行模板）

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
	ios::sync_with_stdio(false);
	cin.tie(nullptr);

	int n = 5;
	vector<int> dp(n + 1);
	dp[0] = 1;
	dp[1] = 1;
	for (int i = 2; i <= n; ++i) {
		dp[i] = dp[i - 1] + dp[i - 2];
	}

	cout << dp[n] << '\n';
	return 0;
}
```

## 📌 One-liner Summary（一句话总结）

👉 动态规划的核心不是“套公式填表”，而是先把问题定义成 **可复用的状态**，再通过正确的状态转移和初始化把大问题一步步推出来。

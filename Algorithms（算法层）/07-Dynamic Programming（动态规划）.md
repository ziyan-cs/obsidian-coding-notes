#算法 #动态规划 #DP
## 一页速览

- **动态规划**：把重复子问题的结果保存下来，避免重复计算
- **核心特点**：大问题由小问题转移而来
- **核心前提**：存在重叠子问题和最优子结构
- **核心优势**：常把指数级递归优化到多项式复杂度
- **常见结构**：状态定义、状态转移、初始化、遍历顺序
- **高频场景**：爬楼梯、背包问题、最长子序列、路径计数、字符串匹配基础
- **最常错**：状态定义不清、转移方程写错、初始化漏掉、遍历顺序不对、把贪心题误写成 DP

## 一、先总后分：这一节到底学什么

### 1. 动态规划解决什么问题

- 有些题目直接递归会重复算很多相同子问题
- 有些题目想求最优解、方案数、可行性
- 这时动态规划会比暴力和裸递归更合适

### 2. 动态规划最核心看什么

- `dp[i]` 或 `dp[i][j]` 到底表示什么
- 当前状态从哪些更小状态转移而来
- 初始状态是什么
- 计算顺序该怎么安排

### 3. 当前阶段先抓主线

- 会理解 DP 的基本思想
- 会写最简单的一维 DP
- 会看懂二维 DP 的基本结构
- 会分清“状态定义”和“状态转移”

---

## 二、动态规划是什么

### 1. 基本理解

- 动态规划简称 **DP**
- 核心思想是：
    - 把大问题拆成小问题
    - 先算小问题
    - 再用小问题结果推出大问题结果

### 2. 为什么它叫“动态规划”

- 重点不在“动态”这两个字
- 也不是现实中的项目规划
- 而是在状态之间做有规律的递推

### 3. 当前阶段先记一句话

- **动态规划 = 记录子问题答案，再递推出更大问题答案**

---

## 三、DP 和递归的关系

### 1. 很多 DP 题最开始都能写成递归

- 先定义一个函数表示子问题
- 再看它如何依赖更小问题

### 2. 为什么还要 DP

- 因为纯递归可能重复计算
- DP 会把已经算过的结果保存下来

### 3. 当前阶段怎么理解

- 递归像“从上往下想”
- DP 常像“从下往上填表”

### 4. 当前阶段先记

- **很多 DP 本质上是“优化过的递归”**

---

## 四、什么样的问题适合 DP

### 1. 重叠子问题

- 同一个子问题会反复出现
- 如果不保存结果，就会重复算很多次

### 2. 最优子结构

- 大问题的最优解，可以由小问题的最优解推出

### 3. 当前阶段怎么理解

- 不是所有递归都能变成 DP
- 但如果“小问题结果值得保存”，就很像 DP

### 4. 当前阶段先记

- **重叠子问题 + 最优子结构，是 DP 的高频信号**

---

## 五、写 DP 最核心的 4 步

### 1. 定义状态

- 先说清楚 `dp[i]` / `dp[i][j]` 表示什么

### 2. 写出状态转移

- 当前状态由哪些更小状态推出

### 3. 初始化

- 最小状态的答案是什么

### 4. 确定遍历顺序

- 先算谁，后算谁
- 必须保证转移时依赖的状态已经算好

### 5. 当前阶段先记

- **状态定义、转移方程、初始化、遍历顺序，是写 DP 的固定主线**

---

## 六、最基础例子：爬楼梯

### 1. 问题理解

- 每次可以爬 1 阶或 2 阶
- 问爬到第 `n` 阶有多少种方法

### 2. 状态定义

- `dp[i]` 表示到第 `i` 阶的方法数

### 3. 状态转移

- 到第 `i` 阶，最后一步可能来自：
    - 第 `i - 1` 阶走 1 步
    - 第 `i - 2` 阶走 2 步
- 所以：

```cpp
dp[i] = dp[i - 1] + dp[i - 2];
```

### 4. 模板

```cpp
vector<int> dp(n + 1);
dp[0] = 1;
dp[1] = 1;
for (int i = 2; i <= n; ++i) {
	dp[i] = dp[i - 1] + dp[i - 2];
}
```

### 5. 当前阶段怎么理解

- 当前问题不是单独想出来的
- 而是由前两个状态推出来的

---

## 七、为什么状态定义最重要

### 1. 状态定义不清，后面全乱

- 如果不知道 `dp[i]` 表示什么
- 就写不出正确转移

### 2. 当前阶段怎么理解

- DP 最关键不是先写代码
- 而是先把“每个格子代表什么”说清楚

### 3. 当前阶段先记

- **状态定义清楚了，转移方程通常就会自然很多**

---

## 八、一维 DP 先有感觉

### 1. 常见形式

- `dp[i]` 表示前 `i` 个位置、到第 `i` 个位置、长度为 `i` 的答案

### 2. 高频场景

- 爬楼梯
- 打家劫舍
- 最大子数组和
- 线性路径问题

### 3. 当前阶段怎么理解

- 一维 DP 常沿着数组从左到右推
- 每个位置依赖前面若干位置

### 4. 当前阶段先记

- **线性问题里，最常见的是一维 DP**

---

## 九、二维 DP 先有感觉

### 1. 常见形式

- `dp[i][j]` 表示前 `i` 个和前 `j` 个的关系
- 或表示走到网格 `(i, j)` 的答案

### 2. 高频场景

- 不同路径
- 最长公共子序列
- 编辑距离基础
- 网格类问题

### 3. 一个简单例子：网格路径

```cpp
dp[i][j] = dp[i - 1][j] + dp[i][j - 1];
```

- 到 `(i, j)` 只能从上边或左边过来

### 4. 当前阶段先记

- **两个维度有关系时，常想到二维 DP**

---

## 十、DP 和贪心的区别

### 1. 贪心

- 当前直接做最优选择
- 不回头

### 2. 动态规划

- 会综合多个子状态结果
- 再决定当前状态的最优值

### 3. 当前阶段怎么理解

- 贪心像“现在直接选最好的”
- DP 像“先把各种可能情况算出来，再取最优”

### 4. 当前阶段先记结论

- **能贪心不代表必须 DP**
- **不能贪心时，DP 常是重要备选**

---

## 十一、滚动数组先有印象

### 1. 为什么能优化空间

- 有些 DP 只依赖前一行或前几个状态
- 不需要把整张表都存下来

### 2. 例子

```cpp
int a = 1, b = 1;
for (int i = 2; i <= n; ++i) {
	int c = a + b;
	a = b;
	b = c;
}
```

### 3. 当前阶段怎么理解

- 如果只用到最近几个状态
- 就可以把空间从数组压成几个变量

### 4. 当前阶段先记

- **空间优化是建立在“你已经完全理解原始 DP”之后的事**

---

## 十二、什么时候能想到 DP

### 1. 最优解问题

- 最大值
- 最小值
- 最多 / 最少

### 2. 方案数问题

- 一共有多少种方法
- 有多少种转移方式

### 3. 可行性问题

- 能不能做到
- 是否存在某种方案

### 4. 当前阶段先记

- **看到“最优值”“方案数”“可不可行”，就要想到 DP 的可能性**

---

## 十三、当前阶段最常见写法

### 1. 一维 DP 模板

```cpp
vector<int> dp(n + 1);
初始化 dp[0], dp[1] ...;
for (int i = 2; i <= n; ++i) {
	dp[i] = 由更小状态转移而来;
}
```

### 2. 二维 DP 模板

```cpp
vector<vector<int>> dp(n + 1, vector<int>(m + 1));
初始化边界;
for (int i = 1; i <= n; ++i) {
	for (int j = 1; j <= m; ++j) {
		dp[i][j] = 由相邻状态转移而来;
	}
}
```

### 3. 记忆化搜索模板

```cpp
int dfs(int x) {
	if (已经算过) return memo[x];
	if (出口条件) return 基础答案;
	return memo[x] = 由更小问题计算出来的结果;
}
```

### 4. 当前阶段怎么理解

- 自底向上是常规 DP
- 自顶向下 + 记忆化，也是一种 DP

---

## 十四、现代 C++ 的基础习惯

### 1. 先写状态定义，再写代码

- 比如先说清：`dp[i]` 到底表示什么
- 不要一上来就急着写循环

### 2. 初始化一定单独检查

- 很多 DP 错误不是转移错
- 而是初始状态漏了或错了

### 3. 遍历顺序一定配合转移方向

- 如果当前状态依赖左边，就要从左往右
- 如果依赖上方，就要先处理上一行

### 4. 以后会继续补

- 背包 DP
- 区间 DP
- 状压 DP
- 树形 DP
- 但当前阶段先把最基础线性和网格 DP 学稳

---

## 十五、高频易错点

1. 状态定义不清，转移方程就容易乱写
2. 初始化常常比转移更容易出错
3. 遍历顺序必须和依赖关系匹配
4. DP 不一定只有求最大最小，也可能求方案数
5. 不是所有递归都值得做 DP
6. 记忆化搜索本质上也是 DP
7. 空间优化不要太早做，不然更容易写乱
8. 先问“状态是什么”，再问“怎么转移”

---

## 十六、最简模板

### 一维 DP 模板

```cpp
vector<int> dp(n + 1);
初始化;
for (int i = 1; i <= n; ++i) {
	dp[i] = 状态转移;
}
```

### 二维 DP 模板

```cpp
vector<vector<int>> dp(n + 1, vector<int>(m + 1));
初始化边界;
for (int i = 1; i <= n; ++i) {
	for (int j = 1; j <= m; ++j) {
		dp[i][j] = 状态转移;
	}
}
```

### 爬楼梯模板

```cpp
vector<int> dp(n + 1);
dp[0] = 1;
dp[1] = 1;
for (int i = 2; i <= n; ++i) {
	dp[i] = dp[i - 1] + dp[i - 2];
}
```

### 记忆化搜索模板

```cpp
int dfs(int x) {
	if (memo[x] != -1) return memo[x];
	if (出口条件) return 基础答案;
	return memo[x] = 由更小问题得到的结果;
}
```

#算法 #贪心
## 一页速览

- **贪心**：每一步都先做当前看起来最优的选择
- **核心特点**：不回头修改之前的决定
- **核心前提**：局部最优能够推出全局最优
- **核心优势**：思路直接，代码通常简洁高效
- **常见操作**：排序后选、按规则优先取、区间覆盖、最少数量选择
- **高频场景**：区间问题、跳跃问题、分发问题、调度问题、最小代价基础
- **最常错**：以为“看起来合理”就是贪心、不会证明正确性、局部最优策略选错、把贪心和 DP 混淆

## 一、先总后分：这一节到底学什么

### 1. 贪心解决什么问题

- 有些题目不需要枚举所有可能
- 每一步只要做一个当前最优的选择
- 最后就能得到整体最优答案
- 这类题通常适合贪心

### 2. 贪心最核心看什么

- 当前这一步最优选择是什么
- 为什么做了这个选择不会后悔
- 这种局部最优能不能推出全局最优
- 题目到底能不能贪

### 3. 当前阶段先抓主线

- 会理解贪心的基本思想
- 会识别常见贪心题型
- 会写最基础的排序 + 贪心代码
- 会知道贪心题最关键的是“策略是否正确”

---

## 二、贪心是什么

### 1. 基本理解

- 贪心就是每一步都先选当前最好的
- 选完以后不回头改
- 一路做下去，得到答案

### 2. 怎么理解

- 很像现实里“当前先拿最划算的”
- 但算法题里不能只凭感觉
- 必须这个策略真的正确才行

### 3. 当前阶段先记一句话

- **贪心 = 每一步先做当前最优选择，并且不回头**

---

## 三、贪心为什么难

### 1. 难点不在代码

- 贪心代码往往不长
- 真正难的是：为什么这样选一定对

### 2. 当前阶段怎么理解

- 回溯难在搜索过程
- DP 难在状态转移
- 贪心难在策略证明

### 3. 当前阶段先记

- **贪心题最难的通常不是实现，而是想对策略**

---

## 四、贪心和暴力 / DP 的区别

### 1. 暴力

- 把所有可能都试一遍
- 一定全面，但通常慢

### 2. 动态规划

- 比较多个子问题结果后，再决定最优解
- 会保留“以后再比较”的空间

### 3. 贪心

- 当前直接做最优选择
- 不再回头修改

### 4. 当前阶段先记结论

- **暴力：全试**
- **DP：多方案比较**
- **贪心：当前直接选最优**

---

## 五、什么时候可能想到贪心

### 1. 每一步都有明显的优先规则

- 比如先选结束最早的区间
- 先选当前能跳最远的位置

### 2. 题目目标是“最少”“最多”“尽量大”“尽量小”

- 这类题常有贪心味道

### 3. 排序后规律特别明显

- 很多贪心题都要先排序
- 排完序后策略会更清楚

### 4. 当前阶段先记

- **排序 + 每步选当前最优**，是贪心的高频信号

---

## 六、最经典例子：区间调度

### 1. 问题理解

- 有很多区间
- 想选最多个互不重叠的区间

### 2. 贪心策略

- 按结束时间从小到大排序
- 每次优先选结束最早且不冲突的区间

### 3. 为什么这样自然

- 结束越早，给后面留下的空间越大
- 越可能接更多区间

### 4. 当前阶段先记

- **区间问题里，“按右端点排序”是超高频贪心套路**

---

## 七、最经典例子：分发饼干先有印象

### 1. 问题理解

- 小孩有需求值
- 饼干有大小
- 每块饼干只能给一个小孩
- 想尽量满足更多小孩

### 2. 贪心策略

- 先排序
- 用最小能满足当前小孩的饼干去满足

### 3. 当前阶段怎么理解

- 大饼干不要轻易浪费给需求小的小孩
- 能用小资源解决的，就先用小资源解决

### 4. 当前阶段先记

- **贪心常有“用刚刚好的资源去完成当前任务”的味道**

---

## 八、最经典例子：跳跃问题先有印象

### 1. 问题理解

- 每个位置能向右跳一定距离
- 问能不能到终点，或者最少跳几次

### 2. 贪心思路

- 维护当前最远能到哪里
- 每一步尽量扩展可到达范围

### 3. 当前阶段怎么理解

- 不一定真的把每种跳法都试一遍
- 只关心“最远能扩到哪”

### 4. 当前阶段先记

- **有些贪心不是直接选元素，而是维护一个最优范围**

---

## 九、排序为什么常和贪心一起出现

### 1. 因为排序能让局部选择规则更清楚

- 小的在前，大的在后
- 结束早的在前，开始早的在前
- 这样更容易定义“当前最优”

### 2. 当前阶段怎么理解

- 很多题不排序时完全没头绪
- 一排序，贪心策略就出来了

### 3. 当前阶段先记

- **很多贪心题第一步不是选，而是先排序**

---

## 十、贪心为什么一定要证明

### 1. 不是所有“看起来最优”的策略都对

- 有些策略当前最优
- 但会破坏后面的更优解

### 2. 当前阶段怎么理解

- 贪心最怕“感觉上对”
- 算法题里必须能说清为什么不后悔

### 3. 常见证明感觉先有印象

- 交换论证
- 局部最优推出全局最优
- 反证法

### 4. 当前阶段先记

- **贪心题不会证明，通常就说明策略还没真正想明白**

---

## 十一、当前阶段最常见写法

### 1. 排序后扫描模板

```cpp
sort(a.begin(), a.end());
for (int i = 0; i < a.size(); ++i) {
	// 按贪心规则选择
}
```

### 2. 区间选择模板

```cpp
sort(intervals.begin(), intervals.end(), cmp);
int ans = 0;
int end = -1e9;
for (auto& seg : intervals) {
	if (seg[0] >= end) {
		++ans;
		end = seg[1];
	}
}
```

### 3. 资源匹配模板

```cpp
sort(a.begin(), a.end());
sort(b.begin(), b.end());
int i = 0, j = 0;
while (i < a.size() && j < b.size()) {
	if (b[j] >= a[i]) {
		++i;
		++j;
	} else {
		++j;
	}
}
```

### 4. 当前阶段怎么理解

- 很多贪心实现都不复杂
- 关键是你前面那个“规则”到底对不对

## 十二、现代 C++ 的基础习惯

### 1. 先问自己“为什么现在选这个”

- 如果说不清原因
- 那大概率还没真的想到贪心本质

### 2. 排序前先想排序依据

- 是按左端点
- 按右端点
- 按权值
- 还是按差值
- 排序依据本身往往就是策略核心

### 3. 看到这些关键词先想到贪心

- 最多 / 最少
- 尽可能多地完成
- 当前最优选择
- 区间安排
- 分配资源

### 4. 以后会继续补

- 更严格的贪心证明
- 区间覆盖类题目
- Huffman / 最小代价类题
- 贪心和堆结合的题型
- 但当前阶段先把基本直觉和常见模型学稳

---

## 十三、高频易错点

1. 不是所有最优化问题都能贪心
2. 贪心代码短，不代表题目简单
3. 排序依据选错，整个策略就会错
4. 局部最优不一定能推出全局最优
5. 区间题常要特别注意按左端点还是右端点排序
6. 贪心常和双指针、排序、堆一起出现
7. 会写代码不等于策略正确
8. 先想“为什么不后悔”，再写贪心

---

## 十四、最简模板

### 排序 + 贪心模板

```cpp
sort(a.begin(), a.end());
for (int i = 0; i < a.size(); ++i) {
	// 如果当前选择更优，就选它
}
```

### 区间调度模板


### 最远可达模板

```cpp
int farthest = 0;
for (int i = 0; i <= farthest; ++i) {
	farthest = max(farthest, i + nums[i]);
}
```

## ⚡ TL;DR（快速决策）

- 信号量的核心是：**通过计数器协调多个执行流对资源或顺序的访问**
- 它既可以用于互斥，也可以用于同步
- 若只抓重点，应优先记住两个操作：
    - P / wait：申请资源，计数减 1
    - V / signal：释放资源，计数加 1

## 🧩 Core Idea（核心本质）

- 与互斥锁只强调“独占”不同，信号量更强调“资源数量”与“执行次序控制”
- 一句话理解：
    - **信号量是带计数能力的同步工具。**

## 🏗️ 关键概念

1. 二值信号量

- 取值常为 0 或 1
- 可近似理解为特殊形式的互斥控制

1. 计数信号量

- 允许多个线程在资源数量范围内同时进入
- 适合表示有限资源池

1. 同步作用

- 通过等待和通知控制不同线程执行顺序

## 🔧 Usage Patterns（可复用理解框架）

1. 用“资源配额”理解信号量

- 计数值表示当前可用资源数

1. 用“先做完才能继续”理解同步信号量

- 某线程 signal 后，其他等待线程才能继续

## ⚠️ Pitfalls（高频错误）

- 把信号量和互斥锁完全等同
- 不理解计数值代表的实际资源含义
- wait / signal 配对关系写乱，导致逻辑错误或阻塞

## 🚀 Performance / Tips（理解深化）

- 信号量是经典同步原语，尤其适合生产者-消费者模型
- 它比互斥锁更灵活，但也更容易写错
- 学信号量时，要始终问：
    - 这个计数器到底在表示什么资源或顺序？

## 🧪 Common Scenarios（常见使用场景）

- 生产者-消费者模型
- 资源池访问控制
- 顺序执行协调
- 有限并发控制


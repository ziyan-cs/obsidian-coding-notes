## ⚡ TL;DR（快速决策）

- DNS 的核心是：**把人类易读的域名解析成机器可用的 IP 地址**
- 浏览网页时，通常要先做 DNS 解析，再发起后续连接
- 若只抓重点，应优先记住：
    - 域名
    - 解析
    - 缓存
    - 分层查询

## 🧩 Core Idea（核心本质）

- 人更擅长记忆域名，机器更依赖 IP 定位
- DNS 充当两者之间的映射系统
- 一句话理解：
    - **DNS 是互联网的命名解析系统。**

## 🏗️ 关键概念

1. 域名

- 面向人类的可读名称

1. 解析结果

- 常见为 A / AAAA 记录，也可以是 CNAME、MX 等

1. 递归与迭代查询

- 本地 DNS 与权威服务器之间分层协作完成解析

1. 缓存

- 减少重复查询，提高访问效率

## 🔧 Usage Patterns（可复用理解框架）

1. 用“名字 -> 地址”理解 DNS 基本作用
2. 用“层层询问”理解解析过程
3. 用缓存理解为什么同一域名第二次访问更快

## ⚠️ Pitfalls（高频错误）

- 把 DNS 理解成单台服务器功能
- 以为域名和 IP 是永久一一固定绑定
- 忽略缓存与 TTL 对结果的影响

## 🚀 Performance / Tips（理解深化）

- DNS 是几乎所有互联网访问的前置步骤
- 理解 DNS 后，更容易理解 CDN、负载均衡、域名切换和解析异常排查

## 🧪 Common Scenarios（常见使用场景）

- 浏览器访问网站前的解析
- 域名配置与网络故障排查
- 负载均衡与地理调度理解

## 🧾 Minimal Template（最小理解模板）

```
域名 -> DNS 解析 -> IP 地址 -> 建立后续网络连接
```

## 📌 One-liner Summary（一句话总结）

- **DNS 的核心，是把域名解析成可用于实际通信的网络地址。**

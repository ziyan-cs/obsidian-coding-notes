## ⚡ TL;DR（快速决策）

- TCP 与 UDP 的核心区别在于：**TCP强调可靠、有序、面向连接；UDP强调轻量、无连接、低额外开销**
- 若只抓重点，应优先记住：
    - TCP：可靠传输
    - UDP：快速直接
    - 选择协议取决于业务需求，而不是固定“谁更高级”

## 🧩 Core Idea（核心本质）

- 二者都属于传输层协议，但设计目标不同
- 一句话理解：
    - **TCP 用更复杂机制换可靠性，UDP 用更少机制换轻量性。**

## 🏗️ 核心对比

1. 连接方式

- TCP：面向连接
- UDP：无连接

1. 可靠性

- TCP：确认、重传、排序、流控、拥塞控制
- UDP：默认不保证可靠、有序或去重

1. 开销

- TCP：头部和状态管理更复杂
- UDP：更轻量

1. 典型场景

- TCP：网页、文件传输、数据库连接
- UDP：音视频、实时通信、DNS 等

## 🔧 Usage Patterns（可复用理解框架）

1. 用“能否接受丢包”选协议

- 能接受少量丢包、强调实时：更偏 UDP
- 不能接受丢失、强调完整性：更偏 TCP

1. 用“业务语义”选协议

- 即时性优先 vs 完整性优先

## ⚠️ Pitfalls（高频错误）

- 把 UDP 理解成“不可靠所以没用”
- 把 TCP 理解成“绝对不会出问题”
- 只记定义，不结合具体业务需求分析

## 🚀 Performance / Tips（理解深化）

- 选择协议本质是系统权衡
- 应用层也可以在 UDP 之上自行补充可靠性逻辑
- 理解 TCP/UDP 区别后，端口、Socket、握手、流控会更好理解

## 🧪 Common Scenarios（常见使用场景）

- HTTP / HTTPS 依赖 TCP
- DNS 常用 UDP
- 实时音视频常用 UDP
- 游戏网络协议设计

## 📌 One-liner Summary（一句话总结）

- **TCP 与 UDP 的核心区别，是一个以可靠性为中心，一个以轻量和实时性为中心。**

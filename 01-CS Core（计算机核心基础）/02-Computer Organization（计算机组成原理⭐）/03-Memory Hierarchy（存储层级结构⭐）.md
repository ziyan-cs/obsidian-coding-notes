## ⚡ TL;DR（快速决策）

- 存储层级结构的核心是：**用多层速度、容量、成本不同的存储共同构成整体系统**
- 典型层次：
    - 寄存器
    - Cache
    - 主存
    - 外存
- 越靠近 CPU：通常越快、越小、越贵
- 越远离 CPU：通常越慢、越大、越便宜

## 🧩 Core Idea（核心本质）

- 单一存储设备无法同时满足高速度、大容量、低成本
- 因此系统采用层级化设计，通过局部性原理获得接近“既快又大”的效果
- 一句话理解：
    - **存储层级结构是工程权衡下的分层方案。**

## 🏗️ 常见层次

1. 寄存器

- CPU 内部，最快，容量最小

1. Cache

- 作为 CPU 与内存之间的高速缓冲
- 常见 L1 / L2 / L3 分层

1. 主存（RAM）

- 运行中主要工作区
- 容量较大，但慢于 Cache

1. 外存（SSD / HDD）

- 长期存储程序和文件
- 容量大，但远慢于主存

## 🔧 Usage Patterns（可复用理解框架）

1. 用“快小贵 / 慢大便宜”理解层次
2. 用“局部性”理解为什么 Cache 有效

- 时间局部性：刚访问过的还可能再访问
- 空间局部性：相邻数据也更可能被访问

1. 用“瓶颈迁移”理解性能

- 当 CPU 越来越快，访存延迟就越来越成为瓶颈

## ⚠️ Pitfalls（高频错误）

- 只背层次名称，不理解它们为何必须分层存在
- 把外存和内存混为一谈
- 忽略局部性原理，导致无法理解 Cache 价值

## 🚀 Performance / Tips（理解深化）

- 存储层级结构是理解程序性能的关键基础
- 很多程序慢，不是算不动，而是取数太慢
- 后续 Cache、虚拟内存、文件系统都和这套分层思想密切相关

## 🧪 Common Scenarios（常见使用场景）

- 理解程序为何需要加载到内存
- 理解缓存命中率为何影响性能
- 理解数据访问模式为何重要

## 🧾 Minimal Template（最小理解模板）

```
寄存器 -> Cache -> 主存 -> 外存
速度下降，容量上升，成本下降
```

## 📌 One-liner Summary（一句话总结）

- **存储层级结构的核心，是通过多层不同特性的存储共同平衡速度、容量与成本。**

## ⚡ TL;DR（快速决策）

- 程序从源码到运行，核心通常要经过：**编译 -> 链接 -> 加载**
- 编译：把高级语言翻译成目标代码
- 链接：把多个目标文件和库组合成可执行程序
- 加载：把可执行程序放入内存并准备运行
- 如果只抓主线，应优先记住：
    - 源码不能直接被 CPU 执行
    - 可执行程序是多个部分协同产物
    - 运行前必须先完成内存映射与环境准备

## 🧩 Core Idea（核心本质）

- 程序执行不是“写完代码立刻跑”，而是一个逐层转化过程
- 一句话理解：
    - **编译解决“看懂源码”，链接解决“拼装程序”，加载解决“把程序送进内存准备运行”。**
- 这是从程序文本到机器执行的关键通路

## 🏗️ 三个核心阶段

1. 编译（Compile）

- 把高级语言源码翻译为目标代码或汇编代码
- 典型过程包含：
    - 词法分析
    - 语法分析
    - 语义分析
    - 优化
    - 代码生成

1. 链接（Link）

- 把多个目标文件与库整合为一个可执行文件
- 解决的问题：
    - 外部符号引用
    - 函数和全局变量地址分配
    - 库代码合并

1. 加载（Load）

- 操作系统把可执行文件装入内存
- 建立代码段、数据段、堆、栈等运行环境
- 然后把控制权交给程序入口点

## 🔧 Usage Patterns（可复用理解框架）

1. 用“翻译—组装—放入内存”理解全过程

- 编译：翻译
- 链接：组装
- 加载：放入内存并初始化

1. 用报错类型倒推阶段

- 语法错误：多发生在编译阶段
- 未定义引用：常发生在链接阶段
- 程序启动失败：可能与加载或运行环境有关

1. 用 C/C++ 工程理解

- `.cpp` 不会直接变成运行程序
- 中间会经历目标文件、链接产物、操作系统加载等步骤

## ⚠️ Pitfalls（高频错误）

- 把编译、链接、运行混为一谈
- 误以为“能编过”就一定能链接通过
- 不理解库、目标文件、可执行文件之间的关系
- 只会背阶段名称，不知道每阶段解决的具体问题

## 🚀 Performance / Tips（理解深化）

- 真正理解这一链路后，很多工程问题会更容易定位
- 例如：
    - 为什么头文件重复定义会报错
    - 为什么静态库和动态库行为不同
    - 为什么程序运行前还要由操作系统介入
- 这是连接编程语言、操作系统和体系结构的重要桥梁

## 🧪 Common Scenarios（常见使用场景）

- 理解 C/C++ 构建流程
- 分析编译错误与链接错误
- 理解可执行文件如何运行
- 为操作系统和编译原理打基础

## 🧾 Minimal Template（最小理解模板）

```
源码 -> 编译 -> 目标文件 -> 链接 -> 可执行文件 -> 加载到内存 -> 执行
```

## 📌 One-liner Summary（一句话总结）

- **编译、链接、加载的核心，是把程序从源码逐步转化为可被操作系统装入内存并执行的机器程序。**

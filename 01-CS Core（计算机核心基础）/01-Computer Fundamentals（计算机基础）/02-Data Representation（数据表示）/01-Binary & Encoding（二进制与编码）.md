## ⚡ TL;DR（快速决策）

- 二进制与编码的本质是：**把信息转换成计算机可以存储和处理的符号形式**
- 计算机底层只稳定处理两种状态，因此普遍采用二进制表示
- 编码解决的问题是：
    - 数字怎么表示
    - 字符怎么表示
    - 图片、音频、文本怎么表示
- 不同编码方式本质上是在回答：**“某种信息如何映射到比特串？”**

## 🧩 Core Idea（核心本质）

- bit（比特）是最小信息单位，只能取 0 或 1
- byte（字节）通常由 8 个 bit 组成
- 二进制是表示基础，编码是映射规则
- 一句话理解：
    - **数据表示回答“用什么比特串表示信息”，编码回答“按什么规则解释这些比特串”。**

## 🏗️ 关键概念

1. 二进制

- 十进制靠 10 个符号，二进制靠 2 个符号：0 和 1
- 权值按 2 的幂展开
- 例如：
    - $1011_2 = 1\cdot 2^3 + 0\cdot 2^2 + 1\cdot 2^1 + 1\cdot 2^0 = 11_{10}$

1. 编码

- 编码是“信息到二进制”的映射规则
- 例如：
    - ASCII：字符编码
    - UTF-8：通用文本编码
    - JPEG：图像压缩编码
    - MP3：音频编码

1. 同样的比特串，不同解释可能代表不同含义

- `01000001` 可以被解释为：
    - 无符号整数 65
    - ASCII 字符 `A`
- 所以数据一定要结合类型和编码语境理解

## 🔧 Usage Patterns（可复用理解框架）

1. 用“表示 + 解释”理解数据

- 表示：底层存了哪些 bit
- 解释：这些 bit 按什么规则读取

1. 用文本编码举例

- 文本在计算机中不是“字”本身，而是字符编码后的字节序列
- 乱码问题本质上往往就是“编码与解码规则不一致”

1. 用媒体文件举例

- 图片、音频、视频都不是“原始现实”，而是编码后的数字表示

## ⚠️ Pitfalls（高频错误）

- 把“二进制”与“编码”混为一谈
- 误以为数据本身天然带意义，实际上意义来自解释规则
- 忽略字符编码差异导致乱码问题
- 只记公式，不理解“同样比特串可以有不同语义”

## 🚀 Performance / Tips（理解深化）

- 学数据表示时，核心不是背编码表，而是理解“映射规则”
- 后续的整数表示、浮点数、存储、网络传输，本质都建立在编码思想之上
- 真正理解编码，才能理解为什么格式、协议、兼容性如此重要

## 🧪 Common Scenarios（常见使用场景）

- 文本乱码分析
- 理解字符编码
- 理解文件格式本质
- 为整数、浮点数、图像音频表示打基础

## 🧾 Minimal Template（最小理解模板）

```
现实信息 -> 编码规则 -> 比特串 -> 存储/传输 -> 按规则解码
```

## 📌 One-liner Summary（一句话总结）

- **二进制与编码的核心，是把各种信息映射成可由计算机处理的比特序列，并按约定规则解释这些比特。**

## ⚡ TL;DR（快速决策）

- 位运算本质是：**直接对二进制位进行逻辑操作**
- 常见操作：
    - 与 `&`
    - 或 `|`
    - 异或 `^`
    - 取反 `~`
    - 左移 `<<`
    - 右移 `>>`
- 如果只抓重点，优先记住：
    - 位运算操作的是 bit，而不是十进制表面形式
    - 它与补码、整数表示密切相关
    - 它常用于状态压缩、掩码判断、快速运算

## 🧩 Core Idea（核心本质）

- 每个整数在机器里本来就是 bit 序列
- 位运算就是直接在这一层操作
- 一句话理解：
    - **位运算是对整数底层二进制表示的直接操作。**
- 它通常比算术层面更底层，也更接近硬件逻辑

## 🏗️ 关键概念

1. 按位与 `&`

- 两位都为 1，结果才为 1
- 常用于掩码判断

1. 按位或 `|`

- 只要一位为 1，结果就为 1
- 常用于设置某些位

1. 按位异或 `^`

- 相同为 0，不同为 1
- 常用于翻转位、消除相同值

1. 取反 `~`

- 0 变 1，1 变 0
- 要结合位宽和补码理解

1. 移位

- `x << k`：通常表示左移 k 位
- `x >> k`：通常表示右移 k 位
- 常与乘 2 / 除 2 的思想关联，但不能机械套用所有场景

## 🔧 Usage Patterns（可复用代码模板）

1. 判断某位是否为 1

```cpp
if (x & (1 << k)) {
    // 第 k 位为 1
}
```

1. 设置某位为 1

```cpp
x |= (1 << k);
```

1. 清除某位

```cpp
x &= ~(1 << k);
```

1. 翻转某位

```cpp
x ^= (1 << k);
```

## ⚠️ Pitfalls（高频错误）

- 把位运算和逻辑运算 `&&`、`||` 混淆
- 不理解补码就分析负数位运算
- 对移位规则理解不完整，特别是有符号右移
- 忽略运算优先级，导致表达式错误

## 🚀 Performance / Tips（理解深化）

- 位运算的重要价值不只是“快”，更重要是“表达某些结构非常自然”
- 高频应用：
    - 状态压缩
    - 子集枚举
    - 权限标记
    - 位掩码技巧
- 先把“每一位代表什么”想清楚，再写位运算

## 🧪 Common Scenarios（常见使用场景）

- 权限系统表示

#数据结构 #数组
## 一页速览

- **数组**：一组相同类型数据连续存放
- **下标**：从 `0` 开始，不是从 `1` 开始
- **核心优势**：随机访问快
- **核心限制**：长度固定
- **常见操作**：遍历、求和、找最大值、查找、排序前处理
- **最常错**：越界、下标起点写错、循环条件写错、把数组名当普通变量用

## 一、先总后分：这一节到底学什么

### 1. 数组解决什么问题

- 当你要存很多“同类型数据”时，不能总写一堆单独变量
- 数组可以把这些数据按顺序放在一起
- 很适合处理批量数据

### 2. 数组最核心看什么

- 怎么定义
- 怎么访问
- 怎么遍历
- 怎么避免越界

### 3. 当前阶段先抓主线

- 会定义一维数组
- 会用下标访问元素
- 会写最基本的遍历
- 会做简单题目里的常见操作

---

## 二、数组是什么

### 1. 基本理解

- 数组是**相同类型元素**的集合
- 这些元素在内存中通常是**连续存放**的

```cpp
int arr[5] = {10, 20, 30, 40, 50};
```

### 2. 怎么理解这个定义

- `int`：数组里每个元素的类型
- `arr`：数组名
- `5`：数组长度

### 3. 当前阶段先记一句话

- **数组 = 连续存放的一组同类型数据**

---

## 三、数组的定义与初始化

### 1. 先定义后赋值

```cpp
int arr[5];
arr[0] = 10;
arr[1] = 20;
```

### 2. 定义时初始化

```cpp
int arr[5] = {1, 2, 3, 4, 5};
```

### 3. 省略长度

```cpp
int arr[] = {1, 2, 3, 4, 5};
```

- 编译器会自动推断长度

### 4. 部分初始化

```cpp
int arr[5] = {1, 2};
```

- 前两个元素是 `1`、`2`
- 其余元素默认补 `0`

---

## 四、数组下标与访问

### 1. 下标从 0 开始

```cpp
int arr[5] = {10, 20, 30, 40, 50};
cout << arr[0] << '\\n'; // 10
cout << arr[2] << '\\n'; // 30
```

### 2. 最后一个元素的位置

- 长度是 `n` 的数组
- 最后一个下标是 `n - 1`

### 3. 最常错的地方

- 不是从 `1` 开始
- 不能访问 `arr[5]` 这种越界位置

### 速记

- 长度 `5` 的数组，下标范围是 `0 ~ 4`

---

## 五、数组的遍历

### 1. 最基础写法

```cpp
int arr[5] = {1, 2, 3, 4, 5};
for (int i = 0; i < 5; ++i) {
	cout << arr[i] << '\\n';
}
```

### 2. 为什么循环条件常写 `i < n`

- 因为最后合法位置是 `n - 1`
- 写成 `i <= n` 很容易越界

### 3. 范围 `for` 先有印象

```cpp
for (int x : arr) {
	cout << x << '\\n';
}
```

- 写法更简洁
- 但做题时很多场景还是常用下标循环

---

## 六、数组最常见操作

### 1. 求和

```cpp
int sum = 0;
for (int i = 0; i < 5; ++i) {
	sum += arr[i];
}
```

### 2. 找最大值

```cpp
int maxv = arr[0];
for (int i = 1; i < 5; ++i) {
	if (arr[i] > maxv) maxv = arr[i];
}
```

### 3. 查找某个值

```cpp
int target = 3;
for (int i = 0; i < 5; ++i) {
	if (arr[i] == target) {
		cout << i << '\\n';
		break;
	}
}
```

### 4. 修改元素

```cpp
arr[2] = 100;
```

- 直接通过下标改值

---

## 七、数组名怎么理解

### 1. 数组名的直观理解

- 数组名代表这组数据的起始位置
- 它和普通变量名不完全一样

### 2. 当前阶段先记结论

- 数组名可以用于访问整组数据
- 但数组名**不是普通变量**，不能随便整体赋值

```cpp
int a[3] = {1, 2, 3};
int b[3] = {4, 5, 6};
// a = b; // ❌ 数组不能这样整体赋值
```

### 3. 做题阶段怎么处理

- 大多数时候先把数组当成“按下标访问的一组数据”来理解
- 深层原理和指针联系，后面再补

## 八、数组与函数

### 1. 数组传参先有印象

```cpp
void printArr(int arr[], int n) {
	for (int i = 0; i < n; ++i) {
		cout << arr[i] << '\\n';
	}
}
```

### 2. 为什么还要传长度 `n`

- 因为函数里通常不能直接可靠知道数组有多长
- 所以常把长度一起传进去

### 3. 当前阶段先记

- **数组传参时，长度经常要单独传**

---

## 九、二维数组先有印象

### 1. 什么是二维数组

- 可以理解成“表格”或“矩阵”

```cpp
int a[2][3] = {
	{1, 2, 3},
	{4, 5, 6}
};
```

### 2. 访问方式

```cpp
cout << a[1][2] << '\\n'; // 6
```

### 3. 当前阶段结论

- 先知道二维数组也是“下标访问”
- 一维数组先学稳，二维数组后面再展开

---

## 十、当前阶段最常见写法

### 1. 定义并遍历

```cpp
int arr[5] = {1, 2, 3, 4, 5};
for (int i = 0; i < 5; ++i) {
	cout << arr[i] << '\\n';
}
```

### 2. 输入数组

```cpp
int arr[5];
for (int i = 0; i < 5; ++i) {
	cin >> arr[i];
}
```

### 3. 输出数组

```cpp
for (int i = 0; i < 5; ++i) {
	cout << arr[i] << ' ';
}
```

## 十一、现代 C++ 的基础习惯

### 1. 下标和长度要配套思考

- 写循环时先想清楚范围
- `0 ~ n - 1` 是最核心的意识

### 2. 小心越界

- 越界很多时候不会立刻报错
- 但结果可能完全错，甚至引发崩溃

### 3. 以后会接触更常用的替代品

- `vector` 更灵活
- `array` 更现代
- 但数组基础必须先会

### 4. 当前阶段怎么学最有效

- 先把一维数组的定义、访问、遍历练熟
- 再做求和、最值、查找、逆序这类基础题

---

## 十二、高频易错点

1. 数组下标从 `0` 开始，不是从 `1` 开始
2. 长度为 `n` 的数组，最后一个下标是 `n - 1`
3. 循环条件常写 `i < n`，不要乱写 `i <= n`
4. 不要访问越界位置
5. 数组名不是普通变量，不能整体赋值
6. 数组传参时常常要额外传长度
7. 求最值时初值通常先设成第一个元素
8. 输入输出数组时最常配合 `for` 循环

---

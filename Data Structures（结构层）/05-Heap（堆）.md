#数据结构 #堆 #排序
## 一页速览

- **堆**：一种特殊的完全二叉树
- **核心特点**：父节点和子节点满足固定大小关系
- **常见类型**：大根堆、小根堆
- **核心优势**：快速取最大值 / 最小值
- **常见操作**：建堆、插入、删除堆顶、取堆顶
- **高频场景**：优先队列、Top K、堆排序、动态维护最值
- **最常错**：堆和内存“堆区”混淆、下标计算写错、调整方向写反、大根堆小根堆搞混

### 1. 堆解决什么问题

- 有些题目不需要把所有数据完全排好序
- 只需要快速拿到当前最大值或最小值
- 这时堆往往比普通排序更合适

### 2. 堆最核心看什么

- 大根堆和小根堆怎么区分
- 堆顶为什么最重要
- 插入和删除堆顶怎么调整

### 3. 当前阶段先抓主线

- 会理解堆的定义
- 会分清大根堆和小根堆
- 会写最基础的堆顶操作
- 会知道什么时候该想到堆

## 二、堆是什么

### 1. 基本理解

- 堆是一种**完全二叉树**
- 它不是普通随便摆放的树
- 它要求父节点和子节点满足固定关系

### 2. 两种最常见的堆

- **大根堆**：父节点值 >= 子节点值
- **小根堆**：父节点值 <= 子节点值

### 3. 当前阶段先记一句话

- **堆 = 能快速拿堆顶最值的完全二叉树**

---

## 三、完全二叉树先有印象

### 1. 什么叫完全二叉树

- 除最后一层外，其余层都满
- 最后一层从左到右连续排列

### 2. 为什么堆常用数组存

- 因为完全二叉树很适合按顺序放进数组
- 不一定非要真的写成链式树结构

### 3. 用数组下标表示节点关系

如果下标从 `0` 开始：

- 左孩子：`2 * i + 1`
- 右孩子：`2 * i + 2`
- 父节点：`(i - 1) / 2`

### 4. 当前阶段先记

- **堆经常“看起来是树”，实现时却常用数组**

---

## 四、大根堆和小根堆

### 1. 大根堆

- 堆顶最大
- 常用于快速拿“当前最大值”

```cpp
50
/ \\
30 40
```

### 2. 小根堆

- 堆顶最小
- 常用于快速拿“当前最小值”

```cpp
10
/ \\
20 30
```

### 3. 当前阶段怎么理解

- 你不用保证整棵树完全有序
- 只要保证每个父节点和子节点满足堆规则

### 4. 最容易混的点

- **堆有序 ≠ 整体完全排序**
- 堆只能保证“局部父子关系正确”

---

## 五、堆顶为什么重要

### 1. 堆顶的意义

- 大根堆的堆顶就是最大值
- 小根堆的堆顶就是最小值

### 2. 为什么很多题喜欢用堆

- 因为取堆顶很快
- 比每次都重新遍历找最值高效得多

### 3. 当前阶段先记

- **学堆，本质上就是学怎么维护堆顶最值**

---

## 六、堆的核心操作

### 1. 插入元素

- 先把新元素放到最后
- 再向上调整，直到满足堆性质

### 2. 删除堆顶

- 先把最后一个元素放到堆顶
- 再向下调整，直到满足堆性质

### 3. 取堆顶

- 直接看堆顶元素
- 不删除它

### 4. 建堆

- 把一组数据调整成合法的堆

### 5. 当前阶段先记

- 堆最常见就是：**建堆、取堆顶、插入、删除堆顶**

---

## 七、向上调整和向下调整

### 1. 向上调整

- 常用于插入新元素后
- 新元素和父节点比较
- 不满足堆规则就交换

### 2. 向下调整

- 常用于删除堆顶后
- 堆顶元素和孩子比较
- 不满足堆规则就往下交换

### 3. 当前阶段怎么理解

- 插入后重点看“往上”
- 删除堆顶后重点看“往下”

---

## 八、C++ 里的堆：优先队列

### 1. 最常用写法

```cpp
#include <queue>
priority_queue<int> pq;
```

- 默认是**大根堆**

### 2. 基本操作示例

```cpp
pq.push(10);
pq.push(30);
pq.push(20);

cout << pq.top() << '\\n'; // 30
pq.pop();
cout << pq.top() << '\\n'; // 20
```

### 3. 小根堆写法先记

```cpp
priority_queue<int, vector<int>, greater<int>> pq;
```

- 这样堆顶就是最小值

### 4. 当前阶段怎么理解

- 做题时大多数直接用 `priority_queue`
- 底层思想还是堆

---

## 九、堆和排序的关系

### 1. 堆排序先有印象

- 先把数组建成大根堆
- 每次把堆顶最大值放到末尾
- 再缩小范围继续调整

### 2. 当前阶段怎么理解

- 堆排序也是一种排序方法
- 但“堆”本身不等于“排序完成”

### 3. 当前阶段先记结论

- **堆擅长维护最值**
- **排序是堆的一种应用，不是堆的全部**

---

## 十、堆最常见应用

### 1. Top K 问题

- 找前 K 大或前 K 小
- 堆特别常见

### 2. 动态最值维护

- 数据不断加入时
- 想随时知道当前最大 / 最小值

### 3. 优先级处理

- 谁优先级高，谁先处理
- 典型就是优先队列思想

### 4. 合并 / 调度类问题

- 某些“每次取当前最小 / 最大”的题目，常想到堆

---

```cpp

```

### 2. 小根堆模板


### 3. 入堆出堆

## 十二、现代 C++ 的基础习惯

### 1. 做题优先会用 `priority_queue`

- 先把应用写熟
- 再去补手写堆调整细节

### 2. 看到这些关键词先想到堆

- Top K
- 当前最大 / 最小
- 优先级
- 每次取最值

### 3. 不要把堆和“堆区内存”混淆

- 数据结构里的堆是 **heap**
- 内存管理里的堆区也是 **heap**
- 但这两个语境完全不是一回事

### 4. 以后会继续补

- 手写建堆
- 手写下沉 / 上浮
- 堆排序
- 但当前阶段先把应用场景学稳

## 十三、高频易错点

1. 堆是完全二叉树，不是普通二叉树
2. 大根堆堆顶最大，小根堆堆顶最小
3. 堆有序不等于整体完全有序
4. 插入后常是向上调整，删除堆顶后常是向下调整
5. `priority_queue` 默认是大根堆
6. 小根堆写法容易忘
7. 数组下标算父子节点关系时容易写错
8. 不要把“数据结构里的堆”和“内存堆区”混为一谈

---

## 十四、最简模板

### 大根堆模板

```cpp
#include <queue>
priority_queue<int> pq;
```

### 小根堆模板

```cpp
#include <queue>
#include <vector>
#include <functional>
priority_queue<int, vector<int>, greater<int>> pq;
```

### 入堆出堆模板

```cpp
pq.push(10);
pq.push(30);
pq.pop();
```

### 访问堆顶模板

```cpp
if (!pq.empty()) {
	cout << pq.top() << '\\n';
}
```

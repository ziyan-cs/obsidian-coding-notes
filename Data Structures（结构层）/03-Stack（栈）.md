#数据结构 #栈
## 一页速览

- **栈**：一种先进后出（LIFO）的线性结构
- **入口位置**：只能在栈顶操作
- **核心操作**：入栈、出栈、取栈顶、判空
- **核心优势**：栈顶操作快
- **核心限制**：不能随机访问中间元素
- **常见场景**：括号匹配、表达式处理、函数调用、单调栈基础
- **最常错**：把先进后出写反、出栈前不判空、把队列和栈混淆

## 一、先总后分：这一节到底学什么

### 1. 栈解决什么问题

- 有些数据只关心“最后放进去的那个”
- 不需要中间随机访问
- 这时栈会比普通数组思路更自然

### 2. 栈最核心看什么

- 什么叫先进后出
- 栈顶怎么变化
- 入栈和出栈怎么写
- 什么题目该想到栈

### 3. 当前阶段先抓主线

- 会理解栈的规则
- 会写最基本的入栈出栈
- 会用栈解决简单问题
- 会分清栈和队列

---

## 二、栈是什么

### 1. 基本理解

- 栈是一种线性数据结构
- 特点是：**后进去的元素先出来**

### 2. 怎么理解这个规则

- 像一摞书
- 只能从最上面放，也只能从最上面拿

### 3. 当前阶段先记一句话

- **栈 = 先进后出，只能操作栈顶**

---

## 三、栈的核心操作

### 1. 入栈 `push`

- 把元素放到栈顶

### 2. 出栈 `pop`

- 把栈顶元素移除

### 3. 取栈顶 `top`

- 查看栈顶元素是谁
- 但不删除它

### 4. 判空 `empty`

- 判断栈里有没有元素

### 5. 当前阶段先记

- 栈最常见就是这 4 个操作：`push`、`pop`、`top`、`empty`

---

## 四、用数组角度理解栈

### 1. 最基础思路

- 可以把栈理解成“受限制的数组”
- 只能在末尾这一端操作

### 2. 栈顶怎么理解

- 栈顶就是最后加入的元素位置
- 新元素总是加到栈顶
- 删除元素也总是删栈顶

### 3. 一个简单过程

```cpp
push(1)
push(2)
push(3)
```

- 现在栈顶是 `3`

```cpp
pop()
```

- `3` 被弹出
- 现在栈顶变成 `2`

---

## 五、C++ 里的栈

### 1. 最常用写法

```cpp
#include <stack>
stack<int> st;
```

### 2. 基本操作示例

```cpp
st.push(10);
st.push(20);
st.push(30);

cout << st.top() << '\\n'; // 30
st.pop();
cout << st.top() << '\\n'; // 20
```

### 3. 判空

```cpp
if (!st.empty()) {
	cout << st.top() << '\\n';
}
```

### 4. 当前阶段怎么理解

- `stack<int>` 表示“存整数的栈”
- 平时做题时，大多数直接用 STL 的 `stack` 就够了

---

## 六、栈和队列的区别

|对比项|栈|队列|
|---|---|---|
|规则|先进后出|先进先出|
|操作位置|只看栈顶|队头出，队尾进|
|典型场景|括号匹配、回退|排队、层序遍历|

### 当前阶段先记结论

- **后进先出**想栈
- **先进先出**想队列

---

## 七、栈最常见应用

### 1. 括号匹配

- 左括号入栈
- 遇到右括号，就和栈顶左括号匹配
- 最后栈空且过程中没出错，说明匹配成功

### 2. 表达式处理

- 中缀、后缀表达式转换常用栈
- 当前阶段先知道“表达式题常和栈有关”即可

### 3. 函数调用

- 函数调用顺序本身就和栈有关
- 后调用的函数通常先返回

### 4. 单调栈先有印象

- 用来处理“下一个更大元素”这类题
- 后面刷题时会经常见到

---

## 八、括号匹配先有印象

### 1. 基本思路

```cpp
string s = "()[]{}";
stack<char> st;
```

### 2. 怎么做

- 遇到左括号：入栈
- 遇到右括号：
    - 先看栈是否为空
    - 再看栈顶是否匹配

### 3. 为什么这题适合用栈

- 因为“最后出现的左括号”要最先匹配
- 这正是先进后出的特点

---

## 九、栈的遍历怎么理解

### 1. STL `stack` 不支持像数组那样直接遍历

- 它只暴露栈顶操作
- 不能随机访问中间元素

### 2. 如果想依次输出

- 常见做法是边 `top()` 边 `pop()`

```cpp
while (!st.empty()) {
	cout << st.top() << '\\n';
	st.pop();
}
```

### 3. 当前阶段先记

- **栈不是拿来随机访问的，重点是栈顶操作**

---


## 十一、现代 C++ 的基础习惯

### 1. 出栈前先判空

- `top()` 和 `pop()` 前都要先考虑栈是不是空的
- 空栈操作是高频错误

### 2. 不要把栈当数组用

- 栈的价值不在“存数据”
- 而在“只操作最后进入的元素”

### 3. 看到这些关键词先想到栈

- 括号匹配
- 撤销 / 回退
- 后进先出
- 最近一个未处理元素

### 4. 以后会接触

- 单调栈
- 用栈实现队列
- 用队列实现栈
- 但基础栈操作必须先熟

---

## 十二、高频易错点

1. 栈是先进后出，不是先进先出
2. `top()` 只看栈顶，不删除元素
3. `pop()` 删除栈顶，但不返回值
4. 空栈时不能直接 `top()` 或 `pop()`
5. 栈不能随机访问中间元素
6. 括号匹配类题目很常用栈
7. 栈和队列的规则不要混
8. 输出栈内容时常会一边访问一边弹出

---

## 十三、最简模板

### 栈定义模板

```cpp
#include <stack>
stack<int> st;
```

### 入栈出栈模板

```cpp
st.push(10);
st.push(20);
st.pop();
```

### 访问栈顶模板

```cpp
if (!st.empty()) {
	cout << st.top() << '\\n';
}
```

### 清空式遍历模板

```cpp
while (!st.empty()) {
	cout << st.top() << '\\n';
	st.pop();
}
```

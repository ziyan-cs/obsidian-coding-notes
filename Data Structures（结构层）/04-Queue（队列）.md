#数据结构 #队列
## 一页速览

- **队列**：一种先进先出（FIFO）的线性结构
- **入口位置**：队尾进入，队头离开
- **核心操作**：入队、出队、取队头、判空
- **核心优势**：适合按进入顺序处理数据
- **核心限制**：不能像数组那样随机访问中间元素
- **常见场景**：排队处理、层序遍历、BFS、任务调度
- **最常错**：把先进先出写反、队头队尾混淆、出队前不判空、把栈和队列混淆

## 一、先总后分：这一节到底学什么

### 1. 队列解决什么问题

- 有些数据需要按“先来先处理”的顺序操作
- 不关心最后加入的元素优先
- 这时队列比栈和普通数组思路更自然

### 2. 队列最核心看什么

- 什么叫先进先出
- 队头和队尾怎么变化
- 入队和出队怎么写
- 什么题目该想到队列

### 3. 当前阶段先抓主线

- 会理解队列的规则
- 会写最基本的入队出队
- 会用队列解决简单问题
- 会分清队列和栈

## 二、队列是什么

### 1. 基本理解

- 队列是一种线性数据结构
- 特点是：**先进去的元素先出来**

### 2. 怎么理解这个规则

- 像排队买东西
- 先到的人先处理，后来的人排在后面

### 3. 当前阶段先记一句话

- **队列 = 先进先出，队尾进，队头出**

---

## 三、队列的核心操作

### 1. 入队 `push`

- 把元素放到队尾

### 2. 出队 `pop`

- 把队头元素移除

### 3. 取队头 `front`

- 查看最前面的元素是谁
- 但不删除它

### 4. 判空 `empty`

- 判断队列里有没有元素

### 5. 当前阶段先记

- 队列最常见就是这 4 个操作：`push`、`pop`、`front`、`empty`

---

## 四、用数组角度理解队列

### 1. 最基础思路

- 可以把队列理解成“受限制的数组”
- 只能从一端进入，从另一端离开

### 2. 队头和队尾怎么理解

- 新元素总是加到队尾
- 旧元素总是从队头离开
- 中间元素不能随便访问和删除

### 3. 一个简单过程

```cpp
push(1)
push(2)
push(3)
```

- 现在队头是 `1`
- 队尾是 `3`

```cpp
pop()
```

- `1` 被移除
- 现在队头变成 `2`

---

## 五、C++ 里的队列

### 1. 最常用写法

```cpp
#include <queue>
queue<int> q;
```

### 2. 基本操作示例

```cpp
q.push(10);
q.push(20);
q.push(30);

cout << q.front() << '\\n'; // 10
q.pop();
cout << q.front() << '\\n'; // 20
```

### 3. 判空

```cpp
if (!q.empty()) {
	cout << q.front() << '\\n';
}
```

### 4. 当前阶段怎么理解

- `queue<int>` 表示“存整数的队列”
- 平时做题时，大多数直接用 STL 的 `queue` 就够了

---

## 六、队列和栈的区别

|对比项|队列|栈|
|---|---|---|
|规则|先进先出|先进后出|
|操作位置|队头出，队尾进|只看栈顶|
|典型场景|排队、BFS、层序遍历|括号匹配、回退|

### 当前阶段先记结论

- **先进先出**想队列
- **后进先出**想栈

---

## 七、队列最常见应用

### 1. 排队处理

- 谁先进入，谁先处理
- 很符合现实中的排队逻辑

### 2. 二叉树层序遍历

- 一层一层访问节点时常用队列
- 因为进入顺序就是访问顺序

### 3. BFS 广度优先搜索

- 图和网格的广搜通常用队列
- 先扩展当前层，再扩展下一层

### 4. 任务调度先有印象

- 某些任务按提交顺序执行
- 背后常有队列思想

---

## 八、层序遍历先有印象

### 1. 基本思路

```cpp
queue<TreeNode*> q;
q.push(root);
```

### 2. 怎么做

- 先把根节点入队
- 每次取出队头节点
- 再把它的左右孩子加入队尾

### 3. 为什么这题适合用队列

- 因为先进入队列的节点，要先被处理
- 这正是先进先出的特点

---

## 九、队列的遍历怎么理解

### 1. STL `queue` 不支持像数组那样直接遍历

- 它只暴露队头 / 队尾相关操作
- 不能随机访问中间元素

### 2. 如果想依次输出

- 常见做法是边 `front()` 边 `pop()`

```cpp
while (!q.empty()) {
	cout << q.front() << '\\n';
	q.pop();
}
```

### 3. 当前阶段先记

- **队列不是拿来随机访问的，重点是队头队尾操作**

### 2. 入队出队

## 十一、现代 C++ 的基础习惯

### 1. 出队前先判空

- `front()` 和 `pop()` 前都要先考虑队列是不是空的
- 空队列操作是高频错误

### 2. 不要把队列当数组用

- 队列的价值不在“存数据”
- 而在“按进入顺序处理元素”

### 3. 看到这些关键词先想到队列

- 排队处理
- 先进先出
- 层序遍历
- BFS

### 4. 以后会接触

- 双端队列 `deque`
- 优先队列 `priority_queue`
- 单调队列
- 但基础队列操作必须先熟

---

## 十二、高频易错点

1. 队列是先进先出，不是先进后出
2. `front()` 只看队头，不删除元素
3. `pop()` 删除队头，但不返回值
4. 空队列时不能直接 `front()` 或 `pop()`
5. 队列不能随机访问中间元素
6. 层序遍历和 BFS 很常用队列
7. 队头和队尾不要搞反
8. 输出队列内容时常会一边访问一边弹出

---

### 队列定义模板


### 入队出队模板

```cpp

```

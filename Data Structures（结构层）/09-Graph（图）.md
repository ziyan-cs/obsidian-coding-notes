#数据结构 #图
## 一页速览

- **图**：由顶点和边组成的数据结构
- **核心特点**：可以表示“点和点之间的关系”
- **常见类型**：有向图、无向图、带权图
- **核心遍历**：DFS 深度优先搜索、BFS 广度优先搜索
- **常见表示法**：邻接矩阵、邻接表
- **高频场景**：路径搜索、连通性判断、最短路、拓扑排序基础
- **最常错**：树和图混淆、忘记判重访问、邻接矩阵和邻接表分不清、有向无向搞混

## 一、先总后分：这一节到底学什么

### 1. 图解决什么问题

- 有些问题不是简单线性结构，也不是普通树结构
- 数据之间可能是“多对多连接”
- 这时就要用图来描述关系

### 2. 图最核心看什么

- 顶点和边分别表示什么
- 图有哪些常见类型
- 图怎么存储
- DFS 和 BFS 怎么遍历图

### 3. 当前阶段先抓主线

- 会理解图的基本概念
- 会分清有向图和无向图
- 会写 DFS 和 BFS 的基础遍历
- 会理解邻接表是最常见存图方式

## 二、图是什么

### 1. 基本理解

- 图由两部分组成：
    - 顶点（点）
    - 边（连接点和点的关系）

### 2. 怎么理解

- 顶点可以是人、城市、课程、网页
- 边表示它们之间的关系
- 例如“朋友关系”“道路连接”“课程依赖”

### 3. 当前阶段先记一句话

- **图 = 点和点之间关系的集合**

---

## 三、图和树的区别

### 1. 树是什么

- 树是一种特殊图
- 没有环
- 连通
- 层次关系很明确

### 2. 图更一般

- 图可以有环
- 图不一定连通
- 一个点可以和很多点相连

### 3. 当前阶段先记结论

- **树是特殊图**
- **图比树更自由，也更复杂**

---

## 四、图的常见类型

### 1. 无向图

- 边没有方向
- `A -- B` 表示 A 能到 B，B 也能到 A

### 2. 有向图

- 边有方向
- `A -> B` 不代表 `B -> A`

### 3. 带权图

- 边上还有权值
- 比如距离、花费、时间

### 4. 当前阶段怎么理解

- 无向图常表示双向关系
- 有向图常表示依赖、流程、单向连接
- 带权图常表示“代价”信息

---

## 五、图怎么存储

### 1. 邻接矩阵

- 用二维数组表示点和点是否相连

```cpp
int g[5][5];
```

### 2. 邻接表

- 对每个点，存它能到哪些点

```cpp
vector<int> g[5];
```

### 3. 当前阶段怎么理解

- 点少边多时，邻接矩阵更直观
- 点多边少时，邻接表更常用、更省空间

### 4. 当前阶段先记

- **刷题里最常见存图方式：邻接表**

---

## 六、邻接矩阵先有印象

### 1. 基本思路

```cpp
int g[4][4] = {0};
g[1][2] = 1;
g[2][1] = 1;
```

### 2. 怎么理解

- `g[i][j] = 1` 表示 `i` 和 `j` 有边
- 如果是带权图，可以把 `1` 换成权值

### 3. 优点和缺点

- 优点：判断两点是否相连很直接
- 缺点：空间开销大

---

## 七、邻接表先有印象

### 1. 基本思路

```cpp
vector<int> g[5];
g[1].push_back(2);
g[1].push_back(3);
```

### 2. 怎么理解

- `g[1]` 里存的是从点 `1` 能走到的点
- 如果是无向图，通常要双向加边

```cpp
g[1].push_back(2);
g[2].push_back(1);
```

### 3. 当前阶段先记

- **图论刷题默认优先想到邻接表**

---

## 八、DFS 深度优先搜索

### 1. 基本思路

- 一条路先走到底
- 走不动了再回头

### 2. 递归模板

```cpp
void dfs(int u) {
	vis[u] = true;
	for (int v : g[u]) {
		if (!vis[v]) {
			dfs(v);
		}
	}
}
```

### 3. 当前阶段怎么理解

- 先处理当前点
- 再递归访问它所有没访问过的邻居

### 4. 高频用途

- 连通块
- 路径搜索
- 图的遍历
- 回溯类图问题基础

---

## 九、BFS 广度优先搜索

### 1. 基本思路

- 一层一层往外扩展
- 先近后远

### 2. 队列模板

```cpp
void bfs(int start) {
	queue<int> q;
	q.push(start);
	vis[start] = true;
	while (!q.empty()) {
		int u = q.front();
		q.pop();
		for (int v : g[u]) {
			if (!vis[v]) {
				vis[v] = true;
				q.push(v);
			}
		}
	}
}
```

### 3. 当前阶段怎么理解

- 队列保证先发现的点先处理
- 所以 BFS 特别适合“最短步数”类问题

### 4. 高频用途

- 无权图最短路
- 层次遍历
- 最少步数问题

---

## 十、DFS 和 BFS 怎么区分

|遍历|核心思路|常用结构|高频印象|
|---|---|---|---|
|DFS|一条路走到底|递归 / 栈|深入搜索、回溯感强|
|BFS|一层一层扩展|队列|最短步数、分层|

### 当前阶段先记结论

- **想一层层扩展**用 BFS
- **想先一路走深**用 DFS

---

## 十一、为什么图遍历一定要判重

### 1. 图可能有环

```cpp
1 -> 2 -> 3 -> 1
```

### 2. 如果不记录访问状态

- 可能一直绕圈
- 会死循环或无限递归

### 3. 最常见写法

```cpp
bool vis[1005] = {false};
```

### 4. 当前阶段先记

- **图遍历几乎总要有 `vis` 数组 / 集合**

---

## 十二、当前阶段最常见写法

### 1. 邻接表建图

```cpp
vector<int> g[1005];
g[u].push_back(v);
```

### 2. 无向图加边

```cpp
g[u].push_back(v);
g[v].push_back(u);
```

### 3. DFS 模板

```cpp
void dfs(int u) {
	vis[u] = true;
	for (int v : g[u]) {
		if (!vis[v]) dfs(v);
	}
}
```

### 4. BFS 模板

```cpp
queue<int> q;
q.push(start);
vis[start] = true;
while (!q.empty()) {
	int u = q.front();
	q.pop();
	for (int v : g[u]) {
		if (!vis[v]) {
			vis[v] = true;
			q.push(v);
		}
	}
}
```

## 十三、现代 C++ 的基础习惯

### 1. 刷题优先考虑邻接表

- 大多数图题都更适合邻接表
- 特别是边不算非常密的时候

### 2. 先判断图有没有方向

- 无向图通常双向加边
- 有向图通常单向加边

### 3. 看到这些关键词先想到图

- 连通关系
- 路径
- 课程依赖
- 最短步数
- 网络 / 城市 / 节点连接

### 4. 以后会继续补

- 最短路算法
- 并查集
- 拓扑排序
- 最小生成树
- 但当前阶段先把存图和遍历学稳

---

## 十四、高频易错点

1. 图不是树，图可能有环
2. 无向图加边通常要加两次
3. 有向图和无向图不要混
4. BFS 用队列，DFS 常用递归 / 栈
5. 图遍历一般要判重访问
6. 邻接矩阵和邻接表不要混
7. 求最短步数时常优先想到 BFS
8. DFS 和 BFS 的访问顺序不要想反

---

### 邻接表模板

```cpp

```

### 加边模板


### DFS 模板

```cpp

	vis[u] = true;
	for (int v : g[u]) {

	}

```

### BFS 模板

```cpp
void bfs(int start) {
	queue<int> q;
	q.push(start);
	vis[start] = true;
	while (!q.empty()) {
		int u = q.front();
		q.pop();
		for (int v : g[u]) {
			if (!vis[v]) {
				vis[v] = true;
				q.push(v);
			}
		}
	}

```

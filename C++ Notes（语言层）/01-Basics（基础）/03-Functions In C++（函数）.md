#cpp #基础 #函数
## 一页速览

- **函数作用**：把一段可复用逻辑封装起来
- **基本结构**：返回类型 + 函数名 + 参数列表
- **调用方式**：`函数名(实参)`
- **参数传递**：值传递、指针传递、引用传递
- **常考点**：返回值、重载、默认参数、内联函数
- **最常错**：声明和定义混淆、只靠返回值区分重载、引用/值传递分不清

## 一、先总后分：函数学什么

### 1. 为什么要有函数

- 减少重复代码
- 让程序结构更清楚
- 便于调试、复用和维护

### 2. 函数最核心的 3 件事

- **输入**：参数
- **处理**：函数体
- **输出**：返回值

### 3. 学函数时先抓主线

- 会写函数
- 会调用函数
- 能分清参数怎么传
- 能看懂常见语法扩展

---

## 二、函数的基本写法

### 1. 基本结构

```cpp
返回类型 函数名(参数列表) {
	函数体;
}
```

### 2. 例子

```cpp
int add(int a, int b) {
	return a + b;
}
```

### 3. 调用

```cpp
int sum = add(3, 5);
cout << sum << '\\n';
```

### 速记

- `a`、`b`：形参位置对应接收数据
- `3`、`5`：调用时传入的实参

---

## 三、声明、定义、调用

### 1. 函数声明

```cpp
int add(int a, int b);
```

- 告诉编译器：这个函数存在
- 常放在前面，或放在头文件中

### 2. 函数定义

```cpp
int add(int a, int b) {
	return a + b;
}
```

- 真正写出函数内容

### 3. 先声明后调用

```cpp
#include <iostream>
using namespace std;

int add(int a, int b);

int main() {
	cout << add(2, 3) << '\\n';
	return 0;
}

int add(int a, int b) {
	return a + b;
}
```

## 四、参数传递

### 1. 值传递

```cpp
void change(int x) {
	x = 100;
}
```

- 改的是副本
- 不会影响原变量

### 2. 引用传递

```cpp
void change(int& x) {
	x = 100;
}
```

- 直接操作原变量
- 想修改外部数据时常用

### 3. 指针传递

```cpp
void change(int* p) {
	*p = 100;
}
```

- 也能修改原变量
- 需要注意空指针问题

### 4. 怎么选

- **不想改原值**：值传递
- **想改原值**：引用传递 / 指针传递
- **只读且避免拷贝大对象**：`const T&`

### 5. 推荐习惯

```cpp
void print(const string& s) {
	cout << s << '\\n';
}
```

- 大对象优先用 `const 引用`
- 更高效，也更安全

## 五、返回值

### 1. 有返回值函数

```cpp
int square(int x) {
	return x * x;
}
```

### 2. 无返回值函数

```cpp
void printHello() {
	cout << "hello" << '\\n';
}
```

### 3. 注意点

- `return` 后面的类型要和返回类型匹配
- `void` 函数一般不返回具体值
- 不是所有分支都返回值会出错

---

## 六、函数重载

### 1. 什么是重载

- 函数名相同
- 参数列表不同（个数 / 类型 / 顺序不同）

```cpp
int add(int a, int b) {
	return a + b;
}

double add(double a, double b) {
	return a + b;
}
```

### 2. 不能这样重载

```cpp
int func(int a);
// double func(int a); // ❌ 仅返回值不同，不算重载
```

### 3. 速记

- **看参数，不看返回值**

---

## 七、默认参数

```cpp
void func(int a, int b = 10, int c = 20) {
	cout << a + b + c << '\\n';
}
```

### 规则

- 默认参数必须从右往左连续给
- 调用时省略的是右边参数

```cpp
func(1);      // 1 + 10 + 20
func(1, 2);   // 1 + 2 + 20
func(1, 2, 3);
```

### 易错点

- 不要中间断开默认值

---

## 八、内联函数 `inline`

```cpp
inline int max2(int a, int b) {
	return a > b ? a : b;
}
```

- 适合短小、频繁调用的函数
- 本质是给编译器的建议，不一定真的展开
- 类内定义的短成员函数通常默认内联

---

## 九、递归的函数视角

### 1. 什么是递归

- 函数自己调用自己

```cpp
int factorial(int n) {
	if (n == 1) return 1;
	return n * factorial(n - 1);
}
```

### 2. 递归必须有

- **结束条件**
- **递归关系**

### 3. 易错点

- 少了结束条件会无限递归
- 层数太深可能栈溢出

---

## 十、现代 C++ 的函数习惯

### 1. 能加 `const` 就加

```cpp
void print(const string& s) {
	cout << s << '\\n';
}
```

### 2. 返回简单值就直接返回

- `int`、`double` 这类小对象直接返回即可

### 3. 参数命名清楚

- 少写 `a`、`b`
- 多写 `count`、`index`、`name`

### 4. 一个函数只做一件事

- 更容易测试
- 更容易复用

---

## 十一、函数高频易错点

1. 声明有分号，定义一般没有分号
2. 调用前若函数写在后面，要先声明
3. 重载看参数列表，不看返回值
4. 默认参数必须从右往左连续设置
5. 值传递不会修改原变量
6. 引用传递能修改原变量
7. 递归必须写结束条件
8. 大对象参数优先 `const T&`

---

## 十二、最简模板

### 普通函数

```cpp
int add(int a, int b) {
	return a + b;
}
```

### 引用参数

```cpp
void swapNum(int& a, int& b) {
	int t = a;
	a = b;
	b = t;
}
```

### 默认参数

```cpp
void printLine(char ch = '-', int n = 10) {
	for (int i = 0; i < n; ++i) cout << ch;
	cout << '\\n';
}
```

<aside> 📌

函数这一节最关键的不是把定义背下来，而是看到代码时能立刻判断：**参数怎么传、会不会改原值、有没有返回值、这是不是重载或默认参数。**

</aside>

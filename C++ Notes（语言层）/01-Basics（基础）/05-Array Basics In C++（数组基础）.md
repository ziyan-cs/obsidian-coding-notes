## ⚡ TL;DR（快速决策）

- 一组同类型数据、按下标访问 → 数组
- 需要连续存储、固定长度 → 原生数组很直接
- 需要动态长度 → 更常用 `vector`，不要硬套原生数组
- 题目强调下标、遍历、区间处理 → 优先想到数组模型
- 数组题真正关键通常不在定义，而在“下标边界、初始化、遍历方式是否正确”

## 🧩 Core Idea（核心本质）

- **本质**：数组是一段连续内存上的同类型元素集合
- **关键机制**：每个元素通过下标定位，因此可以做 O(1) 随机访问
- **核心优势**：结构简单、访问快、非常适合作为最基础的数据存储和算法载体

## 🔧 Usage Patterns（可复用代码模板）

### 1. 最基础一维数组

```cpp
int a[5] = {1, 2, 3, 4, 5};
```

### 2. 下标访问与修改

```cpp
cout << a[0] << '\n';
a[2] = 10;
```

### 3. 遍历数组

```cpp
for (int i = 0; i < n; ++i) {
	cout << a[i] << '\n';
}
```

### 4. 输入一组数据

```cpp
int n;
cin >> n;
int a[100];
for (int i = 0; i < n; ++i) {
	cin >> a[i];
}
```

### 5. 统计和 / 最值

```cpp
int sum = 0;
for (int i = 0; i < n; ++i) {
	sum += a[i];
}
```

### 6. 二维数组

```cpp
int g[3][4] = {0};
g[1][2] = 5;
```

### 7. 初始化为 0

```cpp
int cnt[100] = {0};
```

### 8. 配合算法库

```cpp
sort(a, a + n);
reverse(a, a + n);
```

## ⚠️ Pitfalls（高频错误）

- 数组下标从 `0` 开始，最后一个元素是 `a[n - 1]`，不是 `a[n]`
- `for (int i = 0; i <= n; ++i)` 很容易越界
- 原生数组长度固定，不能像 `vector` 一样动态扩容
- 数组不做边界检查，越界就是未定义行为
- 大数组开在局部作用域时，可能有爆栈风险
- 多维数组最容易写错的是行列下标顺序
- 把“数组长度”和“有效元素个数”混为一谈，会导致遍历错误
- 数组传函数时会退化成指针，函数里拿不到完整长度信息

## 🚀 Performance / Tips（性能优化）

- 数组真正高频的价值是：

```cpp
连续内存
下标访问快
遍历效率高
```

- 写数组题前优先先想清：

```cpp
有效范围是 0 到 n - 1
还是 1 到 n
```

- 需要动态长度时，优先用 `vector`
- 刷题里数组经常和：

```cpp
遍历
排序
前缀和
双指针
二分
```

一起出现

- 原生数组适合基础理解和固定大小场景，实战里很多时候 `vector` 更稳

## 🧪 Common Scenarios（常见使用场景）

- **输入一组数字**：一维数组存储
- **网格 / 矩阵**：二维数组
- **计数统计**：频率数组
- **下标遍历处理**：查找、修改、比较
- **排序与搜索**：数组配合算法库
- **后续算法载体**：前缀和、双指针、二分、滑动窗口

## 🧾 Minimal Template（最小可运行模板）

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
	ios::sync_with_stdio(false);
	cin.tie(nullptr);

	int n;
	cin >> n;
	int a[100];

	for (int i = 0; i < n; ++i) {
		cin >> a[i];
	}

	for (int i = 0; i < n; ++i) {
		cout << a[i] << '\n';
	}

	return 0;
}
```

## 📌 One-liner Summary（一句话总结）

👉 C++ 数组基础的核心不是“会写 `a[i]`”，而是把 **连续存储、下标范围、遍历方式和边界控制** 这几件事写稳，让数组真正成为后续算法的可靠载体。

#cpp #STL #容器 #队列 
## 一页速览

- **`queue`**：C++ 里最常用的队列容器
- **核心特点**：先进先出，只能操作队头和队尾
- **核心优势**：适合处理“先进入的元素先处理”这类问题
- **常见操作**：入队、出队、取队头、判空
- **高频接口**：`push`、`pop`、`front`、`back`、`empty`、`size`
- **高频场景**：层序遍历、BFS、排队处理、任务调度
- **最常错**：把队列和栈混淆、空队列直接 `front` / `pop`、误以为 `pop()` 有返回值

## 一、先总后分：这一节到底学什么

### 1. `queue` 解决什么问题

- 有些题目只关心“最先放进去的元素”
- 不需要中间随机访问
- 这时队列会比数组或栈更自然

### 2. `queue` 最核心看什么

- 什么叫先进先出
- 队头和队尾怎么变化
- 怎么入队和出队
- 什么场景该想到队列

### 3. 当前阶段先抓主线

- 会定义 `queue`
- 会写最常见的队列操作
- 会用队列解决基础问题
- 会分清队列和栈

## 二、`queue` 是什么

### 1. 基本理解

- `queue` 是一种队列结构容器
- 特点是：**先进去的元素先出来**
- 它通常只允许在两端操作：**队尾进入，队头离开**

```cpp
#include <queue>
queue<int> q;
```

### 2. 当前阶段怎么理解

- 可以把它想成“排队买东西”
- 先来的人先处理，后来的人排在后面

### 3. 当前阶段先记一句话

- **`queue` = 先进先出，队尾进，队头出**

---

## 三、`queue` 的定义与初始化

### 1. 最基础定义

```cpp
queue<int> q;
```

### 2. 怎么理解这句

- `int`：队列里存的元素类型
- `q`：队列变量名

### 3. 当前阶段先记

- **做题时最常见就是 `queue<int> q;`**

---

## 四、`queue` 的核心操作

### 1. 入队 `push`

```cpp
q.push(10);
q.push(20);
```

- 把元素放到队尾

### 2. 出队 `pop`

```cpp
q.pop();
```

- 删除队头元素
- **没有返回值**

### 3. 访问队头 `front`

```cpp
cout << q.front() << '\\n';
```

- 查看当前队头元素
- 但不删除它

### 4. 访问队尾 `back`

```cpp
cout << q.back() << '\\n';
```

- 查看当前队尾元素

### 5. 判空 `empty`

```cpp
if (q.empty()) {
	cout << "empty" << '\\n';
}
```

### 6. 获取大小 `size`

```cpp
cout << q.size() << '\\n';
```

### 7. 当前阶段先记

- **最常用：`push`、`pop`、`front`、`back`、`empty`、`size`**

---

## 五、队头和队尾怎么理解

### 1. 队头是什么

- 队头就是当前最先该被处理的元素
- 出队永远从队头开始

### 2. 队尾是什么

- 队尾就是最新进入队列的元素
- 新元素总是加到队尾

### 3. 一个简单过程

```cpp
q.push(1);
q.push(2);
q.push(3);
```

- 当前队头是 `1`
- 当前队尾是 `3`

```cpp
q.pop();
```

- `1` 被删除
- 现在队头变成 `2`

### 4. 当前阶段怎么理解

- **新元素总是在队尾进入**
- **删除元素总是从队头开始**

---

## 六、`queue` 的遍历怎么理解

### 1. STL `queue` 不支持像数组那样直接遍历

- 它不支持下标访问
- 也不暴露中间元素
- 重点就是队头和队尾操作

### 2. 如果想依次输出

```cpp
while (!q.empty()) {
	cout << q.front() << '\\n';
	q.pop();
}
```

### 3. 当前阶段先记

- **队列不是拿来随机访问的，重点是只看队头 / 队尾**

---

## 七、`queue` 和栈的区别

|对比项|队列|栈|
|---|---|---|
|规则|先进先出|先进后出|
|操作位置|队头出，队尾进|只操作栈顶|
|典型场景|排队、BFS|括号匹配、回退|

### 当前阶段先记结论

- **先进先出**想队列
- **后进先出**想栈

---

## 八、`queue` 最常见应用

### 1. 层序遍历

- 一层一层访问二叉树节点时很常用队列
- 先进入队列的节点，先被访问

### 2. BFS 广度优先搜索

- 图和网格的广搜通常用队列
- 先扩展当前层，再扩展下一层

### 3. 排队处理

- 谁先进入，谁先处理
- 很符合现实中的排队逻辑

### 4. 任务调度先有印象

- 某些任务按提交顺序执行
- 背后常有队列思想

---

## 九、层序遍历先有印象

### 1. 基本思路

```cpp
queue<TreeNode*> q;
q.push(root);
```

### 2. 怎么做

- 先把根节点入队
- 每次取出队头节点
- 再把它的左右孩子加入队尾

### 3. 为什么这题适合用队列

- 因为先进入队列的节点，要先被处理
- 这正是先进先出的特点

## 十一、现代 C++ 的基础习惯

### 1. 出队前先判空

- `front()` 和 `pop()` 前都要先考虑队列是否为空
- 空队列操作是高频错误

### 2. 不要把队列当数组用

- 队列的价值不在“存很多数据”
- 而在“最早进入的元素优先处理”

### 3. 看到这些关键词先想到队列

- 层序遍历
- BFS
- 先进先出
- 排队处理

### 4. 以后会继续补

- 双端队列 `deque`
- 优先队列 `priority_queue`
- 单调队列
- 但当前阶段先把基础操作学稳

---

## 十二、高频易错点

1. 队列是先进先出，不是先进后出
2. `front()` 只看队头，不删除元素
3. `pop()` 删除队头，但不返回值
4. 空队列时不能直接 `front()` 或 `pop()`
5. 队列不能随机访问中间元素
6. 队头和队尾不要搞反
7. 层序遍历和 BFS 很常用队列
8. 输出队列内容时常常是一边访问一边弹出

---

## 十三、最简模板

### 定义模板

```cpp

```

### 入队出队模板

```cpp

```

```cpp

```

```cpp
while (!q.empty()) {
	cout << q.front() << '\\n';

}
```

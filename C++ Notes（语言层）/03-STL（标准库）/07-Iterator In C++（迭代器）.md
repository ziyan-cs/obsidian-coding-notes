#cpp #STL #迭代器
## 一页速览

- **迭代器**：用来遍历容器元素的“通用指针式工具”
- **核心特点**：不同容器都能用类似方式访问元素
- **核心优势**：统一遍历、查找、配合 STL 算法
- **常见操作**：获取 `begin` / `end`、解引用、移动迭代器、比较是否结束
- **高频写法**：`auto it = v.begin()`、`*it`、`++it`
- **高频场景**：遍历 `vector`、`set`、`map`，配合 `find`、`sort`、`reverse`
- **最常错**：把迭代器当下标、对 `end()` 解引用、删除元素后继续用失效迭代器、`it++` 和 `++it` 不分

## 一、先总后分：这一节到底学什么

### 1. 迭代器解决什么问题

- 不同容器的访问方式不完全一样
- 有的支持下标，有的不支持下标
- 迭代器提供了一套更统一的访问方式

### 2. 迭代器最核心看什么

- `begin()` 和 `end()` 是什么
- 怎么通过迭代器访问元素
- 怎么移动迭代器
- 什么情况下该用迭代器而不是下标

### 3. 当前阶段先抓主线

- 会定义和使用基础迭代器
- 会遍历常见容器
- 会读懂 STL 算法里的迭代器参数
- 会避开最常见的失效和越界问题

## 二、迭代器是什么

### 1. 基本理解

- 迭代器可以理解成“指向容器元素的位置”
- 它的作用有点像指针，但不是普通裸指针
- STL 里很多容器和算法都围绕它工作

```cpp
vector<int> v = {10, 20, 30};
auto it = v.begin();
```

### 2. 当前阶段怎么理解

- `it` 指向当前元素位置
- `*it` 可以取到这个位置上的值
- `++it` 可以让它往后移动

### 3. 当前阶段先记一句话

- **迭代器 = 用来访问和移动容器元素位置的工具**

---

## 三、`begin()` 和 `end()` 是什么

### 1. `begin()`

- 指向容器的第一个元素

### 2. `end()`

- 指向“最后一个元素的下一个位置”
- 它是结束标记，不是真正元素

```cpp
vector<int> v = {1, 2, 3};
auto it1 = v.begin();
auto it2 = v.end();
```

### 3. 当前阶段怎么理解

- `begin()` 可以访问
- `end()` 不能解引用

### 4. 当前阶段先记

- **遍历时常写：`it != v.end()`**

---

## 四、迭代器怎么访问元素

### 1. 解引用 `*it`

```cpp
vector<int> v = {10, 20, 30};
auto it = v.begin();
cout << *it << '\\n'; // 10
```

### 2. 移动到下一个元素

```cpp
++it;
cout << *it << '\\n'; // 20
```

### 3. 当前阶段怎么理解

- `it`：表示位置
- `*it`：表示该位置上的元素

### 4. 最常错的地方

- `it` 不是元素本身
- `*it` 才是元素

---

## 五、迭代器的基础遍历

### 1. 遍历 `vector`

```cpp
vector<int> v = {1, 2, 3};
for (auto it = v.begin(); it != v.end(); ++it) {
	cout << *it << '\\n';
}
```

### 2. 遍历 `set`

```cpp
set<int> s = {3, 1, 2};
for (auto it = s.begin(); it != s.end(); ++it) {
	cout << *it << '\\n';
}
```

### 3. 遍历 `map`

```cpp
map<string, int> mp;
mp["Tom"] = 95;

for (auto it = mp.begin(); it != mp.end(); ++it) {
	cout << it->first << " " << it->second << '\\n';
}
```

### 4. 当前阶段先记

- **遍历模板核心就是：起点 `begin()`，终点 `end()`，中间 `++it`**

---

## 六、不同容器里的迭代器怎么理解

### 1. `vector` 的迭代器

- 很像指针
- 因为 `vector` 元素通常连续存放

### 2. `set` 的迭代器

- 能遍历，但不能像数组那样下标访问
- 元素值通常不允许直接改

### 3. `map` 的迭代器

- 每个位置是一个键值对
- 常用 `it->first` 和 `it->second`

### 4. 当前阶段先记

- **不是所有容器都支持下标，但大多数 STL 容器都支持迭代器**

---

## 七、`it->first` 和 `it->second` 怎么理解

### 1. 为什么 `map` 特别常见这个写法

- 因为 `map` 里的每个元素本质上是键值对

```cpp
map<string, int> mp;
mp["Tom"] = 100;
auto it = mp.begin();
cout << it->first << '\\n';
cout << it->second << '\\n';
```

### 2. 当前阶段怎么理解

- `it->first`：键
- `it->second`：值

### 3. 速记

- **`map` 迭代器最常考的就是 `first` 和 `second`**

---

## 八、迭代器和 STL 算法

### 1. `find`

```cpp
auto it = find(v.begin(), v.end(), 3);
if (it != v.end()) {
	cout << *it << '\\n';
}
```

### 2. `sort`

```cpp
sort(v.begin(), v.end());
```

### 3. `reverse`

```cpp
reverse(v.begin(), v.end());
```

### 4. 当前阶段怎么理解

- 很多 STL 算法不直接收“容器”
- 而是收“迭代器区间”

### 5. 当前阶段先记

- **`[begin(), end())` 是 STL 里非常高频的区间写法**

---

## 九、`[begin(), end())` 怎么理解

### 1. 这个区间表示什么

- 从 `begin()` 开始
- 到 `end()` 结束
- 包含左边，不包含右边

### 2. 为什么这样设计

- 方便表达空区间
- 方便和长度、范围统一处理

### 3. 当前阶段怎么理解

- `end()` 是结束标记，不参与真正访问

### 4. 当前阶段先记

- **算法里的 `end()` 几乎都是“边界”，不是元素**

---

## 十、当前阶段最常见写法

### 1. 遍历 `vector`

```cpp
for (auto it = v.begin(); it != v.end(); ++it) {
	cout << *it << '\\n';
}
```

### 2. 查找元素

```cpp
auto it = find(v.begin(), v.end(), x);
if (it != v.end()) {
	cout << *it << '\\n';
}
```

### 3. 遍历 `map`

```cpp
for (auto it = mp.begin(); it != mp.end(); ++it) {
	cout << it->first << " " << it->second << '\\n';
}
```

### 4. 排序区间

```cpp
sort(v.begin(), v.end());
```

## 十一、现代 C++ 的基础习惯

### 1. 能用 `auto` 就优先用 `auto`

- 迭代器类型通常很长
- `auto` 可读性更好

```cpp
auto it = v.begin();
```

### 2. 遍历时优先 `++it`

- 这是 STL 里更常见、更推荐的写法
- 尤其在泛型和迭代器语境里更自然

### 3. 看到 STL 算法先想到迭代器区间

- `sort`
- `find`
- `reverse`
- `lower_bound`

### 4. 以后会继续补

- 反向迭代器
- 常量迭代器
- 迭代器失效细节
- 不同迭代器分类
- 但当前阶段先把基础用法学稳

---

## 十二、高频易错点

1. `begin()` 指向第一个元素，`end()` 不是最后一个元素
2. `end()` 不能解引用
3. `it` 是位置，`*it` 才是元素
4. `map` 迭代器常用 `it->first` 和 `it->second`
5. 不是所有容器都支持下标访问
6. 删除元素后，原迭代器可能失效
7. `find` 找不到时返回的是对应容器的 `end()`
8. `it++` 和 `++it` 都能写，但当前阶段更推荐 `++it`

---

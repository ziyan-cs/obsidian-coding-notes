## 一页速览

- **`priority_queue`**：C++ 里最常用的优先队列容器
- **核心特点**：不是先进先出，而是“优先级高的先出来”
- **默认规则**：默认是大根堆，堆顶最大
- **核心优势**：快速维护当前最大值或最小值
- **常见操作**：入堆、出堆、取堆顶、判空
- **高频接口**：`push`、`pop`、`top`、`empty`、`size`
- **高频场景**：Top K、动态最值、贪心、合并问题、调度问题
- **最常错**：把优先队列和普通队列混淆、忘记默认是大根堆、小根堆写法记错、误以为 `pop()` 有返回值

## 一、先总后分：这一节到底学什么

### 1. `priority_queue` 解决什么问题

- 有些题目不要求按进入顺序处理元素
- 只关心“当前最大”或“当前最小”是谁
- 这时优先队列会比普通队列更自然

### 2. `priority_queue` 最核心看什么

- 什么叫优先级高的先出来
- 默认为什么是大根堆
- 小根堆怎么写
- 什么场景该想到优先队列

### 3. 当前阶段先抓主线

- 会定义 `priority_queue`
- 会写最常见的堆操作
- 会分清大根堆和小根堆
- 会用它解决基础最值问题

## 二、`priority_queue` 是什么

### 1. 基本理解

- `priority_queue` 是一种优先队列容器
- 它的出队顺序不是“谁先进去谁先出来”
- 而是“谁优先级高谁先出来”

```cpp
#include <queue>
priority_queue<int> pq;
```

### 2. 当前阶段怎么理解

- 可以把它想成“按重要程度排队”
- 不是按到达顺序，而是按大小规则决定谁先出

### 3. 当前阶段先记一句话

- **`priority_queue` = 按优先级出队的容器**

---

## 三、默认是大根堆

### 1. 默认规则

```cpp
priority_queue<int> pq;
```

- 默认情况下，堆顶是最大值

### 2. 基本操作示例

```cpp
pq.push(10);
pq.push(30);
pq.push(20);

cout << pq.top() << '\\n'; // 30
pq.pop();
cout << pq.top() << '\\n'; // 20
```

### 3. 当前阶段怎么理解

- 进入顺序是 `10 -> 30 -> 20`
- 但出来顺序优先看大小，不看先后

### 4. 当前阶段先记

- **默认 `priority_queue<int>` 就是大根堆**

---

## 四、小根堆怎么写

### 1. 最常用写法

```cpp
#include <queue>
#include <vector>
#include <functional>
priority_queue<int, vector<int>, greater<int>> pq;
```

### 2. 怎么理解这句

- 第一个 `int`：元素类型
- `vector<int>`：底层容器
- `greater<int>`：比较规则，表示小的优先

### 3. 基本操作示例

```cpp
pq.push(10);
pq.push(30);
pq.push(20);

cout << pq.top() << '\\n'; // 10
```

### 4. 当前阶段先记

- **默认是大根堆**
- **想要小根堆要手动写 `greater<int>`**

---

## 五、`priority_queue` 的核心操作

### 1. 入队 `push`

```cpp
pq.push(10);
pq.push(20);
```

- 把元素放入堆中
- 系统会自动调整堆结构

### 2. 出队 `pop`

```cpp
pq.pop();
```

- 删除堆顶元素
- **没有返回值**

### 3. 访问堆顶 `top`

```cpp
cout << pq.top() << '\\n';
```

- 查看当前优先级最高的元素
- 但不删除它

### 4. 判空 `empty`

```cpp
if (pq.empty()) {
	cout << "empty" << '\\n';
}
```

### 5. 获取大小 `size`

```cpp
cout << pq.size() << '\\n';
```

### 6. 当前阶段先记

- **最常用：`push`、`pop`、`top`、`empty`、`size`**

---

## 六、`priority_queue` 和 `queue` 的区别

|对比项|`queue`|`priority_queue`|
|---|---|---|
|出队规则|先进先出|优先级高的先出|
|默认堆顶|队头最早进入|最大值|
|典型场景|BFS、排队处理|Top K、最值维护|

### 当前阶段先记结论

- **按进入顺序处理**想 `queue`
- **按大小或优先级处理**想 `priority_queue`

---

## 七、堆顶怎么理解

### 1. 堆顶是什么

- 堆顶就是当前优先级最高的元素
- 默认大根堆时，堆顶就是最大值
- 小根堆时，堆顶就是最小值

### 2. 一个简单过程

```cpp
pq.push(5);
pq.push(1);
pq.push(8);
```

- 如果是默认大根堆
- 当前堆顶是 `8`

```cpp
pq.pop();
```

- `8` 被删除
- 现在堆顶变成 `5`

### 3. 当前阶段怎么理解

- **`priority_queue` 的重点永远在堆顶**
- 你通常最关心的就是“当前最值是谁”

---

## 八、`priority_queue` 最常见应用

### 1. Top K 问题

- 找前 K 大或前 K 小
- 优先队列是高频工具

### 2. 动态最值维护

- 数据不断加入时
- 想随时知道当前最大 / 最小值

### 3. 贪心问题

- 每次都拿当前最优元素处理
- 这类题常和优先队列有关

### 4. 合并 / 调度问题

- 每次取当前最小或最大继续运算
- 非常适合优先队列

---

## 九、Top K 先有印象

### 1. 典型思路

- 找最小的几个数时，常用大根堆维护答案
- 找最大的几个数时，常用小根堆维护答案

### 2. 当前阶段怎么理解

- 堆里只保留“当前最有价值的 K 个元素”
- 不需要把所有数据完整排序

### 3. 当前阶段先记

- **Top K 很多时候比排序更该先想到优先队列**

## 十、当前阶段最常见写法

### 1. 大根堆

```cpp
priority_queue<int> pq;
```

### 2. 小根堆

```cpp
priority_queue<int, vector<int>, greater<int>> pq;
```

### 3. 入堆出堆

## 十一、现代 C++ 的基础习惯

### 1. 看到这些关键词先想到优先队列

- Top K
- 当前最大 / 最小
- 优先级
- 每次取最优

### 2. 先记默认是大根堆

- 这是最常见的出错点之一
- 小根堆写法一定要单独记牢

### 3. 不要把优先队列当普通队列

- 它不按进入顺序出队
- 它按优先级出队

### 4. 以后会继续补

- 自定义比较器
- 存 `pair`
- 配合贪心和图论
- 但当前阶段先把基础写法学稳

---

## 十二、高频易错点

1. `priority_queue` 默认是大根堆，不是小根堆
2. 小根堆写法容易忘
3. `top()` 只看堆顶，不删除元素
4. `pop()` 删除堆顶，但不返回值
5. 空优先队列时不能直接 `top()` 或 `pop()`
6. 优先队列不是按进入顺序出队
7. `priority_queue` 和普通 `queue` 不要混
8. Top K 问题常更适合优先队列，不一定先排序

#cpp #STL #容器 #栈
## 一页速览

- **`stack`**：C++ 里最常用的栈容器
- **核心特点**：先进后出，只能操作栈顶
- **核心优势**：适合处理“最后进入的元素先处理”这类问题
- **常见操作**：入栈、出栈、取栈顶、判空
- **高频接口**：`push`、`pop`、`top`、`empty`、`size`
- **高频场景**：括号匹配、表达式处理、撤销回退、单调栈基础
- **最常错**：把栈和队列混淆、空栈直接 `top` / `pop`、误以为 `pop()` 有返回值

## 一、先总后分：这一节到底学什么

### 1. `stack` 解决什么问题

- 有些题目只关心“最后放进去的元素”
- 不需要中间随机访问
- 这时栈会比数组或队列更自然

### 2. `stack` 最核心看什么

- 什么叫先进后出
- 栈顶怎么变化
- 怎么入栈和出栈
- 什么场景该想到栈

### 3. 当前阶段先抓主线

- 会定义 `stack`
- 会写最常见的栈操作
- 会用栈解决基础问题
- 会分清栈和队列

## 二、`stack` 是什么

### 1. 基本理解

- `stack` 是一种栈结构容器
- 特点是：**后进去的元素先出来**
- 它只允许在一端操作，这一端叫**栈顶**

```cpp
#include <stack>
stack<int> st;
```

### 2. 当前阶段怎么理解

- 可以把它想成“一摞书”
- 只能从最上面放书，也只能从最上面拿书

### 3. 当前阶段先记一句话

- **`stack` = 先进后出，只能操作栈顶的容器**

---

## 三、`stack` 的定义与初始化

### 1. 最基础定义

```cpp
stack<int> st;
```

### 2. 怎么理解这句

- `int`：栈里存的元素类型
- `st`：栈变量名

### 3. 当前阶段先记

- **做题时最常见就是 `stack<int> st;`**

---

## 四、`stack` 的核心操作

### 1. 入栈 `push`

```cpp
st.push(10);
st.push(20);
```

- 把元素放到栈顶

### 2. 出栈 `pop`

```cpp
st.pop();
```

- 删除栈顶元素
- **没有返回值**

### 3. 访问栈顶 `top`

```cpp
cout << st.top() << '\\n';
```

- 查看当前栈顶元素
- 但不删除它

### 4. 判空 `empty`

```cpp
if (st.empty()) {
	cout << "empty" << '\\n';
}
```

### 5. 获取大小 `size`

```cpp
cout << st.size() << '\\n';
```

### 6. 当前阶段先记

- **最常用：`push`、`pop`、`top`、`empty`、`size`**

---

## 五、栈顶怎么理解

### 1. 栈顶是什么

- 栈顶就是最后一个进入栈的元素
- 后续操作几乎都围绕栈顶展开

### 2. 一个简单过程

```cpp
st.push(1);
st.push(2);
st.push(3);
```

- 当前栈顶是 `3`

```cpp
st.pop();
```

- `3` 被删除
- 现在栈顶变成 `2`

### 3. 当前阶段怎么理解

- **新元素总是在栈顶进入**
- **删除元素也总是从栈顶开始**

---

## 六、`stack` 的遍历怎么理解

### 1. STL `stack` 不支持像数组那样直接遍历

- 它不支持下标访问
- 也不暴露中间元素
- 重点就是栈顶操作

### 2. 如果想依次输出

```cpp
while (!st.empty()) {
	cout << st.top() << '\\n';
	st.pop();
}
```

### 3. 当前阶段先记

- **栈不是拿来随机访问的，重点是只看栈顶**

---

## 七、`stack` 和队列的区别

|对比项|栈|队列|
|---|---|---|
|规则|先进后出|先进先出|
|操作位置|只操作栈顶|队头出，队尾进|
|典型场景|括号匹配、回退|排队、BFS|

### 当前阶段先记结论

- **后进先出**想栈
- **先进先出**想队列

---

## 八、`stack` 最常见应用

### 1. 括号匹配

- 左括号入栈
- 右括号拿来和栈顶匹配
- 最后栈空且过程合法，说明匹配成功

### 2. 表达式处理

- 中缀、后缀表达式转换常和栈有关
- 当前阶段先知道“表达式题常想到栈”即可

### 3. 撤销 / 回退

- 因为最近一步操作，往往最先被撤销
- 这正符合后进先出

### 4. 单调栈先有印象

- 处理“下一个更大元素”这类题很常见
- 后面刷题会经常遇到

---

## 九、括号匹配先有印象

### 1. 基本思路

```cpp
string s = "()[]{}";
stack<char> st;
```

### 2. 怎么做

- 遇到左括号：入栈
- 遇到右括号：
    - 先判断栈是否为空
    - 再判断栈顶是否匹配

### 3. 为什么这题适合用栈

- 因为最后出现的左括号，要最先完成匹配
- 这正是先进后出的特点

## 十一、现代 C++ 的基础习惯

### 1. 出栈前先判空

- `top()` 和 `pop()` 前都要先考虑栈是否为空
- 空栈操作是高频错误

### 2. 不要把栈当数组用

- 栈的价值不在“保存很多数据”
- 而在“最后进入的元素优先处理”

### 3. 看到这些关键词先想到栈

- 括号匹配
- 后进先出
- 回退 / 撤销
- 最近一个未处理元素

### 4. 以后会继续补

- 单调栈
- 用栈实现队列
- 表达式求值
- 但当前阶段先把基础操作学稳

---

## 十二、高频易错点

1. 栈是先进后出，不是先进先出
2. `top()` 只看栈顶，不删除元素
3. `pop()` 删除栈顶，但不返回值
4. 空栈时不能直接 `top()` 或 `pop()`
5. 栈不能随机访问中间元素
6. 栈和队列规则不要混
7. 括号匹配题很常用栈
8. 输出栈内容时常常是一边访问一边弹出

---

## 十三、最简模板

### 定义模板

```cpp
#include <stack>
stack<int> st;
```

### 入栈出栈模板

```cpp
st.push(10);
st.push(20);
st.pop();
```

### 访问栈顶模板

```cpp
if (!st.empty()) {
	cout << st.top() << '\\n';
}
```

### 清空式遍历模板

```cpp
while (!st.empty()) {
	cout << st.top() << '\\n';
	st.pop();
}
```

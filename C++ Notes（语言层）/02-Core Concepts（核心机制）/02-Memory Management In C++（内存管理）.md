#cpp #核心机制 #内存管理
## 一页速览

- **内存四区先有印象**：代码区、全局区、栈区、堆区
- **局部变量**：通常在栈上
- **动态内存**：`new` / `delete`
- **数组动态申请**：`new[]` / `delete[]`
- **空指针**：`nullptr`
- **最常错**：忘记释放、重复释放、野指针、栈内存和堆内存分不清

## 一、先总后分：这一节学什么

### 1. 为什么要学内存管理

- 理解变量为什么“出了作用域就没了”
- 理解为什么有些数据能在函数外继续使用
- 为指针、类对象、STL、面向对象打基础

### 2. 这一节的主线

- 先分清内存区域
- 再理解栈和堆
- 再掌握 `new` / `delete`
- 最后记住高频内存错误

### 3. 当前阶段的重点

- 会判断对象大概在哪块内存
- 会写基础动态申请与释放
- 能识别常见危险写法

## 二、先建立整体图景：内存四区

### 1. 代码区

- 存放函数体的二进制代码
- 一般是只读的

### 2. 全局区

- 存放全局变量、静态变量、字符串常量等
- 生命周期通常贯穿整个程序运行期

### 3. 栈区

- 存放局部变量、函数参数等
- 由编译器自动分配和回收
- 速度快，但作用域结束就会释放

### 4. 堆区

- 由程序员手动申请和释放
- 常用于运行时大小不确定的数据
- 灵活，但最容易出错

### 速记

- **局部变量**：多半在栈
- **动态申请**：在堆
- **全局 / 静态**：在全局区

---

## 三、栈内存

### 1. 栈上的变量

```cpp
void func() {
	int x = 10;
}
```

- `x` 是局部变量
- `func()` 结束后，`x` 的内存通常自动回收

### 2. 栈的特点

- 自动管理，不需要手动释放
- 分配 / 回收快
- 生命周期和作用域强相关

### 3. 高频坑

```cpp
int* func() {
	int x = 10;
	return &x; // ❌ 返回局部变量地址
}
```

- 局部变量离开作用域后就失效
- 返回它的地址会出问题

---

## 四、堆内存与动态申请

### 1. 为什么需要堆

- 有些数据大小运行时才知道
- 有些数据希望超出当前作用域继续存在

### 2. `new` 申请单个对象

```cpp
int* p = new int(10);
cout << *p << '\\n';
```

- `p` 存的是堆上那块内存的地址
- 这块内存不会因为当前作用域结束自动释放

### 3. `delete` 释放单个对象

```cpp
delete p;
p = nullptr;
```

- 释放后最好立刻置空
- 避免悬空指针继续被误用

### 4. `new[]` 申请数组

```cpp
int* arr = new int[5];
```

### 5. `delete[]` 释放数组

```cpp
delete[] arr;
arr = nullptr;
```

### 速记

- `new` 对 `delete`
- `new[]` 对 `delete[]`

---

## 五、动态内存的完整流程

```cpp
int* p = new int(100);
cout << *p << '\\n';
delete p;
p = nullptr;
```

### 标准步骤

1. 定义指针
2. 用 `new` 申请内存
3. 通过指针使用数据
4. 用 `delete` 释放内存
5. 置为 `nullptr`

---

## 六、常见内存错误

### 1. 内存泄漏

```cpp
int* p = new int(10);
// 忘记 delete p; // ❌
```

- 申请了堆内存，却没有释放
- 程序运行久了会越来越占内存

### 2. 野指针

```cpp
int* p;
// *p = 10; // ❌ p 没有初始化
```

- 指向不明位置的指针
- 非常危险

### 3. 悬空指针

```cpp
int* p = new int(10);
delete p;
// cout << *p; // ❌ 释放后还用
```

- 指针还在，但它指向的内存已经没了

### 4. 重复释放

```cpp
int* p = new int(10);
delete p;
// delete p; // ❌
```

- 同一块内存不能重复 `delete`

### 5. 申请和释放不匹配

```cpp
int* arr = new int[5];
// delete arr; // ❌
delete[] arr; // ✅
```

- 数组要用 `delete[]`

---

## 七、静态变量与生命周期

```cpp
void func() {
	static int cnt = 0;
	++cnt;
	cout << cnt << '\\n';
}
```

- `static` 局部变量不随着函数结束而销毁
- 只初始化一次
- 生命周期贯穿整个程序运行期

### 和普通局部变量区别

- 普通局部变量：函数结束就销毁
- `static` 局部变量：一直保留

---

## 八、对象角度理解内存

### 1. 普通对象

```cpp
Person p;
```

- 若写在函数内部，通常在栈上

### 2. 动态对象

```cpp
Person* p = new Person();
```

- 对象本体在堆上
- 指针变量若是局部的，指针本身在栈上

### 3. 常考理解

- **指针在栈上，不代表对象也在栈上**
- 要分清“指针变量在哪”和“它指向的对象在哪”

---

## 九、现代 C++ 的视角

### 1. 能不用手写 `new/delete`，就尽量少手写

- 现代 C++ 更推荐用 `vector`、`string`、智能指针等工具管理资源
- 当前阶段先把底层原理学明白

### 2. 先养成两个习惯

- 申请后记得释放
- 释放后立刻置 `nullptr`

### 3. 以后会接触

- `vector`：自动管理动态数组
- 智能指针：自动管理堆内存
- RAII：对象生命周期绑定资源管理

### 当前阶段结论

- 先会手写基础 `new/delete`
- 再理解为什么现代 C++ 不鼓励到处裸写指针

---

## 十、高频易错点

1. 局部变量通常在栈上，不能随便返回它的地址
2. `new` 出来的内存不会自动释放
3. `delete` 后不要继续使用该指针
4. `new[]` 必须配 `delete[]`
5. 指针未初始化就使用，容易形成野指针
6. 释放后最好设为 `nullptr`
7. 分清“指针变量在哪”和“对象本体在哪”
8. `static` 局部变量生命周期不同于普通局部变量

---

## 十一、最简模板

### 单个变量动态申请

```cpp
int* p = new int(10);
cout << *p << '\\n';
delete p;
p = nullptr;
```

### 动态数组

```cpp
int* arr = new int[5];
for (int i = 0; i < 5; ++i) {
	arr[i] = i;
}
delete[] arr;
arr = nullptr;
```

### 错误示例：返回局部变量地址

```cpp
int* func() {
	int x = 10;
	return &x; // ❌
}
```

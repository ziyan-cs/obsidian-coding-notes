#cpp #核心机制 #指针 #引用
## 一页速览

- **指针**：存地址
- **引用**：变量的别名
- **取地址**：`&x`
- **解引用**：`*p`
- **空指针**：`nullptr`
- **最常用场景**：函数传参、动态内存、遍历与间接修改
- **最常错**：`*` 和 `&` 混淆、空指针解引用、`const` 指针分不清

## 一、先总后分：这一节学什么

### 1. 指针解决什么问题

- 间接访问数据
- 在函数里修改外部变量
- 为动态内存和部分底层机制打基础

### 2. 引用解决什么问题

- 给变量起别名
- 让传参更自然
- 比部分指针写法更简洁

### 3. 学这一节的主线

- 分清“值”和“地址”
- 看懂 `*` 和 `&`
- 分清指针传递与引用传递
- 记住 `const` 与空指针这两个高频坑

---

## 二、先理解地址

```cpp
int x = 10;
cout << x << '\\n';   // 值
cout << &x << '\\n';  // 地址
```

- 变量有自己的值，也有自己的内存地址
- 指针本质上就是“专门存地址的变量”

### 速记

- `x`：值
- `&x`：x 的地址

---

## 三、指针的基本概念

### 1. 定义指针

```cpp
int x = 10;
int* p = &x;
```

- `p` 是指针变量
- `p` 里存的是 `x` 的地址

### 2. 解引用

```cpp
cout << *p << '\\n';
```

- `*p` 表示“根据地址找到那个值”
- 如果 `p` 指向 `x`，那么 `*p` 就是 `x`

### 3. 通过指针修改值

```cpp
*p = 20;
cout << x << '\\n'; // 20
```

- 改 `*p`，本质就是改它指向的变量

### 速记

- `p`：地址
- `*p`：地址对应的值

---

## 四、引用的基本概念

### 1. 定义引用

```cpp
int x = 10;
int& r = x;
```

- `r` 是 `x` 的引用
- 可以把它理解为 `x` 的别名

### 2. 使用引用

```cpp
r = 20;
cout << x << '\\n'; // 20
```

- 改 `r` 就是在改 `x`

### 3. 引用的特点

- 引用定义时必须初始化
- 引用一旦绑定，就不能改绑到别的变量
- 语法上比指针更自然

---

## 五、指针 vs 引用

|对比项|指针|引用|
|---|---|---|
|本质|存地址的变量|变量别名|
|是否必须初始化|不必须|必须|
|能否改指向 / 绑定|可以改指向|不能改绑|
|是否可为空|可以为 `nullptr`|通常不能为空|
|使用语法|要配合 `*`|像普通变量|

### 结论

- **想表达“可能不指向任何对象”**：用指针
- **想表达“就是这个对象的别名”**：优先引用

---

## 六、函数里的指针与引用

### 1. 值传递

```cpp
void change(int x) {
	x = 100;
}
```

- 改的是副本
- 不影响外部变量

### 2. 指针传递

```cpp
void change(int* p) {
	*p = 100;
}
```

- 可以修改外部变量
- 调用时要传地址：`change(&x);`

### 3. 引用传递

```cpp
void change(int& x) {
	x = 100;
}
```

- 也可以修改外部变量
- 调用时更自然：`change(x);`

### 4. 怎么选

- 想写得直观：引用
- 允许为空 / 需要改指向语义：指针

---

## 七、`const` 与指针（高频难点）

```cpp
int x = 10;
int y = 20;

const int* p1 = &x;
int* const p2 = &x;
const int* const p3 = &x;
```

### 1. `const int* p1`

- 不能通过 `p1` 修改值
- 但 `p1` 自己可以改指向

```cpp
p1 = &y;   // ✅
// *p1 = 30; // ❌
```

### 2. `int* const p2`

- 可以通过 `p2` 改值
- 但 `p2` 本身不能改指向

```cpp
*p2 = 30;  // ✅
// p2 = &y; // ❌
```

### 3. `const int* const p3`

- 值不能改
- 指向也不能改

### 速记

- `const` 靠近谁，谁就更“不能改”

---

## 八、空指针 `nullptr`

```cpp
int* p = nullptr;
```

- 表示当前不指向任何对象
- C++11 起优先用 `nullptr`，不要优先用 `NULL`

### 大坑

```cpp
int* p = nullptr;
// cout << *p; // ❌ 不能解引用空指针
```

- 空指针不能直接 `*p`

### 常见防御写法

```cpp
if (p != nullptr) {
	cout << *p << '\\n';
}
```

---

## 九、指针运算（先有印象）

```cpp
int arr[3] = {10, 20, 30};
int* p = arr;
cout << *p << '\\n';       // 10
cout << *(p + 1) << '\\n'; // 20
```

- 指针加 1，不是简单地址加 1
- 而是移动到“下一个同类型元素”
- 这一点和数组联系很紧密

### 当前阶段结论

- 会看懂即可
- 深入细节等学数组 / 内存时再展开

## 十、引用的常见用法

### 1. 交换变量

```cpp
void swapNum(int& a, int& b) {
	int t = a;
	a = b;
	b = t;
}
```

### 2. 只读大对象参数

```cpp
void print(const string& s) {
	cout << s << '\\n';
}
```

- 避免拷贝
- 又防止误修改

### 3. 范围 `for`

```cpp
for (auto& x : vec) {
	x *= 2;
}
```

- `auto&` 表示直接操作原元素

---

## 十一、现代 C++ 的使用习惯

### 1. 优先用 `nullptr`

- 更安全
- 类型更明确

### 2. 能用引用表达的参数，优先考虑引用

- 写法更清楚
- 少一层解引用

### 3. 只读参数优先 `const T&`

- 尤其是 `string`、`vector` 这类对象

### 4. 指针先初始化

```cpp
int* p = nullptr;
```

- 不要让指针野着用

---

## 十二、高频易错点

1. `p` 是地址，`*p` 才是值
2. `&x` 是取地址，不是引用声明
3. 空指针不能解引用
4. 引用必须初始化
5. 引用一旦绑定不能改绑
6. 指针传参调用时传地址，引用传参调用时传变量本身
7. `const int*` 和 `int* const` 含义不同
8. 不要把野指针当成正常指针使用

---

## 十三、最简模板

### 指针基础

```cpp
int x = 10;
int* p = &x;
cout << *p << '\\n';
```

### 引用基础

```cpp
int x = 10;
int& r = x;
r = 20;
```

### 引用传参

```cpp
void addOne(int& x) {
	++x;
}
```

### 指针判空

```cpp
if (p != nullptr) {
	cout << *p << '\\n';
}
```

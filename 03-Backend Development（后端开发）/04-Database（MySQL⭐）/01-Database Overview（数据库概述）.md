
## 1. 核心概念

- 数据库（DB）
	- 有组织、可共享的结构化数据集合

- 数据库管理系统（DBMS）
	- 管理数据库的软件（如 MySQL、Oracle）

- 数据库系统（DBS）
	- DB + DBMS + 应用 + DBA 组成的完整系统

## 2. 三级模式结构

- **外模式（用户视图）**
	- 用户可见的部分数据，屏蔽无关信息

- **概念模式（全局逻辑）**
	- 全体数据的逻辑结构，与存储无关

- **内模式（物理存储）**
	- 数据的物理存储方式，与用户无关

- **核心价值**：实现数据独立性，修改存储不影响上层应用

## 3. 数据模型

- **概念模型**：用户视角描述数据，如 E-R 模型
- **逻辑模型**：计算机视角描述数据结构，核心是**关系模型**
    
    - 关系：二维表
    - 元组：表的一行（记录）
    - 属性：表的一列（字段）
    - 主键：唯一标识记录的字段
    
- **物理模型**：描述数据的物理存储细节

## 4. 数据库特点

- 结构化存储、高共享性、强独立性、安全可控、一致性高

## 5. 主流分类

- **关系型（RDBMS）**：SQL 操作，事务强一致（MySQL、Oracle）
- **非关系型（NoSQL）**：适配非结构化数据（MongoDB、Redis）
- **OLTP**：事务处理（电商、银行）
- **OLAP**：数据分析（报表、大数据）

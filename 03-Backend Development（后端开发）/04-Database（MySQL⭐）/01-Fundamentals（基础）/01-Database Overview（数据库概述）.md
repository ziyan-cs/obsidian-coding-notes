
## 1. 核心概念

- **数据 (Data)**：描述事物的符号记录，是数据库中存储的基本对象

- **数据库 (DB)**：长期存储在计算机内、有组织、可共享的数据集合

- **数据库管理系统 (DBMS)**：位于用户与操作系统之间的一层数据管理软件，用于科学地组织和存储数据、高效地获取和维护数据

- **数据库系统 (DBS)**：由数据库、数据库管理系统、应用程序、数据库管理员（DBA）和用户共同组成的系统


## 2. 三级模式结构

### 模式（概念模式）

是数据库中全体数据的逻辑结构和特征的描述，是所有用户的公共数据视图。
	- 一个数据库只有一个
	- 定义了数据项、记录、记录间的联系、数据的完整性和安全性要求

### 外模式（用户模式）
是数据库用户能够看见和使用的局部数据的逻辑结构和特征的描述，是与某一应用有关的数据的逻辑表示。
面向用户，一个数据库可以有多个外模式。
是模式的子集，保障了数据的安全性（用户只能访问其外模式内的数据）。

### 内模式（存储模式）

是数据物理结构和存储方式的描述，定义了数据的存储位置、存储方式和索引结构等。
一个数据库只有一个内模式。
它与物理硬件直接相关

### 两级映射

- **外模式 / 模式映射**：保证了**逻辑独立性**。当模式改变时，只需修改此映射，用户的外模式和应用程序无需改变
- **模式 / 内模式映射**：保证了**物理独立性**。当数据库的物理存储结构改变时，只需修改此映射，模式和应用程序无需改变

## 3. 数据模型

### 概念模型（信息模型）

- 按用户的观点来对数据和信息建模，主要用于数据库设计。
- 常用工具：**E-R 模型（实体 - 联系模型）**，包含实体、属性、联系（1:1, 1:N, M:N）。

### 逻辑模型

- 按计算机系统的观点对数据建模，用于 DBMS 的实现。
- 常见类型：
    
    - **层次模型**：树形结构。
    - **网状模型**：图状结构。
    - **关系模型**：二维表结构（主流，如 MySQL、Oracle）。
    - **面向对象模型**：对象结构。
    

### 物理模型

- 描述数据在系统内部的表示方式和存取方法，与硬件、操作系统相关。

## 4. 数据库特点

- **数据结构化**：数据整体结构化是数据库的主要特征之一，也是与文件系统的根本区别。
- **数据的共享性高，冗余度低，易扩充**：避免数据不一致，节省存储空间，便于应用扩展。
- **数据独立性高**：分为物理独立性和逻辑独立性，由三级模式和两级映射保障。
- **数据由 DBMS 统一管理和控制**：
    
    - 数据的安全性保护
    - 数据的完整性检查
    - 并发控制
    - 数据库恢复
## 5. 主流分类

### 关系型数据库 (RDBMS)

- **核心**：基于关系模型，数据存储在二维表中，通过 SQL 进行操作。
- **代表**：MySQL, PostgreSQL, Oracle, SQL Server, SQLite
- **特点**：支持事务、ACID 特性，适合结构化数据和复杂查询。

### 非关系型数据库 (NoSQL)

- **核心**：不遵循关系模型，为特定场景设计，具有高扩展性和高性能。
- **主要类型**：
    
    - **键值存储**：Redis, Memcached
    - **文档数据库**：MongoDB, CouchDB
    - **列族数据库**：HBase, Cassandra
    - **图数据库**：Neo4j, JanusGraph
    
- **特点**：灵活的数据模型，水平扩展能力强，适合大数据和高并发场景。


- **OLTP**：事务处理（电商、银行）
- **OLAP**：数据分析（报表、大数据）

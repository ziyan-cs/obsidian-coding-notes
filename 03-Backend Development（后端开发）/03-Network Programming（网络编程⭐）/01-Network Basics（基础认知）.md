

# 1. OSI vs TCP/IP  

> 核心考点：层级、对应关系、每层作用、典型协议

### 层级对应关系
| OSI七层 | TCP/IP四层 | 作用简述 | 典型协议/设备 |
|---|---|---|---|
| 应用层 | 应用层 | 面向用户的应用服务 | HTTP/HTTPS、FTP、DNS、SSH |
| 表示层 | - | 数据格式转换、加密解密 | SSL/TLS |
| 会话层 | - | 建立/维护/终止会话 | RPC、NetBIOS |
| 传输层 | 传输层 | 端到端数据传输、端口寻址 | TCP、UDP |
| 网络层 | 网际层 | 路由寻址、跨网段传输 | IP、ICMP、ARP |
| 数据链路层 | 网络接口层 | 帧传输、MAC寻址、差错检测 | 以太网、PPP、交换机 |
| 物理层 | 网络接口层 | 比特流传输、物理介质 | 网线、集线器、网卡 |

### 核心区别
- OSI：理论模型，层次划分更细，教学用
- TCP/IP：工业事实标准，简化为四层，是互联网的事实协议栈

# 2. 数据如何在网络中传输

### 发送端（封装）

- 从上到下逐层加头部

1. 应用层：生成原始数据（如 HTTP 请求）

2. 传输层：加上 TCP/UDP 头部（端口号、序列号等），形成「段」

3. 网络层：加上 IP 头部（IP 地址），形成「包」

4. 数据链路层：加上 MAC 头部（MAC 地址），形成「帧」

5. 物理层：转为比特流，通过网线 / 电磁波传输

### 接收端（解封装）

- 从下到上逐层拆头部

1. 物理层：接收比特流，还原成数据帧

2. 数据链路层：剥去 MAC 头部，交给网络层

3. 网络层：剥去 IP 头部，交给传输层

4. 传输层：剥去 TCP/UDP 头部，校验数据，交给应用层

5. 应用层：最终数据被应用程序读取。

### 关键概念

- 封装：每层为数据加上自己的头部信息
- 解封装：接收端逐层去掉头部，还原原始数据
- PDU：每一层的数据单位，传输层叫「段」，网络层叫「包」，链路层叫「帧」

# 3. IP / Port / MAC

> 核心考点：作用、唯一性、寻址范围、三者配合流程

### 基础定义

- IP 地址：逻辑地址，用来在全网定位主机（机器在哪）
- 端口号：应用层的门牌号，用来在主机内定位进程（哪个程序）
- MAC 地址：物理地址，烧录在网卡中，用来在局域网内定位网卡设备。

### 三者协作流程（以访问网站为例）
1. 应用通过端口号（如80/443）发起请求
2. 传输层TCP/UDP携带端口号
3. 网络层IP携带目标IP地址，跨网段路由
4. 数据链路层MAC负责在同一网段内把数据交给下一跳设备

### 关键对比
| 特性 | IP地址 | 端口号 | MAC地址 |
|---|---|---|---|
| 作用 | 跨网段定位设备 | 定位设备上的应用 | 局域网内定位设备 |
| 范围 | 公网/内网全局 | 单设备内唯一 | 单局域网内唯一 |
| 是否可改 | 可动态/静态修改 | 可动态分配 | 硬件固化（可软件修改） |
| 对应层级 | 网络层 | 传输层 | 数据链路层 |

# 4. DNS 域名服务器

> 递归查询、迭代查询、解析流程、缓存机制

- 域名系统，将域名解析为IP地址
- 解决IP难记、域名易记的矛盾

- 递归查询：客户端只问一次，由本地DNS服务器替它问到底
- 迭代查询：本地DNS服务器一层层问，每次只得到下一级地址
- DNS缓存：各级服务器都会缓存解析结果，提高效率

- 完整解析流程

>客户端搜索域名→浏览器解析缓存→本地Hosts文件→本地DNS缓存→
>本地域名服务器→根域名服务器→顶级域名服务器→权威域名服务器→
>获得 IP 地址→DNS服务器缓存结果→返回给客户端→可访问目标服务器

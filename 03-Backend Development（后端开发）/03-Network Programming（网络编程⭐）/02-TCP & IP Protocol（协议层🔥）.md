


# 1. IP 协议（网络层）
> 核心考点：路由寻址、分片、IP头部关键字段

### 核心作用
- 无连接、不可靠的**数据报协议**，负责跨网段路由寻址
- 为上层（TCP/UDP）提供统一的寻址和传输服务

### 路由寻址
- 依靠IP地址和路由表，决定数据包从源主机到目标主机的传输路径
- 经过路由器转发，逐跳传递，最终到达目标网段

### 分片（Fragmentation）
- 当数据包长度超过链路层的 MTU（最大传输单元）时，IP 会把数据包拆分成多个分片传输
- 目标主机收到后再重组，还原原始数据报
- 分片带来额外开销，现在通常用 TCP 的 MSS 协商避免不必要分片

# 2. TCP 协议（传输层，重点）
> 核心考点：三次握手/四次挥手、滑动窗口、拥塞控制、重传机制

### TCP 核心特性
- 面向连接、可靠传输、有序交付、全双工通信
- 解决：丢包、乱序、重复、流量控制、拥塞控制问题

---

### 三次握手（建立连接）
- 目的：确认双方的发送和接收能力，同步序列号
- 流程：
  1. 客户端发送 `SYN` 包（seq=x），请求建立连接
  2. 服务端回复 `SYN+ACK` 包（seq=y, ack=x+1），同意连接并确认客户端请求
  3. 客户端回复 `ACK` 包（ack=y+1），连接正式建立

- 关键理解：三次握手是为了**双向确认**，保证双方收发能力正常

---

### 四次挥手（断开连接）
- 目的：双方都确认数据发送完毕，安全关闭连接
- 流程：
  1. 客户端发送 `FIN` 包，请求关闭连接（客户端不再发送数据）
  2. 服务端回复 `ACK` 包，确认关闭请求（客户端进入 FIN-WAIT）
  3. 服务端发送 `FIN` 包，请求关闭服务端方向的连接（服务端数据发送完毕）
  4. 客户端回复 `ACK` 包，服务端收到后关闭连接（客户端等待2MSL后关闭）

- 关键理解：因为 TCP 是全双工，所以每个方向都要独立关闭

---

### 滑动窗口（流量控制）
- 作用：解决发送方发送过快，接收方处理不过来的问题
- 原理：接收方通过 `window` 字段告诉发送方自己的接收缓冲区大小
- 发送方根据窗口大小控制发送速率，避免接收方溢出
- 核心：动态调整窗口大小，实现“边收边调”

---

### 拥塞控制
- 作用：解决网络链路拥塞，避免大量数据同时发送导致网络瘫痪
- 核心算法（四个阶段）：
  1. **慢启动**：窗口从1开始指数增长，探测网络可用带宽
  2. **拥塞避免**：窗口线性增长，避免过快
  3. **快重传**：收到三次重复ACK，认为丢包，直接重传
  4. **快恢复**：不回到慢启动，直接从新阈值线性增长

---

### 重传机制
- 目的：保证数据可靠到达，解决丢包问题
- 触发方式：
  1. **超时重传**：数据包发送后，在规定时间内未收到ACK，认为丢包，重新发送
  2. **快速重传**：收到三次重复ACK，提前判断丢包，立即重传，无需等待超时

---

# 3. UDP 协议（传输层）
> 核心考点：无连接、不可靠、适用场景

### 核心特性
- 无连接：无需握手，直接发送数据
- 不可靠：不保证到达、不保证顺序、不重传
- 开销极小，延迟极低

### 适用场景
- 实时性优先、可容忍少量丢包的场景：
  - 网络游戏、视频通话、直播、语音聊天
  - DNS查询（单次请求/响应，丢了重发即可）

# 4. HTTP 协议（应用层，顺带）
> 核心考点：HTTP/1.1 vs HTTP/2 vs HTTP/3，常见状态码

### 基础定义
- 超文本传输协议，基于 TCP/IP 传输，用于客户端与服务器之间的网页数据交互
- 无状态协议：服务器不保存客户端会话信息（需 Cookie/Session 维持状态）

### 关键版本区别
| 版本       | 底层协议      | 核心特性                |
| -------- | --------- | ------------------- |
| HTTP/1.1 | TCP       | 长连接、管线化、无多路复用       |
| HTTP/2   | TCP       | 二进制分帧、多路复用、头部压缩     |
| HTTP/3   | QUIC（UDP） | 0-RTT握手、多路复用、队头阻塞解决 |
|          |           |                     |

### 常见状态码
- `200 OK`：请求成功
- `301/302`：永久/临时重定向
- `400 Bad Request`：请求错误
- `403 Forbidden`：无权限
- `404 Not Found`：资源不存在
- `500 Internal Server Error`：服务器内部错误

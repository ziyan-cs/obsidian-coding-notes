## ⚡ TL;DR（快速决策）

- Linux 基础命令的核心是：**通过命令行直接操作文件、目录、进程和系统环境**
- 学 Linux 不是背命令表，而是建立“系统资源如何被命令操作”的整体理解
- 若只抓重点，应优先记住：
    - 路径与目录
    - 文件查看与编辑
    - 权限
    - 进程观察
    - 管道与重定向

## 🧩 Core Idea（核心本质）

- Shell 是人与操作系统交互的重要入口
- 命令本质上是在操作：
    - 文件系统
    - 进程
    - 权限
    - 输入输出流
- 一句话理解：
    - **Linux 基础命令就是系统资源的文本化操作接口。**

## 🏗️ 关键命令类别

1. 路径与目录

- `pwd`：查看当前路径
- `ls`：列出目录内容
- `cd`：切换目录
- `mkdir` / `rmdir`：创建 / 删除目录

1. 文件查看与编辑

- `cat`、`less`、`head`、`tail`
- `cp`、`mv`、`rm`
- `touch`

1. 权限与属性

- `chmod`
- `chown`
- `stat`

1. 进程与系统观察

- `ps`
- `top` / `htop`
- `kill`

1. 重定向与管道

- `>`、`>>`、`<`
- `|`

## 🔧 Usage Patterns（可复用理解框架）

1. 用“对象 + 动作”理解命令

- 对象：文件、目录、进程
- 动作：查看、修改、移动、删除、过滤

1. 用“组合”理解命令行价值

- 单个命令解决单点问题
- 管道把多个命令串成工作流

1. 用“标准输入 / 输出 / 错误”理解重定向

- 很多 Linux 工具天然能组合，是因为它们遵守统一 I/O 约定

## ⚠️ Pitfalls（高频错误）

- 不理解相对路径与绝对路径
- 在 `rm -rf` 等危险命令上缺乏谨慎
- 只记命令名字，不理解底层资源对象
- 不理解重定向和管道，导致命令不会组合

## 🚀 Performance / Tips（理解深化）

- Linux 命令行的真正价值在于可组合性和自动化能力
- 学命令时，优先掌握高频命令族和通用思维，而不是追求一次记全
- 后续脚本、系统编程、运维排查都建立在这层基础之上

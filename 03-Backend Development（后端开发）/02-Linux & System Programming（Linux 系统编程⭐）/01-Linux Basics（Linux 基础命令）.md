## 核心

- **通过命令行直接操作文件、目录、进程和系统环境**
- 为建立 “系统资源如何被命令操作” 的整体理解
- 重点：
    - 路径与目录
    - 文件查看与编辑
    - 权限
    - 进程观察
    - 管道与重定向

- Shell 是人与操作系统交互的重要入口
- 命令本质上是在操作：
    - 文件系统
    - 进程
    - 权限
    - 输入输出流

##  关键命令类别

### 1. Vim 快捷键

- `Vim 文件名`：打开/新建文件

- `i`：进入插入模式
- `:`：进入底行命令模式
- `Esc`：回退底行命令模式

- `yy`：复制当前行
- `p`：粘贴剪贴板内容
- `dd`：删除（剪切）当前行
- `u`：撤销上一步操作
- `Ctrl + r`：重做撤销的操作

- `gg`：跳转到文件首行
- `G`：跳转到文件尾行
- `^`：跳转到行首
- `$`：跳转到行尾

- `:w`：保存文件
- `:q`：退出 Vim
- `:q!`：强制退出不保存
- `:wq`：保存并退出



### 2. 用户管理（需 root 权限）

- `useradd 用户名`：创建新用户（默认：`/home/用户名` ）
- `useradd -d /home/*** 用户名`： 指定家目录路径
- `useradd -g 用户组 -u 用户名`： 指定用户组
- `adduser 用户名`:  Ubuntu 强烈推荐 
- `userdel 用户名`：删除用户（保留家目录，无法登录）
- `userdel -r 用户名`：删除用户（删除家目录）

- `su 用户名`：切换用户
- `logout`：退回最初用户
- `exit`：退出进程
- `Whoami`：显示当前身份
- `Who am i`：显示最初身份
- `id`：查看当前用户 `UID GID 组`
- `id 用户名`：查看用户 `UID GID 组`

- `passwd`：修改当前用户密码
- `passwd 用户名`：修改用户密码
- `usermod 用户名`：修改用户属性

- `groupadd`：创建用户组
- `groupdel`：删除用户组
- `groupdel`：删除用户组
- `who`：查看当前登录用户
- `w`：查看登录用户及操作
- `su`：切换用户身份
- `sudo`：以管理员权限执行命令

### 3. 路径与目录

- `pwd`：查看当前路径
- `ls`：列出目录内容
- `cd`：切换目录
- `mkdir` / `rmdir`：创建 / 删除目录

3. 文件查看与编辑

- `cat`：查看文件内容
- `less`：分页查看大文件
- `head`：查看文件头部内容
- `tail`：查看文件尾部内容
- `cp`：复制文件 / 目录
- `mv`：移动 / 重命名文件 / 目录
- `rm`：删除文件 / 目录
- `touch`：创建空文件 / 修改文件时间

4. 权限与属性

- `chmod`：修改文件 / 目录权限
- `chown`：修改文件 / 目录所有者
- `stat`：查看文件详细属性信息

1. 进程与系统观察

- `ps`：查看进程状态
- `top / htop`：实时监控系统进程 / 资源
- `kill`：终止指定进程

1. 重定向与管道

- `>、>>、<`：输出 / 输入重定向
- `|`：管道符，连接命令输出与输入

## 🔧 Usage Patterns（可复用理解框架）

1. 用“对象 + 动作”理解命令

- 对象：文件、目录、进程
- 动作：查看、修改、移动、删除、过滤

1. 用“组合”理解命令行价值

- 单个命令解决单点问题
- 管道把多个命令串成工作流

1. 用“标准输入 / 输出 / 错误”理解重定向

- 很多 Linux 工具天然能组合，是因为它们遵守统一 I/O 约定

## ⚠️ Pitfalls（高频错误）

- 不理解相对路径与绝对路径
- 在 `rm -rf` 等危险命令上缺乏谨慎
- 只记命令名字，不理解底层资源对象
- 不理解重定向和管道，导致命令不会组合

## 🚀 Performance / Tips（理解深化）

- Linux 命令行的真正价值在于可组合性和自动化能力
- 学命令时，优先掌握高频命令族和通用思维，而不是追求一次记全
- 后续脚本、系统编程、运维排查都建立在这层基础之上

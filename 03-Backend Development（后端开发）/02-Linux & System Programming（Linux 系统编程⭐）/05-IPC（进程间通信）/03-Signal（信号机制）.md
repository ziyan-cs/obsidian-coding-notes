## ⚡ TL;DR（快速决策）

- 信号机制的核心是：**操作系统向进程异步通知某类事件已经发生**
- 信号不是传大量数据，而是传递“发生了什么事”的通知语义
- 若只抓重点，应优先记住：
    - 异步
    - 通知
    - 处理中断式打断
    - 默认行为 / 自定义处理

## 🧩 Core Idea（核心本质）

- 进程执行过程中可能被某个事件打断并触发信号处理逻辑
- 一句话理解：
    - **信号是进程级异步事件通知机制。**

## 🏗️ 关键概念

1. 常见信号

- `SIGINT`
- `SIGTERM`
- `SIGKILL`
- `SIGSEGV`
- `SIGCHLD`

1. 处理方式

- 默认处理
- 忽略
- 自定义信号处理函数

1. 异步性

- 信号到达时间不由当前代码线性控制

## 🔧 Usage Patterns（可复用理解框架）

1. 用“通知”理解信号，而不是“消息队列”
2. 用“打断当前流程”理解其异步特征
3. 用 `SIGCHLD` 理解父进程对子进程退出事件的响应

## ⚠️ Pitfalls（高频错误）

- 在信号处理函数里做过多复杂逻辑
- 把信号当作通用数据传输机制
- 混淆可捕获信号和不可捕获信号（如 `SIGKILL`）

## 🚀 Performance / Tips（理解深化）

- 信号适合事件通知，不适合复杂数据交换
- 学信号时，重点应放在异步打断语义和安全处理约束上
- 它与进程控制、僵尸回收、终止控制密切相关

## 🧪 Common Scenarios（常见使用场景）

- Ctrl+C 中断程序
- 优雅终止服务
- 子进程退出通知
- 异常错误响应

## 🧾 Minimal Template（最小理解模板）

```
某事件发生 -> 内核发送信号 -> 进程按默认或自定义方式处理
```

## 📌 One-liner Summary（一句话总结）

- **信号机制的核心，是由操作系统向进程异步传递事件通知并触发相应处理。**

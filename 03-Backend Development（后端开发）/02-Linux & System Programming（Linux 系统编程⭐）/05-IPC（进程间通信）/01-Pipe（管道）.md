## ⚡ TL;DR（快速决策）

- 管道的核心是：**在进程之间建立顺序字节流通信通道**
- 它是 Unix/Linux 中最经典的 IPC 机制之一
- 若只抓重点，应优先记住：
    - 匿名管道通常用于有亲缘关系进程
    - 管道默认半双工
    - 本质是内核缓冲区 + 文件描述符接口

## 🧩 Core Idea（核心本质）

- 一个进程写入管道，另一个进程从管道读取
- 一句话理解：
    - **管道就是内核提供的字节流型进程间通信通道。**

## 🏗️ 关键概念

1. 匿名管道

- 常配合 `fork` 使用
- 适合父子进程通信

1. 命名管道（FIFO）

- 通过文件系统路径存在
- 无亲缘关系进程也可通信

1. 读端与写端

- 一端负责读，一端负责写

## 🔧 Usage Patterns（可复用代码模板）

```cpp
int fd[2];
pipe(fd);
// fd[0] 读端，fd[1] 写端
```

## ⚠️ Pitfalls（高频错误）

- 不关闭不需要的读写端
- 忽略管道容量有限和阻塞行为
- 把管道误认为天然双向通道

## 🚀 Performance / Tips（理解深化）

- 管道非常适合顺序流式通信，但不适合复杂共享状态场景
- 理解管道后，更容易看懂 Shell 管道和父子进程协作

## 🧪 Common Scenarios（常见使用场景）

- Shell 中的 `|`
- 父子进程数据传递
- 简单流式 IPC

## 🧾 Minimal Template（最小理解模板）

```
进程 A 写入 -> 内核管道缓冲区 -> 进程 B 读取
```

## 📌 One-liner Summary（一句话总结）

- **管道的核心，是通过内核字节流缓冲区在进程间传递顺序数据。**

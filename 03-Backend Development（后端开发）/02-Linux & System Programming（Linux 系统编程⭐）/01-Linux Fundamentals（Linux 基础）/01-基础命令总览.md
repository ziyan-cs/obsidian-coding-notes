

# 0. 核心

- `Shell` 是 **用户与操作系统内核交互的命令行解释器**
- `sudo [cmd]`：以 `root` 权限执行命令（*Ubuntu* 常用）
- 命令本质是操作四大核心对象：
    - 文件系统（目录 / 文件）
    - 进程
    - 权限
    - 输入输出流

# 1. 帮助命令

- `type [cmd]`： 判断命令类型（ `builtin`  / `external` ）

- `builtin` 内置命令：
	- `Shell` 自带，不创建新进程，直接作用于当前 `Shell`
	- `help [builtin_cmd]`： 只能查看内置命令
	- 代表：`cd` `help` `type` `echo` `export`

- `external` 外部命令：
	- 独立二进制文件，`fork` 子进程执行
	- `man [external_cmd / config_file]`：只能查看外部命令/配置文件
	- `[external_cmd]--help`：只能查看外部命令
	- 代表：`ls` `cat` `find` `date` `which`

# 2. 关机 & 重启

- 关机
	- `halt`：立即关机
	- `reboot`：立即重启
	- `sync`：同步内存数据到磁盘
	- `poweroff`：立即关机

- 重启
	- `shutdown`：默认一分钟后关机
	- `shutdown -h now`：立即关机
	- `shutdown -h 1`：一分钟后关机（可自定义时间）
	- `shutdown -r now`：立即重启

# 3. 运行级别

- 传统运行级别：
	- `0`：关机
	- `1`：单用户模式【找回丢失密码】
	- `2`：多用户状态无网络服务
	- `3`：多用户状态有网络服务
	- `4`：系统未使用保留给用户
	- `5`：图形界面（默认）
	- `6`：系统重启
	- `init [级别]`：切换运行级别（例如  `init 3`）

- 现代 *Ubuntu* 等系统用 `systemd 目标（target）` 替代传统运行级别，`init` 命令仅做兼容：
	- `0` ↔ `poweroff.target`（关机）
	- `3` ↔ `multi-user.target`（多用户命令行，服务器常用）
	- `5` ↔ `graphical.target`（图形界面，桌面常用）
	- `6` ↔ `reboot.target`（重启）
	- `systemctl get-default`：查看当前运行级别
	- `systemctl set-default 对应运行级别` ：切换到对应运行级别

# 4. 文件目录指令

### 4.1 目录导航与查看

- 导航
	- `cd`：回到家目录
	- `cd ~`：回到家目录
	- `cd [path]`：切换到指定目录
	- `cd ..`：回到工作目录上一级目录

- 查看
	- `pwd`：查看当前工作目录绝对路径
	- `ls`：列出当前工作目录的文件和目录
	- `ls -l`：（以列表形式列出）
	- `ls -h`：（文件大小按单位显示）
	- `ls -a`：（列出所有的，包括以 `.` 开头隐藏的）

### 4.2 目录/文件创建

- 新建
	- `mkdir [path]`：创建单级目录
	- `mkdir -p [path]`：创建多级目录（自动创建不存在的父目录）
	- `touch [file]`：创建空文件

### 4.3 目录/文件复制移动

- 复制
	- `cp [src] [dst]`：复制文件/目录
	- `cp -r [src] [dst]`：递归复制整个目录
	- `cp -rf [src] [dst]`：递归复制并强制覆盖（无需确认）
	- `/cp -r [src] [dst]`：递归复制并强制覆盖（无需确认）

- 移动
	- `mv [src] [dst]`：移动/重命名文件/目录
	- `mv -f [src] [dst]`：强制移动/重命名（⚠️ 强制覆盖，谨慎使用）

### 4.4 文件内容查看

- 全量查看（小文件）
	- `cat [file]`：输出小文件的完整内容
	- `cat -n [file]`：输出文件内容，并显示行号

- 节选查看
	- `head [file]`：输出文件头部内容（默认前 10 行）
	- `head -n [num] [file]`：输出文件头部指定行数内容
		- 如 `head -n 5 [file]`：查看前 5 行
		
	- `tail [file]`：输出文件尾部内容（默认显示后 10 行）
	- `tail -n [num] [file]`：输出文件尾部指定行数内容
		- 如 `tail -n 5 [file]`：查看后 5 行

- 实时监控（日志文件）
	- `tail -f [file]`：实时追踪文件尾部更新内容（常用于日志）
		- 持续占用当前终端进程，不返回命令行，无法输入新命令
		- 需打开 **多终端 / 多窗口** 执行后续操作
		- `Ctrl + C`：退出执行中的命令

- 分页查看（大文件）
	- `more [file]`：分页查看文件（仅支持向下翻页）
		- `q`：退出 `more` 查看
		
	- `less [file]`：分页查看大文件（支持上下翻页、搜索）
	    - `PageDown / Space`：向下翻动一页
	    - `PageUp / b`：向上翻动一页
	    - `↓ / Enter`：向下翻动一行
	    - `↑`：向上翻动一行
	    - `g`：跳转到文件开头
	    - `G`：跳转到文件末尾
	    - `/[关键字]`：向下搜索关键字（n 下一个，N 上一个）
	    - `?[关键字]`：向上搜索关键字
	    - `q`：退出 `less` 查看

- 输出与重定向
	- `echo "内容"`：输出指定内容到控制台
    - `echo [环境变量]`：查看系统环境变量
		- `$PATH`：命令搜索路径
		- `$HOME`：当前用户家目录
		- `$USER`：登录用户名
		- `$PWD`：当前所在目录
		-  `$SHELL`：当前使用的 shell
		- `$LANG`：系统语言
		- `$HOSTNAME`：主机名
	- `echo "内容" > file`：输出重定向（覆盖文件原有内容）
	- `echo "内容" >> file`：追加（在文件末尾新增内容，不覆盖）

### 4.5 压缩与解压

- **打包压缩（最常用，.tar.gz 格式）**
	- `tar -zcvf [bag].tar.gz [src]：打包并压缩文件/目录
	- `tar -zxvf 压缩包名.tar.gz`：解压 .tar.gz 包到当前目录
	- `tar -zxvf 压缩包名.tar.gz -C 目标路径`：解压到指定目录
	- `tar -ztvf 压缩包名.tar.gz`：查看包内文件，不解压

- **仅打包（不压缩，.tar 格式）**
	- `tar -cvf 打包名.tar [src]`：仅归档，不压缩
	- `tar -xvf 打包名.tar`：解压 .tar 包

- **单文件压缩（.gz 格式）**
	- `gzip 文件名`：压缩单个文件，原文件替换为 .gz
	- `gzip -k 文件名`：压缩并保留原文件
	- `gunzip 压缩包名.gz`：解压 .gz 文件，恢复原文件

- **跨平台压缩（.zip 格式）**
	- `zip -r 压缩包名.zip 源文件/目录`：递归压缩目录/文件
	- `unzip 压缩包名.zip`：解压到当前目录
	- `unzip 压缩包名.zip -d 目标路径`：解压到指定目录
	- `unzip -l 压缩包名.zip`：查看包内文件，不解压

- **高压缩率（.bz2 格式）**
	- `tar -jcvf 压缩包名.tar.bz2 源文件/目录`：打包并高压缩
	- `tar -jxvf 压缩包名.tar.bz2`：解压 .tar.bz2 包

- **关键注意事项**
	- `tar` 的 `-f` 参数必须放在最后，紧跟压缩包名
	- `tar` 操作默认保留原文件，`gzip` 默认删除原文件（需 `-k` 保留）
	- 跨平台传输优先用 `.tar.gz` 或 `.zip`，`.tar.bz2` 适合归档大文件
	- 解压前建议先查看包内文件，避免覆盖同名文件

### 4.6 目录/文件删除与属性

- 删除
	- `rmdir [path]`：删除指定空目录
	- `rm -r [path]`：递归删除整个目录（含非空目录）
	- `rm -f [path]`：强制删除文件/目录（⚠️ 谨慎使用）
	- `rm -rf [path]`：强制递归删除整个目录（⚠️ 谨慎使用）
	
- 属性
	- `chown [user:group] [file/dir]`：修改文件/目录所有者
	- `chown -R [user:group] [path]`：递归修改目录下所有文件/目录的所有者
	- `stat [file/dir]`：查看文件详细属性信息

### 4.7 软连接和硬连接

- 软连接
	- `ln -s [src] [dst]`：给原文件/目录创建一个软连接（快捷方式）
	- `rm -rf [dst]`：删除软连接（末尾不加 `/`，防误删源目录）
		- 删除 **软连接** ，不影响源文件

- 硬连接
	- `ln [src] [dst]`：给原文件创建一个硬连接（文件备份）
	- `rm -rf [dst]`：删除硬连接
		- 仅当 **所有硬链接** + **原文件** 都删除时，数据真正释放

### 4.8 历史命令

- `history`：显示所有的历史命令
- `history 10`：显示最近使用过的 10 个指令
- `history -c`：清空当前所有历史命令（⚠️ 谨慎使用）
- `!`：执行上一条命令
- `!5`：执行历史编号为 5 的指令
- `!!`：重复执行上一条命令

### 4.9 时间日期命令

- `cal`：显示本月日历
- `ncal`：垂直显示日历
- `cal -3`：显示 **上月** + **本月** + **下月** 三连日历
- `cal -m 3`：显示指定月份日历
- `cal -m 3 2020`：显示指定年份月份日历
- `cal -j`：显示是一年中的第几天
- `cal -w`：显示是一年中的第几周
- `cal 2020`：显示 `2020` 年所有月份日历

- `date`：显示当前完整时间
- `date "+%Y-%m-%d %H:%M:%S"`：显示自定义格式时间
	- `date +%Y`：完整年份（2020）
	- `date +%y`：两位年份（20）
	- `date +%m`：月（01-12）
	- `date +%d`：日（00-31）
	- `date +%H`：时（00-23）
	- `date +%M`：分（00-59）
	- `date +%S`：秒（00-59）
	- **多空格 / 多参数** 必须加引号（单 / 双都可以）
- `sudo date -s "字符串具体时间"`：设置当前时间
	- `hwclock -w`：同步网络时间（设置时间后必做）
	- `ntpdate ntp.aliyun.com`：`ntp` 同步时间（推荐）

# 7. 管道命令

### 7.1 原理
- `[命令] | [命令]`：将前一个命令的 **标准输出**，作为后一个命令的 **标准输入**
	- 可无限串联，实现命令链式处理
	- 仅处理标准输出，不处理标准错误

### 7.2 常用场景

- 文本过滤
    - `cat [file] | grep "hello"`：查看文件，过滤包含"hello"的行
    - `ps aux | grep "关键字"`：查看进程，过滤nginx相关进程

- 统计计数
    - `ls -l | grep ".cpp" | wc -l`：统计当前目录下 `.cpp` 文件数量

- 分页查看
    - `cat long.txt | less`：长文件分页查看

- 多管道串联
    - `cat app.log | grep "ERROR" | wc -l | less`：过滤日志错误，统计数量，分页显示

### 7.3  关键注意

- 前一个必须有 **输出**，后一个必须能接收 **输入**
- 仅传递 **标准输出**，**错误输出** 需用 `2>&1` 重定向
- 避免无意义的管道嵌套，保证命令可读性



# 0. 核心

-  编写 `CMakeLists.txt`：配置编译规则、C++ 标准、调试信息
- **插件自动解析**：VSCode CMake 插件读取配置，自动生成调试环境
- **一键调试运行**：点击左下角按钮，直接启动调试 / 运行

# 1. 环境配置

- 确保 `CMakeLists.txt` 配置：

```cpp
# 强制指定使用 C++17 标准
set(CMAKE_CXX_STANDARD 17)
# 生成compile_commands.json，供 IDE 做代码补全、跳转
set(CMAKE_EXPORT_COMPILE_COMMANDS ON)
```
  
# 2. VSCode 使用

- ⚙**生成 (Build)** ：
    - 作用：执行 `cmake --build .`
    - 效果：自动编译代码，生成  `文件名/项目名/main.exe` 可执行文件到`build` 目录

- 🐞**调试 (Debug)** ：
    - 作用：一键启动调试。
    - 效果：自动加载 `launch.json` 配置，程序运行并暂停在断点处

- ▶️**运行 (Run)** ：
    - 作用：直接运行程序（无需调试，查看输出）

##### 3. 调试面板（Ctrl+Shift+P -> 开始调试）

| 操作       | 简写 / 按钮  | 作用                                 |
| :------- | :------- | :--------------------------------- |
| **断点**   | 🔴       | 在代码行号左侧点击， toggles 断点。             |
| **继续**   | F5 / ▶️  | 程序运行到下一个断点或结束。                     |
| **单步跳过** | F10 / ⏭️ | 执行一行代码，不进入函数内部。                    |
| **单步进入** | F11 / ↓  | 执行一行代码，**进入**函数内部查看。               |
| **查看变量** | 左侧面板     | 自动监控当前作用域的变量值，**完全可视化**，不用 `p` 命令。 |
| **修改变量** | 左侧面板     | 直接双击变量值修改，实时生效。                    |

# 4. 常见场景

- **代码红波浪线 / 跳转失效**：
    - 检查 `CMAKE_EXPORT_COMPILE_COMMANDS ON` 是否配置。
    - 检查 VSCode 右下角是否提示 “已安装 CMake 工具”。

- **调试报错**：
    - 确保编译时没有加 `-O2` 等优化选项（Debug 模式下默认不加）。


# 0. 核心认知

- **GDB**：Linux 下 C/C++ 程序的**命令行调试工具**，用于定位代码 bug、查看运行状态
- **前提**：编译时必须加 `-g` 参数，生成调试信息（CMake 已自动支持，无需额外配置）

# 1. 基础准备

### 1.1 编译要求

- 手动编译：`g++ -g main.cpp -o main`
- CMake 项目：无需额外操作，`CMAKE_BUILD_TYPE=Debug` 自动生成调试信息（默认配置）

### 1.2 启动 GDB

```cpp
# 启动调试（替换为你的可执行文件名）
gdb ./可执行文件名
# 示例：gdb ./main
```

# 2. 高频核心命令

| 命令          | 简写   | 作用                   |
| :---------- | :--- | :------------------- |
| `run`       | `r`  | 启动 / 重新运行程序          |
| `break 行号`  | `b`  | 在指定行设置断点             |
| `break 函数名` | `b`  | 在函数入口设置断点            |
| `next`      | `n`  | 单步跳过（执行一行，不进入函数内部）   |
| `step`      | `s`  | 单步进入（执行一行，进入函数内部）    |
| `continue`  | `c`  | 继续运行到下一个断点           |
| `print 变量名` | `p`  | 打印变量当前值              |
| `list`      | `l`  | 查看源码（默认显示当前行附近 10 行） |
| `backtrace` | `bt` | 查看函数调用栈（定位崩溃位置）      |
| `quit`      | `q`  | 退出 GDB               |

# 3. 标准调试流程

```cpp
# 1. 编译（CMake项目直接make，无需额外操作）
cd build && make

# 2. 启动GDB
gdb ./main

# 3. 设置断点（示例：在main函数第5行打断点）
(gdb) b main
(gdb) b 10

# 4. 运行程序
(gdb) r

# 5. 单步调试/查看变量
(gdb) n  # 单步跳过
(gdb) s  # 单步进入
(gdb) p 变量名  # 查看变量值

# 6. 继续运行/退出
(gdb) c  # 继续到下一个断点
(gdb) q  # 退出
```
